6. 학습할 코드 1개

프로젝트 단계별 일정과 진행률을 관리하는 계획 도구

project = {
    "title": "교실 공기질 알림 시스템",
    "goal": "이산화탄소 농도를 분석해 환기 시점을 안내한다.",
    "tasks": [
        {"name": "문제 분석", "days": 2, "done": True},
        {"name": "데이터 수집 계획", "days": 3, "done": True},
        {"name": "알고리즘 설계", "days": 3, "done": False},
        {"name": "프로그램 구현", "days": 5, "done": False},
        {"name": "테스트와 개선", "days": 3, "done": False},
        {"name": "결과 공유", "days": 2, "done": False}
    ]
}


def summarize_plan(project):
    total_days = sum(task["days"] for task in project["tasks"])

    completed_days = sum(
        task["days"]
        for task in project["tasks"]
        if task["done"]
    )

    progress = completed_days / total_days * 100

    print("프로젝트:", project["title"])
    print("목표:", project["goal"])
    print("예상 기간:", total_days, "일")
    print("진행률:", round(progress, 1), "%")

    print("\n단계별 계획")
    for index, task in enumerate(project["tasks"], start=1):
        status = "완료" if task["done"] else "예정"
        print(index, task["name"], task["days"], "일", status)


summarize_plan(project)

상세 설명

코드 요소설명
project프로젝트 제목, 목표, 단계별 할 일을 하나의 딕셔너리로 구조화한다.
tasks프로젝트 수행 단계를 리스트로 저장하며 각 단계는 이름, 기간, 완료 여부를 가진다.
total_days모든 단계의 예상 소요 기간을 합하여 전체 프로젝트 기간을 계산한다.
completed_days완료된 단계의 기간만 합하여 실제 진행된 작업량을 나타낸다.
if task["done"]완료 여부가 참인 단계만 진행률 계산에 포함한다.
progress완료된 기간을 전체 기간으로 나누어 백분율 진행률을 구한다.
enumerate(..., start=1)각 프로젝트 단계에 1번부터 순서를 붙인다.
조건 표현식완료 여부에 따라 ‘완료’ 또는 ‘예정’을 출력해 계획 상태를 쉽게 확인한다.

확장 활동: 담당자, 시작일, 종료일, 필요한 데이터, 예상 위험, 검증 기준 항목을 딕셔너리에 추가하면 더 완성도 높은 프로젝트 관리 도구가 된다.

7. 수업 끝 멘트

“시는 첫 문장을 품고 끝 행을 향해 가며, 소설은 인물의 선택과 사건의 순서를 설계해 하나의 세계를 완성합니다. 희곡은 막이 오르기 전부터 무대와 동선을 준비하고, 연극은 보이지 않는 수많은 연습이 있어야 관객 앞에서 빛납니다. 음악도 즉흥처럼 들리는 순간 뒤에는 박자와 화성의 질서가 숨어 있습니다. 여러분의 꿈도 계획표에 모두 담기지는 않지만, 한 걸음의 방향을 정하는 설계는 반드시 힘이 됩니다. 단, 계획서에 ‘1단계 계획 세우기, 2단계 다시 계획 세우기’만 적으면 프로젝트가 아니라 계획의 무한 재귀입니다!”
