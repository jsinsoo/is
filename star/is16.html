상태·연산·목표 상태 구조화
모든 후보 탐색 알고리즘 예제 10선

문제를 현재 상태, 가능한 연산, 목표 상태로 나누고 모든 후보를 빠짐없이 생성·검사하는 완전 탐색과 백트래킹 알고리즘을 설계합니다.

설계 원리1. 비밀번호2. 두 수 합 3. 세 수 합4. 순열5. 조합 6. 부분집합7. 동전 조합8. 좌석 배치 9. 미로 후보10. 종합 백트래킹

1. 모든 후보 탐색 설계 절차

상태현재까지 선택한 값, 위치, 합계 등 문제의 현재 모습을 정의합니다.
연산현재 상태에서 수행할 수 있는 선택과 이동을 정의합니다.
목표 상태정답으로 인정할 조건이나 탐색을 멈출 조건을 정합니다.
후보 탐색가능한 선택을 모두 시도하고 조건을 만족하는 후보를 기록합니다.
완전 탐색은 후보 수가 작을 때 모든 경우를 검사하여 정답을 확실히 찾는 방법입니다. 후보 수가 커지면 불가능한 선택을 일찍 중단하는 가지치기와 선택을 되돌리는 백트래킹이 필요합니다.
설계 질문확인 내용
상태는 무엇인가?현재 인덱스, 선택 목록, 누적 합, 방문 위치 등
가능한 연산은 무엇인가?선택·미선택, 이동, 숫자 추가, 자리 교환 등
목표 상태는 무엇인가?목표 합, 선택 개수, 도착 지점, 모든 자리 결정 등
가지치기 조건은 무엇인가?합이 목표 초과, 중복 선택, 경계 밖, 충돌 발생 등
예제 1

세 자리 비밀번호 모든 후보 탐색

중첩 반복문완전 탐색
target = "314"
count = 0

for a in range(10):
    for b in range(10):
        for c in range(10):
            candidate = f"{a}{b}{c}"
            count += 1

            if candidate == target:
                print("찾은 비밀번호:", candidate)
                print("검사 횟수:", count)
                break

문제 구조

상태: 현재까지 결정한 세 자리 숫자

연산: 각 자리에 0~9 선택

목표: 후보가 314와 일치

예제 2

목표 합을 만드는 두 수 찾기

두 후보 선택인덱스
numbers = [2, 7, 11, 15, 3, 6]
target = 9
solutions = []

for i in range(len(numbers)):
    for j in range(i + 1, len(numbers)):
        if numbers[i] + numbers[j] == target:
            solutions.append(
                (numbers[i], numbers[j])
            )

print("해:", solutions)
중복 방지
j를 i+1부터 시작하여 같은 원소를 두 번 선택하거나 순서만 다른 중복 후보를 만들지 않습니다.
예제 3

세 수의 합이 n이 되는 경우 세기

세 변수모든 경우
n = 8
count = 0
solutions = []

for a in range(1, n + 1):
    for b in range(1, n + 1):
        for c in range(1, n + 1):
            if a + b + c == n:
                count += 1
                solutions.append((a, b, c))

print("경우의 수:", count)
print("일부 해:", solutions[:10])

상태와 목표

상태는 현재 선택한 (a, b, c)입니다.

가능한 모든 양의 정수 후보를 검사합니다.

합이 n이면 목표 상태입니다.

예제 4

재귀함수로 모든 순열 생성

순열백트래킹
def generate_permutations(items, selected):
    if len(selected) == len(items):
        print(selected)
        return

    for item in items:
        if item in selected:
            continue

        selected.append(item)
        generate_permutations(items, selected)
        selected.pop()

generate_permutations(["A", "B", "C"], [])
백트래킹
후보를 하나 선택하고 재귀 호출한 뒤 pop()으로 선택을 취소하여 다음 후보를 탐색합니다.
예제 5

n개 중 r개 조합 생성

조합시작 인덱스
def generate_combinations(
    items, r, start=0, selected=None
):
    if selected is None:
        selected = []

    if len(selected) == r:
        print(selected)
        return

    for index in range(start, len(items)):
        selected.append(items[index])

        generate_combinations(
            items,
            r,
            index + 1,
            selected
        )

        selected.pop()

generate_combinations(
    ["A", "B", "C", "D"],
    2
)
상태: 현재 선택 목록과 시작 인덱스
목표: 선택한 원소 수가 r개
예제 6

부분집합 전체 생성

포함·제외이진 선택
def generate_subsets(
    items, index=0, selected=None
):
    if selected is None:
        selected = []

    if index == len(items):
        print(selected)
        return

    # 현재 원소 포함
    selected.append(items[index])
    generate_subsets(
        items, index + 1, selected
    )

    # 현재 원소 제외
    selected.pop()
    generate_subsets(
        items, index + 1, selected
    )

generate_subsets([1, 2, 3])

후보 공간

각 원소마다 포함 또는 제외 두 가지 연산이 있습니다.

원소가 n개이면 후보 수는 2ⁿ개입니다.

예제 7

목표 금액을 만드는 동전 조합

누적 합가지치기
coins = [1, 2, 5]
target = 7
solutions = []

def search(start, total, selected):
    if total == target:
        solutions.append(selected.copy())
        return

    if total > target:
        return

    for index in range(start, len(coins)):
        coin = coins[index]
        selected.append(coin)

        search(
            index,
            total + coin,
            selected
        )

        selected.pop()

search(0, 0, [])

print("경우의 수:", len(solutions))
for solution in solutions:
    print(solution)
가지치기
누적 합이 목표 금액을 초과하면 그 뒤의 후보는 모두 불가능하므로 즉시 탐색을 중단합니다.
예제 8

학생 좌석 배치 모든 경우 탐색

배치조건 검사
students = ["민준", "서연", "지우"]
valid_arrangements = []

def arrange(selected):
    if len(selected) == len(students):
        # 민준과 서연이 붙어 앉지 않는 조건
        for i in range(len(selected) - 1):
            pair = {selected[i], selected[i + 1]}

            if pair == {"민준", "서연"}:
                return

        valid_arrangements.append(
            selected.copy()
        )
        return

    for student in students:
        if student not in selected:
            selected.append(student)
            arrange(selected)
            selected.pop()

arrange([])

for arrangement in valid_arrangements:
    print(arrangement)
목표 상태
모든 학생의 자리가 결정되고 금지 조건을 위반하지 않은 상태입니다.
예제 9

미로에서 모든 가능한 경로 탐색

상태 공간방문 복원
maze = [
    [0, 0, 0],
    [0, 1, 0],
    [0, 0, 0]
]

rows = len(maze)
cols = len(maze[0])
target = (2, 2)

visited = set()
paths = []

def explore(row, col, path):
    if not (0 <= row < rows and 0 <= col < cols):
        return

    if maze[row][col] == 1:
        return

    if (row, col) in visited:
        return

    visited.add((row, col))
    path.append((row, col))

    if (row, col) == target:
        paths.append(path.copy())
    else:
        for dr, dc in [
            (1, 0), (-1, 0),
            (0, 1), (0, -1)
        ]:
            explore(
                row + dr,
                col + dc,
                path
            )

    path.pop()
    visited.remove((row, col))

explore(0, 0, [])

print("경로 수:", len(paths))
for path in paths:
    print(path)
상태 복원
다른 경로에서 같은 칸을 다시 사용할 수 있도록 재귀 호출이 끝난 뒤 방문 표시와 경로를 원상 복구합니다.
예제 10

종합: 목표 합 부분집합 백트래킹

종합 설계가지치기해 기록
numbers = [2, 3, 5, 7, 8]
target = 10
solutions = []

def subset_sum(
    index,
    total,
    selected
):
    # 목표 상태
    if total == target:
        solutions.append(selected.copy())
        return

    # 실패 상태와 가지치기
    if total > target:
        return

    if index == len(numbers):
        return

    # 현재 숫자를 선택
    selected.append(numbers[index])
    subset_sum(
        index + 1,
        total + numbers[index],
        selected
    )

    # 선택 복원
    selected.pop()

    # 현재 숫자를 선택하지 않음
    subset_sum(
        index + 1,
        total,
        selected
    )

subset_sum(0, 0, [])

print("목표 합:", target)
print("해의 개수:", len(solutions))

for solution in solutions:
    print(solution)
구조화 요소이 문제의 정의
상태현재 인덱스, 누적 합, 선택된 숫자 목록
연산현재 숫자를 선택하거나 선택하지 않음
목표 상태누적 합이 target과 같음
실패 상태합이 목표를 초과하거나 모든 숫자를 검사함
복원선택한 숫자를 pop하여 이전 상태로 돌아감
효율성최악에는 2ⁿ개 후보를 검사하지만 가지치기로 일부 후보 제거
알고리즘 설명 예시
“현재 상태를 검사 중인 숫자의 위치, 누적 합, 선택 목록으로 정의하였다. 각 단계에서 숫자를 선택하는 연산과 선택하지 않는 연산을 모두 수행하였다. 누적 합이 목표와 같으면 해를 기록하고, 목표를 초과하면 더 탐색하지 않도록 가지치기하였다. 따라서 가능한 후보를 빠짐없이 확인하면서 불필요한 탐색을 줄였다.”

마무리 진단 문제

1. 문제의 상태란 무엇을 의미하는가?
정답 보기현재까지의 선택, 위치, 누적값 등 문제 해결 과정의 현재 모습을 의미합니다.
2. 완전 탐색의 가장 큰 장점은 무엇인가?
정답 보기모든 후보를 검사하므로 후보 수가 감당 가능하다면 정답을 놓치지 않습니다.
3. 백트래킹에서 선택 후 pop()을 하는 이유는 무엇인가?
정답 보기현재 선택을 취소하고 이전 상태로 복원하여 다른 후보를 탐색하기 위해서입니다.
4. 가지치기는 어떤 역할을 하는가?
정답 보기정답이 될 수 없는 후보를 조기에 중단하여 불필요한 탐색을 줄입니다.
5. 부분집합 탐색에서 각 원소에 대해 수행하는 두 연산은 무엇인가?
정답 보기현재 원소를 선택하는 연산과 선택하지 않는 연산입니다.
