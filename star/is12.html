평균 대기시간 분석 예제 10선

실생활의 대기 상황을 FIFO 큐로 모델링하고, 도착 시각과 처리 시간을 바탕으로 서비스 시작 시각·종료 시각·개인별 대기시간·평균 대기시간을 계산합니다.

분석 원리1. 급식 기본2. 개인 대기시간 3. 평균 대기시간4. 도착 시각5. 두 배식대 6. 프린터 작업7. 우선 프린터8. 상담 예약 9. 통계 분석10. 종합 시뮬레이션

1. 대기시간 분석의 핵심

도착 시각학생이나 작업이 대기열에 들어온 시각입니다.
시작 시각실제 서비스나 처리가 시작된 시각입니다.
대기시간시작 시각에서 도착 시각을 뺀 값입니다.
평균 대기시간모든 대기시간의 합을 인원수로 나눈 값입니다.
한 명의 처리 시작 시각은 그 사람의 도착 시각과 바로 앞 사람의 종료 시각 중 더 늦은 값입니다. 따라서 start = max(arrival, current_time)으로 계산할 수 있습니다.
계산값식의미
시작 시각max(도착 시각, 현재 처리 종료 시각)도착 전에는 처리할 수 없음
대기시간시작 시각 - 도착 시각줄에서 기다린 시간
종료 시각시작 시각 + 처리 시간다음 사람 처리 가능 시각
평균대기시간 합 / 인원수전체 대기 수준
예제 1

급식 대기열 기본 모델

FIFOdeque
from collections import deque

students = deque([
    "민준", "서연", "지우", "하준"
])

order = 1

while students:
    student = students.popleft()
    print(f"{order}번째 입장: {student}")
    order += 1

모델 의미

먼저 줄을 선 학생이 먼저 입장합니다.

큐의 앞에서 popleft()하여 FIFO 순서를 유지합니다.

예제 2

처리 시간이 같은 급식 대기시간

개인 대기시간누적 시간
students = ["민준", "서연", "지우", "하준"]
service_time = 2
current_time = 0

for student in students:
    waiting_time = current_time

    print(
        f"{student}: 대기 {waiting_time}분, "
        f"식판 수령 {current_time}분"
    )

    current_time += service_time
결과 해석
모두 동시에 도착했다고 가정하면 첫 학생은 0분, 다음 학생들은 2분씩 더 기다립니다.
예제 3

급식 평균 대기시간 계산

평균리스트 저장
students = ["민준", "서연", "지우", "하준"]
service_times = [2, 3, 1, 2]

current_time = 0
waiting_times = []

for student, service in zip(
    students, service_times
):
    waiting_time = current_time
    waiting_times.append(waiting_time)

    print(
        student,
        "대기:", waiting_time,
        "처리:", service
    )

    current_time += service

average = sum(waiting_times) / len(waiting_times)

print("평균 대기시간:", average, "분")

계산 흐름

각 학생의 대기시간은 앞 학생들의 처리 시간 합입니다.

대기시간을 목록에 저장한 뒤 산술평균을 계산합니다.

예제 4

도착 시각이 다른 급식 대기열

도착 시각유휴 시간
students = [
    ("민준", 0, 3),
    ("서연", 1, 2),
    ("지우", 5, 2),
    ("하준", 6, 1)
]

current_time = 0
waiting_times = []

for name, arrival, service in students:
    start = max(arrival, current_time)
    waiting = start - arrival
    finish = start + service

    waiting_times.append(waiting)

    print(
        f"{name}: 도착={arrival}, 시작={start}, "
        f"종료={finish}, 대기={waiting}"
    )

    current_time = finish

print(
    "평균 대기시간:",
    sum(waiting_times) / len(waiting_times)
)
유휴 시간
다음 학생이 아직 도착하지 않았다면 배식대는 쉬게 되며, 시작 시각은 학생의 도착 시각이 됩니다.
예제 5

배식대 1개와 2개 비교

병렬 처리개선 분석
students = [
    ("민준", 0, 3),
    ("서연", 0, 2),
    ("지우", 1, 4),
    ("하준", 2, 2),
    ("예린", 3, 3)
]

def simulate(server_count):
    servers = [0] * server_count
    waiting_times = []

    for name, arrival, service in students:
        server_index = min(
            range(server_count),
            key=lambda index: servers[index]
        )

        start = max(
            arrival,
            servers[server_index]
        )

        waiting = start - arrival
        waiting_times.append(waiting)

        servers[server_index] = start + service

    return sum(waiting_times) / len(waiting_times)

print("배식대 1개:", simulate(1))
print("배식대 2개:", simulate(2))
분석
처리 창구를 늘리면 동시에 여러 학생을 처리할 수 있어 평균 대기시간이 줄어드는지 비교할 수 있습니다.

예제 7

긴 작업이 앞에 있을 때의 영향

작업 순서평균 비교
jobs = [
    ("긴 보고서", 10),
    ("짧은 문서1", 1),
    ("짧은 문서2", 2),
    ("짧은 문서3", 1)
]

def average_waiting(job_list):
    current_time = 0
    waits = []

    for name, service in job_list:
        waits.append(current_time)
        current_time += service

    return sum(waits) / len(waits)

fifo_average = average_waiting(jobs)

short_first = sorted(
    jobs,
    key=lambda job: job[1]
)
short_average = average_waiting(short_first)

print("FIFO 평균:", fifo_average)
print("짧은 작업 우선 평균:", short_average)
윤리·운영 관점
짧은 작업 우선은 평균 대기시간을 줄일 수 있지만 먼저 도착한 작업이 늦어질 수 있습니다. 공정성과 효율성을 함께 고려해야 합니다.

예제 9

평균·최대·중앙 대기시간 분석

통계median
from statistics import mean, median

waiting_times = [0, 2, 5, 5, 8, 14, 21]

print("평균:", mean(waiting_times))
print("중앙값:", median(waiting_times))
print("최대:", max(waiting_times))
print("최소:", min(waiting_times))

over_10 = [
    value
    for value in waiting_times
    if value >= 10
]

print("10분 이상 인원:", len(over_10))

왜 여러 지표를 볼까?

평균은 전체 수준을 보여 줍니다.

중앙값은 매우 긴 대기시간의 영향을 덜 받습니다.

최댓값은 가장 불편한 사례를 보여 줍니다.

예제 10

종합 대기열 시뮬레이션 함수

종합 프로그램재사용 함수
from statistics import mean, median

def analyze_queue(customers, server_count=1):
    servers = [0] * server_count
    results = []

    for name, arrival, service in customers:
        server_index = min(
            range(server_count),
            key=lambda index: servers[index]
        )

        start = max(
            arrival,
            servers[server_index]
        )

        waiting = start - arrival
        finish = start + service

        servers[server_index] = finish

        results.append({
            "이름": name,
            "도착": arrival,
            "시작": start,
            "종료": finish,
            "대기": waiting,
            "창구": server_index + 1
        })

    waits = [item["대기"] for item in results]

    summary = {
        "평균": mean(waits),
        "중앙값": median(waits),
        "최대": max(waits),
        "대기 0명": waits.count(0)
    }

    return results, summary

students = [
    ("민준", 0, 3),
    ("서연", 0, 2),
    ("지우", 1, 4),
    ("하준", 2, 2),
    ("예린", 5, 3)
]

results, summary = analyze_queue(
    students,
    server_count=2
)

for item in results:
    print(item)

print("\n[요약]")
for key, value in summary.items():
    print(key, ":", value)

마무리 진단 문제

1. 한 사람의 대기시간은 어떻게 계산하는가?
정답 보기서비스 시작 시각에서 도착 시각을 뺍니다.
2. 서비스 시작 시각을 계산할 때 도착 시각과 현재 시각 중 어느 값을 선택하는가?
정답 보기두 값 중 더 큰 값을 선택합니다.
3. 처리 창구 수를 늘리면 일반적으로 어떤 변화가 예상되는가?
정답 보기동시 처리량이 늘어 평균 대기시간이 감소할 수 있습니다.
4. 평균만으로 대기 상황을 판단하기 어려운 이유는 무엇인가?
정답 보기일부 매우 긴 대기시간이나 개인별 불편 정도를 충분히 보여 주지 못할 수 있기 때문입니다.
5. 짧은 작업 우선 방식의 장점과 단점은 무엇인가?
정답 보기평균 대기시간을 줄일 수 있지만 먼저 도착한 긴 작업의 처리가 늦어져 공정성 문제가 생길 수 있습니다.
