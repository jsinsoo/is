예제 1

학생 점수 순서 관리 — 리스트 선택

리스트순서 유지
scores = [85, 92, 78, 96, 88]

scores.append(91)
scores[2] = 80

average = sum(scores) / len(scores)

print("점수:", scores)
print("세 번째 점수:", scores[2])
print("평균:", average)

선택 근거

점수의 입력 순서를 유지하고, 특정 위치의 값을 수정하거나 전체를 순회해야 하므로 리스트를 선택합니다.

효율성

인덱스 접근은 빠르지만, 앞이나 중간 삽입·삭제는 뒤쪽 원소 이동이 필요합니다.

예제 2

행사 참가자 중복 제거 — 집합 선택

집합중복 제거
participants = [
    "민준", "서연", "민준",
    "지우", "서연", "하준"
]

unique_participants = set(participants)

print("고유 참가자:", unique_participants)

선택 근거
순서보다 고유한 이름과 빠른 포함 여부 확인이 중요하므로 집합이 적합합니다. 원래 입력 순서가 필요하다면 별도 처리가 필요합니다.
예제 3

단어 빈도 분석 — 딕셔너리 선택

딕셔너리빈도 누적
text = "데이터 분석 데이터 시각화 분석"
words = text.split()

frequency = {}

for word in words:
    frequency[word] = frequency.get(word, 0) + 1

for word, count in frequency.items():
    print(word, ":", count)

most_common = max(
    frequency,
    key=frequency.get
)

print("가장 많이 나온 단어:", most_common)

선택 근거

단어를 키로 사용하고 해당 단어의 출현 횟수를 값으로 누적해야 합니다.

결과

각 단어의 빈도와 최빈 단어를 빠르게 조회할 수 있습니다.

예제 4

최근 작업 실행 취소 — 스택 선택

스택LIFO
document = ""
history = []

def write(text):
    global document

    history.append(document)
    document += text

def undo():
    global document

    if history:
        document = history.pop()

write("자료")
write("구조")
write(" 선택")

print("현재:", document)
undo()
print("취소 후:", document)
선택 근거
가장 최근 작업을 가장 먼저 취소해야 하므로 후입선출 스택이 적합합니다. 리스트의 맨 뒤에서 삽입·삭제하면 효율적입니다.
예제 5

상담 대기 순서 — 큐 선택

큐FIFOdeque
from collections import deque

waiting = deque()

waiting.append("민준")
waiting.append("서연")
waiting.append("지우")

while waiting:
    student = waiting.popleft()
    print("상담:", student)

선택 근거

먼저 신청한 학생을 먼저 상담해야 하므로 선입선출 큐를 선택합니다.

효율성

deque.popleft()는 리스트의 pop(0)보다 앞쪽 삭제에 효율적입니다.

예제 6

양쪽 문자 비교 — 덱 선택

덱회문 검사
from collections import deque

def is_palindrome(text):
    characters = deque(
        character.lower()
        for character in text
        if character.isalnum()
    )

    while len(characters) > 1:
        left = characters.popleft()
        right = characters.pop()

        if left != right:
            return False

    return True

print(is_palindrome("level"))
print(is_palindrome("python"))
선택 근거
문자열의 왼쪽과 오른쪽을 반복해서 제거해야 하므로 양쪽 끝 연산이 빠른 덱이 적합합니다.
예제 7

가장 긴급한 작업 처리 — 최소 힙 선택

힙우선순위 큐
import heapq

tasks = []

heapq.heappush(tasks, (3, "교실 정리"))
heapq.heappush(tasks, (1, "서버 장애 확인"))
heapq.heappush(tasks, (2, "수업 자료 출력"))

while tasks:
    priority, task = heapq.heappop(tasks)
    print(priority, task)

선택 근거

입력 순서가 아니라 우선순위가 가장 높은 작업을 반복해서 꺼내야 합니다.

효율성

힙은 최솟값 확인과 삭제가 효율적이며, 전체를 매번 정렬하는 비용을 줄입니다.

예제 8

친구 관계와 공통 친구 — 그래프 선택

그래프인접리스트
friends = {
    "민준": {"서연", "지우"},
    "서연": {"민준", "하준", "지우"},
    "지우": {"민준", "서연"},
    "하준": {"서연"}
}

def common_friends(a, b):
    return friends[a] & friends[b]

print(
    "민준·하준의 공통 친구:",
    common_friends("민준", "하준")
)

for person, neighbors in friends.items():
    print(person, "차수:", len(neighbors))
선택 근거
학생과 친구 관계처럼 대상과 연결을 표현해야 하므로 그래프가 적합합니다. 이웃을 집합으로 저장하면 공통 친구 교집합도 쉽게 계산할 수 있습니다.
예제 9

교실 이동의 최소 단계 — 그래프와 BFS 선택

그래프BFS큐
from collections import deque

building = {
    "교실": ["복도"],
    "복도": ["교실", "계단", "도서관"],
    "계단": ["복도", "과학실"],
    "도서관": ["복도", "과학실"],
    "과학실": ["계단", "도서관"]
}

def shortest_path(start, target):
    queue = deque([(start, [start])])
    visited = {start}

    while queue:
        place, path = queue.popleft()

        if place == target:
            return path

        for next_place in building[place]:
            if next_place not in visited:
                visited.add(next_place)
                queue.append(
                    (next_place, path + [next_place])
                )

    return None

path = shortest_path("교실", "과학실")

print("경로:", path)
print("이동 단계:", len(path) - 1)
선택 근거
장소 연결은 그래프로 나타내고, 모든 이동 비용이 같을 때 최소 간선 수 경로는 큐 기반 BFS로 찾습니다.
예제 10

문제 설명에 따른 자료구조 추천 프로그램

종합 판단선택 근거
def recommend_structure(
    keep_order=False,
    remove_duplicates=False,
    key_lookup=False,
    recent_first=False,
    arrival_first=False,
    both_ends=False,
    priority=False,
    relationships=False
):
    if relationships:
        return "그래프", "대상 사이의 연결과 경로를 표현할 수 있습니다."

    if priority:
        return "힙", "최우선 자료를 반복해서 효율적으로 꺼낼 수 있습니다."

    if recent_first:
        return "스택", "가장 최근 자료를 먼저 처리하는 LIFO 구조입니다."

    if arrival_first:
        return "큐", "먼저 들어온 자료를 먼저 처리하는 FIFO 구조입니다."

    if both_ends:
        return "덱", "양쪽 끝의 삽입과 삭제가 모두 효율적입니다."

    if remove_duplicates:
        return "집합", "중복을 자동으로 제거하고 포함 여부를 빠르게 확인합니다."

    if key_lookup:
        return "딕셔너리", "키를 이용해 값을 빠르게 조회하고 갱신합니다."

    if keep_order:
        return "리스트", "자료의 순서와 인덱스를 유지합니다."

    return "리스트", "일반적인 순차 자료 처리에 사용할 수 있습니다."

cases = [
    {"recent_first": True},
    {"arrival_first": True},
    {"remove_duplicates": True},
    {"relationships": True},
    {"priority": True}
]

for case in cases:
    structure, reason = recommend_structure(**case)
    print(structure, ":", reason)

최종 설명 예시
“상담 신청은 먼저 들어온 학생부터 처리해야 하므로 큐를 선택하였다. deque의 뒤 삽입과 앞 삭제를 사용하여 신청 순서를 유지했으며, 모든 학생이 접수 순서대로 처리되었다. 리스트의 앞 삭제와 달리 앞쪽 원소를 반복해서 이동하지 않으므로 대기자가 많아져도 효율적이다.”
