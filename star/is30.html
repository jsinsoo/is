표준·외부 라이브러리 조사와 핵심 기능 구현 예제 10선

소프트웨어와 데이터의 라이선스를 확인하고 출처를 올바르게 표시하며, 파이썬 표준 라이브러리와 외부 라이브러리의 역할을 조사한 뒤 프로젝트 핵심 기능을 구현합니다.

핵심 원리1. 라이선스 기록2. 출처 표시 3. 표준 라이브러리4. 외부 라이브러리5. CSV 분석 6. JSON 처리7. 시각화8. 데이터프레임 9. 환경 기록10. 종합 프로젝트

1. 라이브러리 활용 전 확인 절차

① 출처 확인공식 문서·저장소·배포 페이지에서 개발자와 원출처를 확인합니다.
② 라이선스 확인사용·수정·배포·상업적 이용 조건과 고지 의무를 확인합니다.
③ 기능 조사표준 라이브러리로 가능한지 먼저 보고 외부 라이브러리 필요성을 판단합니다.
④ 재현성 기록버전, 설치 명령, 데이터 출처, 변경 사항을 문서에 남깁니다.
오픈소스는 “무료”와 동일한 뜻이 아닙니다. 각 라이선스가 허용하는 범위와 요구하는 의무를 확인해야 하며, 코드·데이터·이미지·글꼴은 서로 다른 라이선스를 가질 수 있습니다.
라이선스 유형일반적 특징확인할 의무
MIT사용·수정·배포 허용 범위가 넓음저작권 표시와 라이선스 문구 유지
Apache-2.0폭넓은 사용 허용과 특허 조항 포함라이선스·NOTICE·변경 사항 표시 확인
BSD 계열비교적 간결한 허용형 라이선스저작권과 면책 문구 유지
GPL 계열파생물 배포 시 같은 라이선스 요구 가능소스 공개와 동일 라이선스 적용 범위 확인
CC 라이선스주로 문서·이미지·교육 자료에 사용저작자 표시, 비영리·변경 금지·동일 조건 여부 확인

예제 2

출처 표시 문장 자동 생성

저작자 표시문서화
def make_attribution(
    title,
    author,
    source,
    license_name,
    modified=False
):
    text = (
        f"자료명: {title}\n"
        f"저작자: {author}\n"
        f"출처: {source}\n"
        f"라이선스: {license_name}\n"
    )

    if modified:
        text += "변경 사항: 원본을 수정하여 사용함\n"
    else:
        text += "변경 사항: 없음\n"

    return text

print(make_attribution(
    title="기후 데이터",
    author="데이터 제공 기관",
    source="공공데이터 포털",
    license_name="해당 페이지의 이용 조건 확인",
    modified=True
))
출처 표시 요소
자료명, 저작자, 출처, 라이선스, 수정 여부를 함께 기록하면 재사용 조건을 확인하기 쉽습니다.
예제 3

표준 라이브러리만으로 파일 통계 계산

표준 라이브러리statistics
from statistics import mean, median

values = [12, 15, 14, 18, 21, 16]

print("평균:", mean(values))
print("중앙값:", median(values))
print("최댓값:", max(values))
print("최솟값:", min(values))

조사 결과

statistics는 파이썬 설치에 포함되는 표준 라이브러리입니다.

별도 설치 없이 기본 통계 기능을 구현할 수 있습니다.

선택 근거
단순한 평균·중앙값 계산만 필요하다면 외부 라이브러리를 추가하지 않아도 됩니다.
예제 4

외부 라이브러리 설치 가능 여부 점검

importlib오류 처리
from importlib.util import find_spec

libraries = [
    "pandas",
    "matplotlib",
    "numpy"
]

for library in libraries:
    if find_spec(library) is None:
        print(
            library,
            "미설치 - 설치와 라이선스 확인 필요"
        )
    else:
        print(
            library,
            "사용 가능"

중요

예제 5

csv 표준 라이브러리로 센서 자료 읽기

csv데이터 처리
import csv
from io import StringIO

sample = """time,co2
09:00,720
09:10,860
09:20,1040
09:30,1280
"""

file_like = StringIO(sample)
reader = csv.DictReader(file_like)

values = []

for row in reader:
    value = int(row["co2"])
    values.append(value)

    print(
        row["time"],
        value,
        "환기 필요" if value >= 1000 else "정상"
    )

print("평균:", sum(values) / len(values))
선택 근거
CSV 파일을 단순히 읽고 행 단위로 처리할 때는 표준 라이브러리만으로도 충분합니다.

예제 7

matplotlib으로 핵심 시각화 구현

plt.rcParams["axes.unicode_minus"] = False

times = ["09:00", "09:10", "09:20", "09:30"]

plt.plot(

    label="환기 기준"
)

plt.title("교실 이산화탄소 농도")
plt.xlabel("시간")
plt.ylabel("농도(ppm)")
plt.legend()

plt.show()

예제 8

pandas로 데이터프레임 분석

pandas표 데이터
import pandas as pd

data = {
    "시간": ["09:00", "09:10", "09:20", "09:30"],
    "이산화탄소": [720, 860, 1040, 1280]
}

df = pd.DataFrame(data)

df["상태"] = df["이산화탄소"].apply(
    lambda value: (
        "환기 필요"
        if value >= 1000
        else "정상"
    )
)

print(df)
print(
    "평균:",
    df["이산화탄소"].mean()
)
선택 근거
행과 열이 많은 데이터에서 조건 열 생성, 결측값 처리, 그룹별 집계가 필요할 때 데이터프레임이 편리합니다.
예제 9

실행 환경과 버전 기록

재현성버전 기록
import platform
import sys
from importlib.metadata import version, PackageNotFoundError

print("운영체제:", platform.platform())
print("파이썬:", sys.version)

packages = [
    "pandas",
    "matplotlib",
    "numpy"
]

for package in packages:
    try:
        print(
            package,
            version(package)
        )
    except PackageNotFoundError:
        print(
            package,
            "설치되지 않음"
        )
왜 필요한가?
같은 코드라도 파이썬과 라이브러리 버전에 따라 결과나 사용법이 달라질 수 있으므로 보고서에 환경을 기록합니다.
예제 10

종합: 라이선스 점검과 핵심 기능 실행

종합 프로젝트체크리스트
project = {
    "name": "교실 환경 분석",
    "data_source": "교실 센서 실험 자료",
    "data_license": "직접 수집 자료",
    "libraries": [
        {
            "name": "csv",
            "kind": "표준 라이브러리",
            "purpose": "파일 읽기",
            "license_checked": True
        },
        {
            "name": "statistics",
            "kind": "표준 라이브러리",
            "purpose": "평균 계산",
            "license_checked": True
        },
        {
            "name": "matplotlib",
            "kind": "외부 라이브러리",
            "purpose": "그래프 출력",
            "license_checked": False
        }
    ]
}

def check_project(project):
    print("프로젝트:", project["name"])
    print("데이터 출처:", project["data_source"])
    print("데이터 조건:", project["data_license"])

    ready = True

    for library in project["libraries"]:
        status = (
            "확인 완료"
            if library["license_checked"]
            else "확인 필요"
        )

        print(
            library["name"],
            library["kind"],
            library["purpose"],
            status
        )

        if not library["license_checked"]:
            ready = False

    return ready

ready = check_project(project)

if ready:
    print("핵심 기능 구현 진행 가능")
else:
    print("공개·배포 전 라이선스 보완 필요")
단계확인 내용산출물
자료 조사공식 문서·공식 저장소·배포 페이지 확인출처 목록
라이선스 확인사용·수정·배포·고지 의무 확인라이선스 표
기술 선택표준 라이브러리로 가능한지 먼저 검토선택 근거
설치·버전 기록설치 명령과 버전 저장requirements.txt 또는 환경표

배포 점검라이선스 문구·NOTICE·출처 표시 포함 여부 확인최종 점검표
프로젝트 설명 예시
“교실 환경 분석 프로그램은 파이썬 표준 라이브러리인 csv와 statistics를 사용하여 센서 자료를 읽고 평균을 계산한다. 시간대별 변화를 시각화하기 위해 외부 라이브러리 matplotlib을 추가로 조사하였다. 사용 전 공식 문서와 라이선스를 확인하고, 보고서에 라이브러리 이름·버전·출처·사용 목적을 기록하였다. 데이터는 직접 측정한 비식별 자료를 사용하며, 그래프와 분석 결과에 데이터 수집 방법을 명시하였다.”

마무리 진단 문제

1. 오픈소스 라이브러리를 사용할 때 가장 먼저 확인해야 할 것은 무엇인가?
정답 보기공식 출처와 라이선스의 사용·수정·배포 조건을 확인해야 합니다.
2. 표준 라이브러리와 외부 라이브러리의 차이는 무엇인가?
정답 보기표준 라이브러리는 파이썬 설치에 기본 포함되고, 외부 라이브러리는 보통 별도 설치가 필요합니다.
3. 출처 표시에 포함하면 좋은 항목은 무엇인가?
정답 보기자료명, 저작자, 출처, 라이선스, 수정 여부입니다.
4. 라이브러리 버전을 기록해야 하는 이유는 무엇인가?
정답 보기같은 실행 환경을 재현하고 버전 차이로 발생하는 오류를 분석하기 위해서입니다.
5. 외부 라이브러리를 선택하기 전에 표준 라이브러리를 검토하는 이유는 무엇인가?
정답 보기불필요한 설치와 의존성을 줄이고 실행 환경을 단순하게 유지할 수 있기 때문입니다.
