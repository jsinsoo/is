1. 중복 호출과 메모이제이션

중복 호출같은 입력값에 대한 함수가 여러 번 반복 실행되는 현상입니다.
메모이제이션한 번 계산한 결과를 저장해 두었다가 같은 입력이 들어오면 다시 계산하지 않고 꺼내 쓰는 방법입니다.
성능 개선호출 횟수와 실행 시간을 크게 줄여 재귀함수를 효율적으로 만듭니다.
피보나치 재귀함수는 fibonacci(n-1)과 fibonacci(n-2)를 각각 호출하면서 같은 값을 여러 번 계산합니다. 메모이제이션을 적용하면 각 입력값은 보통 한 번만 계산되므로 실행 시간이 크게 줄어듭니다.
분석 항목확인 방법
호출 횟수전역 변수 또는 딕셔너리로 함수가 몇 번 호출되는지 셉니다.
실행 시간time.perf_counter()로 시작과 종료 시각의 차이를 측정합니다.
저장 여부계산 전에 입력값이 메모에 있는지 확인합니다.
예제 1

피보나치 중복 호출 관찰하기

중복 계산호출 출력
def fibonacci(n):
    print(f"fibonacci({n}) 호출")

    if n == 0:
        return 0
    if n == 1:
        return 1

    return fibonacci(n - 1) + fibonacci(n - 2)

print("결과:", fibonacci(5))

관찰 포인트

fibonacci(3), fibonacci(2), fibonacci(1) 등이 여러 번 출력됩니다.

fibonacci(5)
├─ fibonacci(4)
│  ├─ fibonacci(3)
│  └─ fibonacci(2)
└─ fibonacci(3)
   ├─ fibonacci(2)
   └─ fibonacci(1)
문제점
같은 값을 반복 계산하므로 n이 커질수록 호출 수가 매우 빠르게 증가합니다.
예제 2

재귀 피보나치 호출 횟수 세기

전역 변수호출 횟수
call_count = 0

def fibonacci(n):
    global call_count
    call_count += 1

    if n == 0:
        return 0
    if n == 1:
        return 1

    return fibonacci(n - 1) + fibonacci(n - 2)

result = fibonacci(10)

print("결과:", result)
print("호출 횟수:", call_count)
분석
결과값은 작지만 함수 호출은 훨씬 많이 발생합니다. 입력값을 10, 20, 30으로 바꾸어 호출 횟수 증가를 비교할 수 있습니다.
예제 3

실행 시간 측정하기

perf_counter시간 비교
import time

def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

start = time.perf_counter()
result = fibonacci(30)
end = time.perf_counter()

print("결과:", result)
print(f"실행 시간: {end - start:.6f}초")

측정 원리

함수 실행 전 시각과 실행 후 시각의 차이를 계산합니다.

컴퓨터 상태에 따라 시간은 달라질 수 있으므로 여러 번 실행해 비교하는 것이 좋습니다.

예제 4

딕셔너리 메모이제이션

딕셔너리저장·재사용
memo = {0: 0, 1: 1}
call_count = 0

def fibonacci(n):
    global call_count
    call_count += 1

    if n in memo:
        return memo[n]

    memo[n] = fibonacci(n - 1) + fibonacci(n - 2)
    return memo[n]

result = fibonacci(30)

print("결과:", result)
print("호출 횟수:", call_count)
print("저장된 값 개수:", len(memo))

실행 흐름

  1. 입력값 n이 딕셔너리에 있는지 확인합니다.
  2. 있으면 저장값을 즉시 반환합니다.
  3. 없으면 계산한 뒤 딕셔너리에 저장합니다.
개선 효과
각 피보나치 값이 한 번씩만 계산되어 큰 입력에서도 빠르게 실행됩니다.
예제 5

리스트를 이용한 메모이제이션

리스트인덱스 저장
def fibonacci(n, memo=None):
    if memo is None:
        memo = [-1] * (n + 1)
        memo[0] = 0
        if n >= 1:
            memo[1] = 1

    if memo[n] != -1:
        return memo[n]

    memo[n] = fibonacci(n - 1, memo) + fibonacci(n - 2, memo)
    return memo[n]

print(fibonacci(40))
특징
입력값이 0부터 연속된 정수라면 리스트 인덱스를 활용해 빠르게 접근할 수 있습니다.
주의
기본 매개변수에 빈 리스트를 직접 작성하면 호출 사이에 값이 공유될 수 있으므로 memo=None 방식이 안전합니다.
예제 6

functools.lru_cache 활용

데코레이터자동 캐시
from functools import lru_cache

@lru_cache(maxsize=None)
def fibonacci(n):
    if n < 2:
        return n

    return fibonacci(n - 1) + fibonacci(n - 2)

print("결과:", fibonacci(100))
print("캐시 정보:", fibonacci.cache_info())

장점

직접 딕셔너리를 만들지 않아도 함수의 입력과 결과를 자동으로 저장합니다.

cache_info()로 적중 횟수, 누락 횟수, 현재 저장 크기를 확인할 수 있습니다.

예제 7

계단 오르기 문제에 메모이제이션 적용

동적 계획법경우의 수
memo = {0: 1, 1: 1}

def count_ways(n):
    if n in memo:
        return memo[n]

    memo[n] = count_ways(n - 1) + count_ways(n - 2)
    return memo[n]

print("10칸 계단을 오르는 방법:", count_ways(10))
문제 구조
마지막에 1칸을 오른 경우와 2칸을 오른 경우로 나누면 같은 작은 문제가 반복됩니다. 메모이제이션이 적합한 구조입니다.
예제 8

격자 경로 수 구하기

2차원 상태튜플 키
memo = {}

def count_paths(row, col):
    if row == 0 or col == 0:
        return 1

    key = (row, col)

    if key in memo:
        return memo[key]

    memo[key] = (
        count_paths(row - 1, col)
        + count_paths(row, col - 1)
    )
    return memo[key]

print(count_paths(10, 10))

메모의 키

상태가 두 값으로 결정되므로 (row, col) 튜플을 딕셔너리 키로 사용합니다.

같은 좌표의 경로 수를 다시 계산하지 않습니다.

예제 9

최소 비용 경로 재귀함수

최솟값부분 문제
cost = [1, 3, 2, 5, 4, 2]
memo = {}

def min_cost(index):
    if index == 0:
        return cost[0]
    if index == 1:
        return cost[1]

    if index in memo:
        return memo[index]

    memo[index] = cost[index] + min(
        min_cost(index - 1),
        min_cost(index - 2)
    )
    return memo[index]

print("최소 비용:", min_cost(len(cost) - 1))
분석
현재 위치까지 오는 최소 비용은 직전 칸과 두 칸 전 위치 중 더 작은 비용을 선택해 계산합니다.
예제 10

일반 재귀와 메모이제이션 성능 종합 비교

실행 시간호출 횟수종합 실습
import time

normal_calls = 0
memo_calls = 0

def fibonacci_normal(n):
    global normal_calls
    normal_calls += 1

    if n < 2:
        return n

    return (
        fibonacci_normal(n - 1)
        + fibonacci_normal(n - 2)
    )

memo = {0: 0, 1: 1}

def fibonacci_memo(n):
    global memo_calls
    memo_calls += 1

    if n in memo:
        return memo[n]

    memo[n] = (
        fibonacci_memo(n - 1)
        + fibonacci_memo(n - 2)
    )
    return memo[n]

n = 30

start = time.perf_counter()
normal_result = fibonacci_normal(n)
normal_time = time.perf_counter() - start

start = time.perf_counter()
memo_result = fibonacci_memo(n)
memo_time = time.perf_counter() - start

print("[일반 재귀]")
print("결과:", normal_result)
print("호출 횟수:", normal_calls)
print(f"실행 시간: {normal_time:.6f}초")

print("\n[메모이제이션]")
print("결과:", memo_result)
print("호출 횟수:", memo_calls)
print(f"실행 시간: {memo_time:.6f}초")
비교 항목일반 재귀메모이제이션
중복 계산같은 값을 여러 번 계산함계산 결과를 저장하여 재사용함
호출 횟수매우 빠르게 증가함입력 크기에 비례해 증가함
실행 시간n이 커지면 급격히 느려짐큰 n에서도 빠르게 실행됨
추가 메모리호출 스택 사용호출 스택과 저장 공간 사용

마무리 진단 문제

1. 메모이제이션의 핵심 원리는 무엇인가?
정답 보기한 번 계산한 결과를 저장해 같은 입력이 다시 들어오면 재계산하지 않는 것입니다.
2. 함수 호출 횟수를 세기 위해 사용할 수 있는 방법은 무엇인가?
정답 보기전역 변수나 딕셔너리 카운터를 사용해 함수가 호출될 때마다 값을 1 증가시킬 수 있습니다.
3. 실행 시간을 정밀하게 측정할 때 사용할 수 있는 함수는 무엇인가?
정답 보기time.perf_counter()입니다.
4. 입력 상태가 두 개의 값으로 결정될 때 딕셔너리 키로 무엇을 사용할 수 있는가?
정답 보기(row, col)과 같은 튜플을 사용할 수 있습니다.
5. 메모이제이션의 단점은 무엇인가?
정답 보기계산 결과를 저장하기 위한 추가 메모리 공간이 필요합니다.
