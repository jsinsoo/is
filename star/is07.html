재귀관계 파악과 반복·재귀 해결
차이 설명 예제 10선

문제에서 이전 상태와 현재 상태의 관계를 찾아 재귀관계와 초기값을 정리하고, 동일 문제를 반복문과 재귀함수로 해결한 뒤 가독성·메모리·실행 과정의 차이를 논리적으로 설명합니다.

설계 절차1. 자연수 합2. 팩토리얼 3. 등차수열4. 피보나치5. 계단 오르기 6. 타일 채우기7. 최대공약수8. 거듭제곱 9. 자릿수 합10. 종합 성능

1. 재귀관계 파악 절차

① 상태 정의무엇을 n번째 값으로 볼지 정합니다.
② 이전 상태 연결현재 값을 이전 값들로 표현합니다.
③ 초기값 설정더 이상 나눌 수 없는 가장 작은 값을 정합니다.
④ 구현 비교반복문과 재귀함수로 구현하고 차이를 설명합니다.
논리적 설명 틀
“이 문제의 현재 상태는 이전 상태와 ○○의 관계를 가진다. 재귀함수는 이 관계를 코드에 직접 표현하여 가독성이 높지만 호출 스택을 사용한다. 반복문은 이전 상태를 변수에 저장하며 갱신하므로 메모리 사용과 실행 시간이 유리하다.”
예제 1

1부터 n까지의 자연수 합

누적 합기초 재귀관계
재귀관계: S(n) = n + S(n-1)
초기값: S(1) = 1

반복문

def sum_loop(n):
    total = 0
    for value in range(1, n + 1):
        total += value
    return total

재귀함수

def sum_recursive(n):
    if n == 1:
        return 1
    return n + sum_recursive(n - 1)
차이 설명
재귀 방식은 수학적 관계를 그대로 표현하지만 n번의 함수 호출이 필요합니다. 반복 방식은 누적 변수 하나를 갱신하여 메모리 사용이 적습니다.
예제 2

팩토리얼

곱셈 누적초기값
재귀관계: F(n) = n × F(n-1)
초기값: F(0) = 1

반복문

def factorial_loop(n):
    result = 1
    for value in range(1, n + 1):
        result *= value
    return result

재귀함수

def factorial_recursive(n):
    if n == 0:
        return 1
    return n * factorial_recursive(n - 1)
논리적 판단
정의 학습에는 재귀 방식이 직관적이고, 큰 n의 실제 계산에는 반복 방식이 안전합니다.
예제 3

등차수열의 n번째 항

수열이전 항
재귀관계: a(n) = a(n-1) + d
초기값: a(1) = first

반복문

def arithmetic_loop(first, d, n):
    value = first
    for _ in range(2, n + 1):
        value += d
    return value

재귀함수

def arithmetic_recursive(first, d, n):
    if n == 1:
        return first
    return arithmetic_recursive(first, d, n - 1) + d
추가 관점
등차수열은 일반항을 이용하면 반복도 재귀도 없이 한 번에 계산할 수 있습니다. 알고리즘 선택에서는 문제 구조뿐 아니라 더 직접적인 공식의 존재도 확인해야 합니다.
예제 4

피보나치 수열

두 이전 항중복 호출
재귀관계: F(n) = F(n-1) + F(n-2)
초기값: F(0) = 0, F(1) = 1

반복문

def fibonacci_loop(n):
    if n < 2:
        return n

    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b

재귀함수

def fibonacci_recursive(n):
    if n < 2:
        return n
    return (
        fibonacci_recursive(n - 1)
        + fibonacci_recursive(n - 2)
    )
차이 설명
재귀 방식은 관계를 명확히 보여 주지만 같은 항을 중복 계산합니다. 반복 방식은 직전 두 항만 저장하므로 시간과 메모리 측면에서 효율적입니다.
예제 5

1칸 또는 2칸 계단 오르기

경우의 수동적 계획법
재귀관계: W(n) = W(n-1) + W(n-2)
초기값: W(0) = 1, W(1) = 1

반복문

def stairs_loop(n):
    if n <= 1:
        return 1

    prev2, prev1 = 1, 1
    for _ in range(2, n + 1):
        current = prev1 + prev2
        prev2, prev1 = prev1, current
    return prev1

재귀함수

def stairs_recursive(n):
    if n <= 1:
        return 1
    return (
        stairs_recursive(n - 1)
        + stairs_recursive(n - 2)
    )
재귀관계 해석
마지막 이동이 1칸인 경우와 2칸인 경우를 나누면 두 개의 더 작은 계단 문제로 분해됩니다.
예제 6

2×n 직사각형을 1×2 타일로 채우기

타일링분해
재귀관계: T(n) = T(n-1) + T(n-2)
초기값: T(1) = 1, T(2) = 2

반복문

def tiling_loop(n):
    if n == 1:
        return 1

    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b

재귀함수

def tiling_recursive(n):
    if n == 1:
        return 1
    if n == 2:
        return 2
    return (
        tiling_recursive(n - 1)
        + tiling_recursive(n - 2)
    )
논리적 설명
맨 오른쪽을 세로 타일 하나로 채우면 n-1 문제, 가로 타일 두 개로 채우면 n-2 문제로 줄어듭니다.
예제 7

유클리드 호제법 최대공약수

나머지상태 축소
재귀관계: G(a, b) = G(b, a mod b)
초기값: G(a, 0) = a

반복문

def gcd_loop(a, b):
    while b != 0:
        a, b = b, a % b
    return a

재귀함수

def gcd_recursive(a, b):
    if b == 0:
        return a
    return gcd_recursive(b, a % b)
차이 설명
두 방식 모두 같은 재귀관계를 사용합니다. 재귀 코드는 관계를 직접 드러내고, 반복 코드는 함수 호출 없이 상태를 갱신합니다.
예제 8

거듭제곱

지수 감소분할 가능
기본 재귀관계: P(a, n) = a × P(a, n-1)
초기값: P(a, 0) = 1

반복문

def power_loop(base, exponent):
    result = 1
    for _ in range(exponent):
        result *= base
    return result

재귀함수

def power_recursive(base, exponent):
    if exponent == 0:
        return 1
    return base * power_recursive(
        base, exponent - 1
    )
확장
지수가 짝수일 때 절반으로 나누는 재귀관계를 사용하면 호출 깊이를 크게 줄이는 빠른 거듭제곱으로 개선할 수 있습니다.
예제 9

정수의 자릿수 합

정수 분해나머지
재귀관계: D(n) = n mod 10 + D(n div 10)
초기값: n < 10이면 D(n) = n

반복문

def digit_sum_loop(n):
    total = 0
    while n > 0:
        total += n % 10
        n //= 10
    return total

재귀함수

def digit_sum_recursive(n):
    if n < 10:
        return n
    return (
        n % 10
        + digit_sum_recursive(n // 10)
    )
가독성 비교
재귀 코드는 “마지막 자리와 나머지 자리”라는 분해를 명확히 나타냅니다. 반복 코드는 변수 변화가 직접 보여 실행 과정을 추적하기 쉽습니다.
예제 10

실행 시간·호출 수 종합 비교

성능 측정논리적 설명
import time

recursive_calls = 0

def fibonacci_recursive(n):
    global recursive_calls
    recursive_calls += 1

    if n < 2:
        return n

    return (
        fibonacci_recursive(n - 1)
        + fibonacci_recursive(n - 2)
    )

def fibonacci_loop(n):
    if n < 2:
        return n

    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b

n = 30

start = time.perf_counter()
r1 = fibonacci_recursive(n)
recursive_time = time.perf_counter() - start

start = time.perf_counter()
r2 = fibonacci_loop(n)
loop_time = time.perf_counter() - start

print("결과 일치:", r1 == r2)
print("재귀 호출 수:", recursive_calls)
print(f"재귀 시간: {recursive_time:.6f}초")
print(f"반복 시간: {loop_time:.6f}초")
설명 항목재귀 방식반복 방식
재귀관계 표현문제 정의를 코드에 직접 표현이전 상태를 변수로 저장하여 갱신
가독성수학적·자기 유사 구조에 유리순차적인 처리 흐름에 유리
메모리호출 스택 사용일정한 수의 변수 사용 가능
실행 시간중복 호출과 함수 호출 비용 발생 가능각 상태를 한 번씩 계산하여 빠른 경우가 많음
선택 기준구조 설명과 분할이 중요한 문제큰 입력과 효율이 중요한 문제

마무리 진단 문제

1. 재귀관계를 작성할 때 함께 정해야 하는 가장 작은 문제의 값은 무엇인가?
정답 보기초기값 또는 종료 조건입니다.
2. 피보나치 일반 재귀함수의 가장 큰 성능 문제는 무엇인가?
정답 보기같은 피보나치 항을 여러 번 중복 계산하는 것입니다.
3. 반복문 방식은 이전 상태를 어떻게 관리하는가?
정답 보기변수에 저장하고 반복할 때마다 새로운 상태로 갱신합니다.
4. 재귀 방식이 반복 방식보다 가독성이 높을 수 있는 경우는 언제인가?
정답 보기문제가 자기와 같은 더 작은 문제로 자연스럽게 분해될 때입니다.
5. 두 방식의 차이를 설명할 때 비교해야 할 세 가지 핵심 요소는 무엇인가?
정답 보기가독성, 메모리 사용, 실행 시간입니다.
