입력 크기별 연산 횟수 분석
O(1)·O(log n)·O(n)·O(n²) 그래프 비교 예제 10선

알고리즘의 실행 시간을 단순한 초 단위가 아니라 입력 크기 n에 따른 핵심 연산 횟수로 분석하고, 대표 시간 복잡도의 증가 양상을 표와 그래프로 비교합니다.

핵심 개념1. O(1)2. O(log n) 3. O(n)4. O(n²)5. 횟수 표 6. 통합 그래프7. 로그 축8. 실제 코드 측정 9. 입력 2배 비교10. 분석기

1. 시간 복잡도 분석의 핵심

O(1)입력 크기와 관계없이 핵심 연산 수가 거의 일정합니다.
O(log n)입력을 매 단계 일정 비율로 줄여 연산 수가 천천히 증가합니다.
O(n)입력 원소를 한 번씩 처리하여 연산 수가 n에 비례합니다.
O(n²)모든 원소 쌍처럼 이중 반복으로 연산 수가 n의 제곱에 비례합니다.
빅오 표기법은 정확한 실행 시간이 아니라 입력 크기가 매우 커질 때 연산량이 어떤 비율로 증가하는지를 나타냅니다. 상수 배수와 낮은 차수 항은 생략하고 가장 빠르게 증가하는 항을 중심으로 표현합니다.
복잡도대표 연산 수입력 2배 시 변화대표 사례
O(1)1거의 동일리스트 인덱스 접근
O(log n)log₂n약 1회 증가이진 탐색
O(n)n약 2배선형 탐색
O(n²)n²약 4배모든 원소 쌍 비교
예제 1

O(1): 인덱스로 한 원소 접근

상수 시간리스트 접근
def get_middle(data):
    operation_count = 1
    value = data[len(data) // 2]

    return value, operation_count

for n in [10, 100, 1000, 10000]:
    data = list(range(n))
    value, count = get_middle(data)

    print(
        f"n={n:5} | "
        f"연산 횟수={count}"
    )

분석

리스트의 크기가 커져도 가운데 인덱스를 계산하고 한 번 접근합니다.

따라서 핵심 연산 횟수는 입력 크기 n에 의존하지 않아 O(1)입니다.

예제 2

O(log n): 이진 탐색 비교 횟수

로그 시간범위 절반
def binary_search_count(data, target):
    left = 0
    right = len(data) - 1
    count = 0

    while left <= right:
        count += 1
        middle = (left + right) // 2

        if data[middle] == target:
            return middle, count

        if data[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

    return -1, count

for n in [8, 16, 32, 64, 128, 256]:
    data = list(range(n))
    _, count = binary_search_count(
        data, n - 1
    )

    print(f"n={n:3} | 비교={count}")
입력 크기 관계
입력이 8→16→32처럼 두 배가 되어도 최악의 비교 횟수는 대체로 한 번씩만 증가합니다.
예제 3

O(n): 선형 탐색 연산 횟수

선형 시간전체 순회
def linear_search_count(data, target):
    count = 0

    for index, value in enumerate(data):
        count += 1

        if value == target:
            return index, count

    return -1, count

for n in [10, 100, 1000, 10000]:
    data = list(range(n))

    _, count = linear_search_count(
        data, n - 1
    )

    print(
        f"n={n:5} | "
        f"최악 비교={count}"
    )
분석
목표값이 마지막에 있으면 모든 원소를 한 번씩 확인하므로 연산 횟수는 n과 같습니다.
예제 4

O(n²): 모든 원소 쌍 비교

제곱 시간이중 반복문
def compare_all_pairs(data):
    count = 0

    for i in range(len(data)):
        for j in range(len(data)):
            count += 1

    return count

for n in [10, 20, 40, 80]:
    count = compare_all_pairs(
        list(range(n))
    )

    print(
        f"n={n:3} | "
        f"연산 횟수={count}"
    )

입력 2배 효과

n=10 → 100회
n=20 → 400회
n=40 → 1600회
n=80 → 6400회

n이 두 배가 되면 n²은 네 배가 됩니다.

예제 8

같은 입력에서 실제 핵심 연산 횟수 측정

실험이론 검증
def operation_counts(n):
    # O(1)
    constant = 1

    # O(log n)
    value = n
    logarithmic = 0

    while value > 1:
        value //= 2
        logarithmic += 1

    # O(n)
    linear = 0
    for _ in range(n):
        linear += 1

    # O(n²)
    quadratic = 0
    for _ in range(n):
        for _ in range(n):
            quadratic += 1

    return {
        "O(1)": constant,
        "O(log n)": logarithmic,
        "O(n)": linear,
        "O(n²)": quadratic
    }

for n in [4, 8, 16, 32]:
    print(n, operation_counts(n))
결과
반복문의 실제 카운터 값이 이론식 1, log₂n, n, n²과 같은 증가 경향을 보이는지 확인합니다.
예제 9

입력을 두 배로 늘렸을 때 증가 비율

성장 비율2n / n
import math

def growth_ratio(n):
    functions = {
        "O(1)": lambda x: 1,
        "O(log n)": lambda x: math.log2(x),
        "O(n)": lambda x: x,
        "O(n²)": lambda x: x ** 2
    }

    for name, function in functions.items():
        before = function(n)
        after = function(2 * n)
        ratio = after / before

        print(
            f"{name:8} | "
            f"n={before:.2f} | "
            f"2n={after:.2f} | "
            f"비율={ratio:.2f}"
        )

예제 10

입력 크기 기반 연산 횟수 분석기

종합 실습상호작용
import math

def analyze(n):
    results = {
        "O(1)": 1,
        "O(log n)": math.ceil(math.log2(n)),
        "O(n)": n,
        "O(n²)": n ** 2
    }

    for name, operations in results.items():
        print(name, operations)

n = int(input("입력 크기 n: "))
analyze(n)
분석 문장 예시
“입력 크기가 두 배가 될 때 O(1)은 연산 수가 거의 변하지 않고, O(log n)은 약 한 단계 증가한다. O(n)은 약 두 배, O(n²)은 약 네 배 증가한다. 따라서 입력이 매우 커질수록 제곱 시간 알고리즘의 실행 부담이 선형·로그 시간보다 빠르게 커진다.”
