같은 부분 문제가 반복되는지와 부분 문제의 최적해가 전체 최적해를 구성하는지 확인하고, 하향식 메모이제이션과 상향식 테이블 방식으로 동적 계획법을 구현하여 차이를 비교합니다.

1. 동적 계획법 적용 조건

중복 부분 문제서로 다른 계산 경로에서 같은 입력 상태가 반복해서 등장합니다.
최적 부분 구조부분 문제의 최적해를 조합하여 전체 문제의 최적해를 만들 수 있습니다.
메모이제이션필요한 상태를 재귀적으로 계산하고 결과를 저장하는 하향식 방식입니다.
테이블 방식작은 상태부터 순서대로 계산해 큰 상태를 만드는 상향식 방식입니다.
동적 계획법은 단순히 값을 저장하는 기술이 아니라, 문제의 상태와 상태 전이 관계를 설계하는 방법입니다. 같은 상태가 반복되지 않거나 부분 문제의 최적 선택이 전체 최적해를 보장하지 않으면 다른 알고리즘이 더 적합할 수 있습니다.
비교 항목메모이제이션테이블 방식
계산 방향큰 문제에서 필요한 작은 문제로 이동작은 문제에서 큰 문제로 순서대로 이동
구현 형태재귀함수와 캐시반복문과 배열·표
계산 상태실제로 필요한 상태 중심정한 순서의 상태를 대부분 계산
호출 스택사용함사용하지 않음
장점점화식을 코드에 자연스럽게 표현재귀 깊이 제한이 없고 실행 흐름이 명확
예제 1

피보나치에서 중복 부분 문제 확인

호출 횟수중복 상태
calls = {}

def fibonacci(n):
    calls[n] = calls.get(n, 0) + 1

    if n < 2:
        return n

    return (
        fibonacci(n - 1)
        + fibonacci(n - 2)
    )

print("결과:", fibonacci(6))
print("입력별 호출 횟수:")

for n in sorted(calls):
    print(n, ":", calls[n])

중복 구조

F(6)
├─ F(5)
│  ├─ F(4)
│  └─ F(3)
└─ F(4)
   ├─ F(3)
   └─ F(2)

F(4), F(3), F(2) 등이 여러 경로에서 반복됩니다.

판단
같은 n에 대한 결과가 항상 같으므로 한 번 계산한 값을 저장해 재사용할 수 있습니다.
예제 2

피보나치 하향식 메모이제이션

Top-down딕셔너리 캐시
memo = {0: 0, 1: 1}
calculation_count = 0

def fibonacci_memo(n):
    global calculation_count

    if n in memo:
        return memo[n]

    calculation_count += 1

    memo[n] = (
        fibonacci_memo(n - 1)
        + fibonacci_memo(n - 2)
    )

    return memo[n]

print("결과:", fibonacci_memo(30))
print("새 계산 횟수:", calculation_count)
print("저장 상태 수:", len(memo))
계산 흐름
큰 문제 F(30)에서 시작해 필요한 작은 문제를 재귀적으로 계산하고, 이미 저장된 상태는 즉시 반환합니다.
예제 3

피보나치 상향식 테이블 방식

Bottom-upDP 테이블
def fibonacci_table(n):
    if n < 2:
        return n

    dp = [0] * (n + 1)
    dp[1] = 1

    for index in range(2, n + 1):
        dp[index] = (
            dp[index - 1]
            + dp[index - 2]
        )

    print("테이블:", dp)
    return dp[n]

print("결과:", fibonacci_table(10))

계산 순서

dp[0], dp[1]
→ dp[2]
→ dp[3]
→ dp[4]
→ ...
→ dp[n]

필요한 이전 값이 먼저 계산되도록 인덱스가 작은 순서로 채웁니다.

예제 4

피보나치 테이블 공간 최적화

O(1) 공간이전 두 상태
def fibonacci_optimized(n):
    if n < 2:
        return n

    previous2 = 0
    previous1 = 1

    for _ in range(2, n + 1):
        current = previous1 + previous2
        previous2, previous1 = (
            previous1,
            current
        )

    return previous1

print(fibonacci_optimized(100))
공간 분석
F(n)을 구할 때 F(n-1)과 F(n-2)만 필요하므로 전체 테이블 대신 두 개의 이전 값만 저장할 수 있습니다.
예제 5

계단 오르기: 메모이제이션과 테이블 비교

경우의 수두 방식

메모이제이션

memo = {0: 1, 1: 1}

def stairs_memo(n):
    if n in memo:
        return memo[n]

    memo[n] = (
        stairs_memo(n - 1)
        + stairs_memo(n - 2)
    )

    return memo[n]

print(stairs_memo(10))

테이블 방식

def stairs_table(n):
    dp = [0] * (n + 1)
    dp[0] = 1

    if n >= 1:
        dp[1] = 1

    for step in range(2, n + 1):
        dp[step] = (
            dp[step - 1]
            + dp[step - 2]
        )

    return dp[n]

print(stairs_table(10))
최적 부분 구조
n칸에 도착하는 방법은 n-1칸까지의 모든 방법 뒤에 1칸 이동을 붙이거나, n-2칸까지의 모든 방법 뒤에 2칸 이동을 붙여 구성됩니다.
예제 6

격자 경로 수: 2차원 상태

2차원 DP행·열 상태
def count_paths(rows, cols):
    dp = [
        [0] * cols
        for _ in range(rows)
    ]

    for row in range(rows):
        dp[row][0] = 1

    for col in range(cols):
        dp[0][col] = 1

    for row in range(1, rows):
        for col in range(1, cols):
            dp[row][col] = (
                dp[row - 1][col]
                + dp[row][col - 1]
            )

    for line in dp:
        print(line)

    return dp[rows - 1][cols - 1]

print("경로 수:", count_paths(4, 5))
상태 전이
현재 칸으로 오는 방법은 위쪽 칸에서 내려오는 방법과 왼쪽 칸에서 오른쪽으로 오는 방법의 합입니다.
예제 7

최소 동전 수: 최적 부분 구조

최소화동전 교환
def min_coin_table(coins, amount):
    dp = [float("inf")] * (amount + 1)
    dp[0] = 0

    for value in range(1, amount + 1):
        for coin in coins:
            if coin <= value:
                dp[value] = min(
                    dp[value],
                    dp[value - coin] + 1
                )

    return (
        dp[amount]
        if dp[amount] != float("inf")
        else -1
    )

print(min_coin_table([1, 3, 4], 6))

점화 관계

dp[x] =
min(
 dp[x-1] + 1,
 dp[x-3] + 1,
 dp[x-4] + 1
)

마지막에 사용한 동전을 제거하면 더 작은 금액의 최적 동전 문제가 됩니다.

탐욕법과 차이
동전 [1, 3, 4]로 6을 만들 때 큰 동전 우선은 4+1+1이지만, 최적해는 3+3입니다.
예제 8

연속 부분 배열의 최대 합

카데인 알고리즘1차원 테이블
def maximum_subarray(numbers):
    dp = [0] * len(numbers)
    dp[0] = numbers[0]

    best = dp[0]

    for index in range(1, len(numbers)):
        dp[index] = max(
            numbers[index],
            dp[index - 1] + numbers[index]
        )

        best = max(best, dp[index])

    print("상태값:", dp)
    return best

numbers = [-2, 1, -3, 4, -1, 2, 1, -5, 4]

print("최대 합:", maximum_subarray(numbers))
상태 정의
dp[i]는 반드시 i번째 원소에서 끝나는 연속 부분 배열 중 최대 합입니다. 새로 시작할지 이전 부분 배열에 이어 붙일지 선택합니다.
예제 9

0/1 배낭 문제의 2차원 테이블

선택·미선택최대 가치
def knapsack(weights, values, capacity):
    item_count = len(weights)

    dp = [
        [0] * (capacity + 1)
        for _ in range(item_count + 1)
    ]

    for item in range(1, item_count + 1):
        weight = weights[item - 1]
        value = values[item - 1]

        for limit in range(capacity + 1):
            # 현재 물건을 넣지 않음
            dp[item][limit] = dp[item - 1][limit]

            # 현재 물건을 넣음
            if weight <= limit:
                dp[item][limit] = max(
                    dp[item][limit],
                    dp[item - 1][limit - weight]
                    + value
                )

    for row in dp:
        print(row)

    return dp[item_count][capacity]

weights = [2, 3, 4, 5]
values = [3, 4, 5, 8]

print("최대 가치:", knapsack(
    weights,
    values,
    capacity=8
))
최적 부분 구조
현재 물건을 넣지 않은 최적값과, 현재 물건을 넣고 남은 용량에서 이전 물건들로 만든 최적값을 비교합니다.
예제 10

메모이제이션·테이블 방식 종합 비교

호출 수실행 시간종합 실험
import time

def compare_fibonacci(n):
    memo = {0: 0, 1: 1}
    memo_calls = 0

    def top_down(value):
        nonlocal memo_calls
        memo_calls += 1

        if value in memo:
            return memo[value]

        memo[value] = (
            top_down(value - 1)
            + top_down(value - 2)
        )

        return memo[value]

    start = time.perf_counter()
    top_result = top_down(n)
    top_time = time.perf_counter() - start

    start = time.perf_counter()

    if n < 2:
        bottom_result = n
        table = [n]
    else:
        table = [0] * (n + 1)
        table[1] = 1

        for index in range(2, n + 1):
            table[index] = (
                table[index - 1]
                + table[index - 2]
            )

        bottom_result = table[n]

    bottom_time = time.perf_counter() - start

    return {
        "결과 일치": top_result == bottom_result,
        "메모 호출 수": memo_calls,
        "메모 저장 수": len(memo),
        "테이블 칸 수": len(table),
        "메모 시간": top_time,
        "테이블 시간": bottom_time
    }

result = compare_fibonacci(500)

for key, value in result.items():
    if "시간" in key:
        print(f"{key}: {value:.8f}초")
    else:
        print(key, ":", value)
판단 항목메모이제이션테이블 방식
점화식 표현재귀관계와 유사하여 직관적상태 계산 순서를 직접 설계
필요한 상태만 계산가능일반적으로 정한 범위를 계산
재귀 깊이큰 입력에서 제한 가능재귀 깊이 문제 없음
상태 추적호출 흐름을 따라가야 함테이블을 출력해 확인하기 쉬움
공간 최적화캐시 제거가 복잡할 수 있음필요한 이전 상태만 남기기 쉬움
시간 복잡도상태 수 × 상태당 전이 비용상태 수 × 상태당 전이 비용
논리적 설명 예시
“피보나치 문제에서는 F(n-2), F(n-3) 등의 부분 문제가 여러 재귀 경로에서 반복되므로 중복 부분 문제가 존재한다. 또한 F(n)의 해는 F(n-1)과 F(n-2)의 해로 구성되므로 최적 부분 구조와 같은 부분 문제 결합 구조를 가진다. 메모이제이션은 필요한 값을 재귀적으로 계산하여 저장하고, 테이블 방식은 작은 인덱스부터 순서대로 값을 채운다.”

마무리 진단 문제

1. 중복 부분 문제란 무엇인가?
정답 보기문제 해결 과정에서 입력 상태가 같은 부분 문제가 여러 번 반복해서 계산되는 성질입니다.
2. 최적 부분 구조란 무엇인가?
정답 보기전체 문제의 최적해가 부분 문제들의 최적해를 이용해 구성되는 성질입니다.
3. 메모이제이션은 어느 방향으로 문제를 해결하는가?
정답 보기큰 문제에서 시작해 필요한 작은 문제를 재귀적으로 계산하는 하향식 방식입니다.
4. 테이블 방식의 대표적인 장점은 무엇인가?
정답 보기재귀 호출 스택을 사용하지 않아 재귀 깊이 제한이 없고 계산 순서를 표로 확인하기 쉽습니다.
5. 피보나치 테이블을 두 변수로 줄일 수 있는 이유는 무엇인가?
정답 보기현재 값을 계산할 때 바로 이전 두 값만 필요하기 때문입니다.
