산출물 상호 평가와 개선 예제 10선

프로그램과 데이터 기반 산출물을 여섯 관점에서 동료 평가하고, 평가 점수와 근거를 바탕으로 코드·화면·데이터 처리·출처 표시를 개선합니다.

평가 원리1. 평가표2. 정확성 3. 효율성4. 사용성5. 개인정보 6. 편향7. 저작권8. 동료 피드백 9. 개선 우선순위10. 종합 평가

1. 산출물 평가의 여섯 관점

정확성요구 사항과 테스트 결과가 일치하고 오류 처리가 적절한가?
효율성입력 크기에 비해 시간·메모리 사용이 적절한가?
사용성사용자가 쉽게 이해하고 실수 없이 사용할 수 있는가?
개인정보불필요한 개인 정보를 수집·보관·공개하지 않는가?
편향특정 집단이나 조건에 불리한 결과를 만들지 않는가?
저작권코드·데이터·이미지·글꼴의 출처와 라이선스를 지켰는가?
상호 평가는 단순한 점수 부여가 아니라 “근거 있는 장점 1개, 개선점 1개, 실행 가능한 제안 1개”를 남기는 과정입니다.
평가 수준판단 기준
5점기준을 충분히 만족하며 근거와 검증 자료가 명확함
4점대부분 만족하지만 일부 보완이 필요함
3점기본 기능은 있으나 근거·예외 처리·문서화가 부족함
2점중요한 오류나 위험이 있어 사용 전에 수정이 필요함
1점기준을 거의 만족하지 못하며 전면 재설계가 필요함
예제 1

여섯 관점 상호 평가표 자동 계산

평가 점수평균
scores = {
    "정확성": 4,
    "효율성": 3,
    "사용성": 4,
    "개인정보": 5,
    "편향": 3,
    "저작권": 4
}

average = sum(scores.values()) / len(scores)

for category, score in scores.items():
    print(category, ":", score, "점")

print("평균:", round(average, 2))

lowest = min(
    scores,
    key=scores.get
)

print("우선 개선 영역:", lowest)
활용
가장 낮은 영역을 우선 개선 대상으로 정하고, 점수만이 아니라 근거 문장을 함께 기록합니다.
예제 2

정확성 평가: 테스트 통과율 확인

테스트정확성
def classify_score(score):
    if not 0 <= score <= 100:
        raise ValueError(
            "점수 범위 오류"
        )

    return "합격" if score >= 60 else "불합격"

tests = [
    (75, "합격"),
    (60, "합격"),
    (59, "불합격"),
    (0, "불합격"),
    (100, "합격")
]

passed = 0

for value, expected in tests:
    actual = classify_score(value)

    if actual == expected:
        passed += 1
        print("PASS", value)
    else:
        print("FAIL", value, actual, expected)

print("통과율:", passed / len(tests) * 100)

평가 질문

정상·경계값을 모두 통과했는가?

잘못된 입력에서 의미 있는 오류를 제공하는가?

요구 사항과 코드 결과가 일치하는가?

예제 3

효율성 평가: 이중 반복문을 집합으로 개선

O(n²)평균 O(n)

개선 전

def has_duplicate_slow(data):
    for i in range(len(data)):
        for j in range(i + 1, len(data)):
            if data[i] == data[j]:
                return True

    return False

개선 후

def has_duplicate_fast(data):
    seen = set()

    for value in data:
        if value in seen:
            return True

        seen.add(value)

    return False
평가
개선 후 시간 복잡도는 평균 O(n)으로 줄지만, 최대 n개의 값을 저장하는 추가 메모리가 필요합니다. 시간과 공간을 함께 평가합니다.
예제 4

사용성 평가: 모호한 출력 개선

사용자 메시지가독성

개선 전

score = 58
print(False)

개선 후

score = 58
minimum = 60

if score >= minimum:
    print(
        f"합격입니다. 점수: {score}"
    )
else:
    shortage = minimum - score

    print(
        f"불합격입니다. "
        f"합격까지 {shortage}점 부족합니다."
    )
사용성 개선
단순한 참·거짓 대신 현재 상태와 다음 행동에 필요한 정보를 제공합니다.
예제 5

개인정보 평가: 이름을 익명 식별자로 변환

비식별화최소 수집

개선 전

records = [
    {
        "name": "김민준",
        "phone": "010-1234-5678",
        "score": 85
    }
]

print(records)

개선 후

import hashlib

def anonymize(name):
    return hashlib.sha256(
        name.encode("utf-8")
    ).hexdigest()[:10]

records = [
    {
        "student_id": anonymize("김민준"),
        "score": 85
    }
]

print(records)
주의
해시값도 다른 정보와 결합하면 개인을 추정할 수 있습니다. 분석 목적에 불필요한 전화번호와 이름은 수집하지 않는 것이 가장 안전합니다.
예제 6

편향 평가: 집단별 정확도 비교

편향집단별 성능
results = [
    {"group": "A", "correct": True},
    {"group": "A", "correct": True},
    {"group": "A", "correct": False},
    {"group": "B", "correct": True},
    {"group": "B", "correct": False},
    {"group": "B", "correct": False}
]

stats = {}

for item in results:
    group = item["group"]

    if group not in stats:
        stats[group] = {
            "total": 0,
            "correct": 0
        }

    stats[group]["total"] += 1

    if item["correct"]:
        stats[group]["correct"] += 1

for group, value in stats.items():
    accuracy = (
        value["correct"]
        / value["total"]
    )

    print(
        group,
        "정확도:",
        round(accuracy, 3)
    )
평가 해석
전체 정확도만 보면 특정 집단에서 성능이 낮은 문제를 놓칠 수 있습니다. 표본 수와 데이터 대표성도 함께 확인합니다.
예제 7

저작권 평가: 출처·라이선스 누락 점검

출처 표시라이선스
assets = [
    {
        "name": "배경 이미지",
        "source": "직접 제작",
        "license": "자체 제작"
    },
    {
        "name": "기후 데이터",
        "source": "공공데이터 포털",
        "license": "공공누리 조건 확인"
    },
    {
        "name": "아이콘",
        "source": "",
        "license": ""
    }
]

for asset in assets:
    missing = []

    if not asset["source"]:
        missing.append("출처")

    if not asset["license"]:
        missing.append("라이선스")

    if missing:
        print(
            asset["name"],
            "보완 필요:",
            ", ".join(missing)
        )
    else:
        print(
            asset["name"],
            "표시 완료"
        )
개선
출처가 불분명한 아이콘은 삭제하거나 사용 조건이 명확한 자료로 교체합니다.
예제 8

동료 피드백을 구체적인 개선 과제로 변환

피드백실행 계획
feedback = [
    {
        "category": "사용성",
        "comment": "오류 메시지가 어렵다.",
        "action": "사용자용 쉬운 문장으로 변경",
        "priority": 1
    },
    {
        "category": "개인정보",
        "comment": "이름을 저장할 필요가 없다.",
        "action": "익명 번호만 저장",
        "priority": 1
    },
    {
        "category": "효율성",
        "comment": "중복 탐색이 많다.",
        "action": "집합 기반 탐색 적용",
        "priority": 2
    }
]

ordered = sorted(
    feedback,
    key=lambda item: item["priority"]
)

for item in ordered:
    print(
        item["priority"],
        item["category"],
        "→",
        item["action"]
    )
좋은 피드백
“별로다”가 아니라 문제 상황, 근거, 수정 방법, 우선순위를 포함합니다.
예제 9

위험도와 비용을 반영한 개선 우선순위

우선순위위험도
issues = [
    {
        "name": "개인정보 노출",
        "impact": 5,
        "urgency": 5,
        "effort": 2
    },
    {
        "name": "느린 중복 탐색",
        "impact": 3,
        "urgency": 3,
        "effort": 2
    },
    {
        "name": "출력 문장 불명확",
        "impact": 3,
        "urgency": 4,
        "effort": 1
    }
]

for issue in issues:
    issue["score"] = (
        issue["impact"]
        * issue["urgency"]
        / issue["effort"]
    )

issues.sort(
    key=lambda item: item["score"],
    reverse=True
)

for issue in issues:
    print(
        issue["name"],
        round(issue["score"], 2)
    )
우선 개선 원칙
개인정보 노출·차별·저작권 침해처럼 피해가 큰 문제는 기능 개선보다 먼저 수정합니다.
예제 10

종합: 상호 평가 결과와 개선 전후 비교

종합 평가개선 기록
before = {
    "정확성": 3,
    "효율성": 2,
    "사용성": 3,
    "개인정보": 2,
    "편향": 2,
    "저작권": 3
}

after = {
    "정확성": 5,
    "효율성": 4,
    "사용성": 4,
    "개인정보": 5,
    "편향": 4,
    "저작권": 5
}

for category in before:
    change = after[category] - before[category]

    print(
        category,
        before[category],
        "→",
        after[category],
        f"(+{change})"
    )

before_average = (
    sum(before.values())
    / len(before)
)

after_average = (
    sum(after.values())
    / len(after)
)

print("개선 전 평균:", round(before_average, 2))
print("개선 후 평균:", round(after_average, 2))
관점발견 문제개선 내용검증 방법
정확성경계값 60 처리 오류>를 >=로 수정경계 테스트 통과 확인
효율성중복 검사 O(n²)집합 기반 평균 O(n) 방식 적용입력 크기별 실행 시간 비교
사용성참·거짓만 출력상태와 다음 행동 안내사용자 설문과 과제 성공률
개인정보이름·전화번호 저장불필요 항목 삭제·익명화저장 파일 항목 점검
편향집단별 성능 차이균형 데이터 추가·집단별 평가집단별 정확도 격차 확인
저작권아이콘 출처 누락출처·라이선스 표시 또는 교체자산 목록 점검
상호 평가 서술 예시
“산출물을 정확성, 효율성, 사용성, 개인정보, 편향, 저작권 관점에서 평가하였다. 정확성에서는 경계값 처리 오류를 발견해 테스트를 추가하였고, 효율성에서는 이중 반복문을 집합 기반 탐색으로 개선하였다. 개인정보 측면에서는 불필요한 이름과 전화번호를 삭제하였으며, 편향 여부를 확인하기 위해 집단별 정확도를 비교하였다. 출처가 누락된 아이콘은 삭제하고 사용 조건이 명확한 자료로 교체하였다. 개선 후 동일한 평가표로 다시 점검하여 모든 영역의 점수가 향상되었음을 확인하였다.”

마무리 진단 문제

1. 상호 평가에서 점수와 함께 반드시 기록해야 할 것은 무엇인가?
정답 보기점수의 근거, 구체적인 문제점, 실행 가능한 개선 제안입니다.
2. 정확성 평가에 적합한 자료는 무엇인가?
정답 보기정상·경계·예외 입력 테스트 결과와 요구 사항 일치 여부입니다.
3. 개인정보 보호에서 가장 우선하는 원칙은 무엇인가?
정답 보기목적에 불필요한 개인정보를 처음부터 수집하지 않는 최소 수집 원칙입니다.
4. 편향을 확인할 때 전체 정확도만 보면 안 되는 이유는 무엇인가?
정답 보기특정 집단에서만 성능이 낮은 문제를 전체 평균이 가릴 수 있기 때문입니다.
5. 저작권 문제가 발견되었을 때 가능한 개선 방법은 무엇인가?
정답 보기출처와 라이선스를 올바르게 표시하거나, 사용 허가가 명확한 자료 또는 직접 제작한 자료로 교체합니다.
