프로그램의 정상 동작뿐 아니라 경계값과 잘못된 입력을 체계적으로 검사하고, 오류를 같은 조건에서 재현한 뒤 원인을 분석하여 수정하고 변경 이력을 기록합니다.

테스트 원리1. 입력 분류2. 점수 판정 3. 나눗셈4. 리스트 접근5. 문자열 숫자 6. 평균 계산7. 파일 입력8. 단위 테스트 9. 오류 기록10. 종합 회귀 테스트

1. 테스트와 디버깅의 기본 절차

① 정상 입력일반적인 사용 상황에서 기대한 결과가 나오는지 확인합니다.
② 경계 입력최솟값·최댓값·0·빈 값·임계값 바로 전후를 검사합니다.
③ 예외 입력자료형 오류, 범위 초과, 누락 데이터, 존재하지 않는 파일을 확인합니다.
④ 재현·수정·기록오류 조건을 고정하고 수정 전후 결과와 원인을 기록합니다.
좋은 테스트는 단순히 많은 값을 넣는 것이 아니라 입력 범주를 나누어 대표값을 선택합니다. 오류 수정 후에는 같은 오류가 다시 나타나지 않도록 회귀 테스트를 남겨야 합니다.
테스트 유형예시목적
정상 테스트점수 75, 양수 목록, 존재하는 파일기본 기능 확인
경계 테스트0, 100, 임계값 60, 빈 목록조건식의 경계 오류 확인
예외 테스트"abc", 음수, 인덱스 초과, 파일 없음오류 처리 확인
회귀 테스트이전에 실패했던 입력 재실행수정한 오류의 재발 방지

def is_pass(score):
    if not 0 <= score <= 100:
        raise ValueError(
            "점수는 0~100이어야 합니다."
        )

    return score >= 60

print(is_pass(60))
# 실제값: True

예제 3

0으로 나누기 예외 처리

ZeroDivisionError예외 입력

오류 재현

def average(total, count):
    return total / count

print(average(100, 0))
# ZeroDivisionError

수정

def average(total, count):
    if count <= 0:
        raise ValueError(
            "개수는 1 이상이어야 합니다."
        )

    return total / count

tests = [
    (100, 4),
    (0, 1),
    (100, 0)
]

for total, count in tests:
    try:
        print(average(total, count))
    except ValueError as error:
        print("오류:", error)
해석
내부의 ZeroDivisionError를 그대로 노출하기보다 사용자에게 의미 있는 입력 오류로 변환합니다.
예제 4

리스트 인덱스 경계 테스트

IndexError범위 검사
def get_item(data, index):
    if not isinstance(index, int):
        raise TypeError(
            "인덱스는 정수여야 합니다."
        )

    if index < 0 or index >= len(data):
        raise IndexError(
            "인덱스가 범위를 벗어났습니다."
        )

    return data[index]

data = ["A", "B", "C"]

test_indexes = [0, 2, 3, -1, 1.5]

for index in test_indexes:
    try:
        print(index, "→", get_item(data, index))
    except (TypeError, IndexError) as error:
        print(index, "→ 오류:", error)
경계값
첫 인덱스 0, 마지막 인덱스 len(data)-1, 바로 바깥값 len(data)를 모두 검사합니다.

def parse_integer(text):
    cleaned = text.strip()

    if cleaned == "":
        raise ValueError(
            "빈 문자열은 정수로 바꿀 수 없습니다."
        )

    try:
        return int(cleaned)

    except ValueError as error:
        raise ValueError(
            f"정수 형식이 아닙니다: {text!r}"
        ) from error

tests = [
    "42",
    "  17 ",
    "0",
    "-3",
    "",
    "12.5",
    "abc"
]

for text in tests:
    try:
        print(text, "→", parse_integer(text))
    except ValueError as error:
        print(text, "→ 오류:", error)
테스트 범주
공백 포함 정상값, 0, 음수, 빈 문자열, 실수 문자열, 문자 문자열을 구분해 검사합니다.
예제 6

빈 목록 평균 오류와 수정

빈 입력경계 조건

오류 코드

def mean(values):
    return sum(values) / len(values)

print(mean([]))
# ZeroDivisionError

수정 코드

def mean(values):
    if not values:
        raise ValueError(
            "평균을 계산할 값이 없습니다."
        )

    return sum(values) / len(values)

for values in [
    [10, 20, 30],
    [0],
    [],
]:
    try:
        print(values, mean(values))
    except ValueError as error:
        print(values, "오류:", error)
설계 선택
빈 목록에서 0을 반환할지 예외를 발생시킬지는 프로젝트 규칙에 따라 결정해야 합니다. 중요한 점은 모든 모듈이 같은 규칙을 사용하는 것입니다.

from pathlib import Path

def read_text_file(filename):
    path = Path(filename)

    if not path.exists():
        raise FileNotFoundError(
            f"파일을 찾을 수 없습니다: {filename}"
        )

    if path.suffix.lower() != ".txt":
        raise ValueError(
            "txt 파일만 읽을 수 있습니다."
        )

    return path.read_text(
        encoding="utf-8"
    )

tests = [
    "data.txt",
    "missing.txt",
    "image.png"
]

for filename in tests:
    try:
        text = read_text_file(filename)
        print(filename, len(text))
    except (FileNotFoundError, ValueError) as error:
        print(filename, "오류:", error)
재현 정보
운영체제, 파일 경로, 확장자, 인코딩, 실행 위치를 함께 기록하면 파일 관련 오류를 다른 컴퓨터에서도 재현하기 쉽습니다.
예제 8

unittest로 정상·경계·예외 테스트 자동화

unittestassertRaises
import unittest

def classify_score(score):
    if not isinstance(score, (int, float)):
        raise TypeError(
            "점수는 숫자여야 합니다."
        )

    if not 0 <= score <= 100:
        raise ValueError(
            "점수 범위는 0~100입니다."
        )

    return "합격" if score >= 60 else "불합격"

class TestClassifyScore(unittest.TestCase):
    def test_normal(self):
        self.assertEqual(
            classify_score(75),
            "합격"
        )

    def test_boundaries(self):
        self.assertEqual(
            classify_score(60),
            "합격"
        )
        self.assertEqual(
            classify_score(59),
            "불합격"
        )
        self.assertEqual(
            classify_score(0),
            "불합격"
        )
        self.assertEqual(
            classify_score(100),
            "합격"
        )

    def test_invalid_type(self):
        with self.assertRaises(TypeError):
            classify_score("75")

    def test_out_of_range(self):
        with self.assertRaises(ValueError):
            classify_score(101)

if __name__ == "__main__":
    unittest.main()
자동화
코드를 수정할 때마다 테스트를 다시 실행하면 이전 기능이 깨졌는지 빠르게 확인할 수 있습니다.

예제 10

종합: 테스트 실행기와 회귀 테스트

종합 테스트PASS·FAIL
def calculate_discount(price, rate):
    if not isinstance(price, (int, float)):
        raise TypeError(
            "가격은 숫자여야 합니다."
        )

    if not isinstance(rate, (int, float)):
        raise TypeError(
            "할인율은 숫자여야 합니다."
        )

    if price < 0:
        raise ValueError(
            "가격은 0 이상이어야 합니다."
        )

    if not 0 <= rate <= 100:
        raise ValueError(
            "할인율은 0~100이어야 합니다."
        )

    return price * (1 - rate / 100)

test_cases = [
    {
        "name": "정상 할인",
        "args": (10000, 20),
        "expected": 8000
    },
    {
        "name": "경계 할인 0%",
        "args": (10000, 0),
        "expected": 10000
    },
    {
        "name": "경계 할인 100%",
        "args": (10000, 100),
        "expected": 0
    },
    {
        "name": "가격 0원",
        "args": (0, 50),
        "expected": 0
    },
    {
        "name": "음수 가격",
        "args": (-1000, 10),
        "exception": ValueError
    },
    {
        "name": "할인율 초과",
        "args": (10000, 120),
        "exception": ValueError
    },
    {
        "name": "문자 가격",
        "args": ("10000", 20),
        "exception": TypeError
    }
]

passed = 0

for case in test_cases:
    try:
        result = calculate_discount(
            *case["args"]
        )

        if "exception" in case:
            print(
                "FAIL",
                case["name"],
                "예외가 발생하지 않음"
            )
            continue

        if result == case["expected"]:
            print("PASS", case["name"])
            passed += 1
        else:
            print(
                "FAIL",
                case["name"],
                result,
                case["expected"]
            )

    except Exception as error:
        expected_error = case.get(
            "exception"
        )

        if (
            expected_error
            and isinstance(error, expected_error)
        ):
            print("PASS", case["name"])
            passed += 1
        else:
            print(
                "FAIL",
                case["name"],
                type(error).__name__,
                error
            )

print(
    f"통과: {passed}/{len(test_cases)}"
)

오류 수정 과정 설명 예시
“정상 입력 75와 경계 입력 60·59, 예외 입력 문자열을 테스트 사례로 설계하였다. 60점 입력에서 기대 결과는 합격이었으나 실제로는 불합격이 출력되어 오류를 재현하였다. 원인은 조건식에 score > 60을 사용한 것이었으며, 이를 score >= 60으로 수정하고 0~100 범위 검증을 추가하였다. 동일 오류의 재발을 막기 위해 60점 경계 테스트를 단위 테스트에 포함하고 수정 내용을 버전 기록에 남겼다.”

마무리 진단 문제

1. 경계값 테스트가 중요한 이유는 무엇인가?
정답 보기조건식의 포함·제외 오류와 배열 범위 오류가 경계에서 자주 발생하기 때문입니다.
2. 오류 재현 절차에 포함해야 할 정보는 무엇인가?
정답 보기실행 환경, 입력값, 실행 순서, 기대 결과, 실제 결과를 포함해야 합니다.
3. 회귀 테스트란 무엇인가?
정답 보기수정한 오류가 다시 발생하지 않는지 확인하기 위해 이전 실패 입력을 반복 실행하는 테스트입니다.
4. 예외 테스트에서 확인해야 하는 것은 무엇인가?
정답 보기예상한 예외 종류와 메시지가 발생하고 프로그램이 오류를 적절히 처리하는지 확인해야 합니다.
5. 오류 기록에 원인과 수정 내용 외에 추가하면 좋은 항목은 무엇인가?
정답 보기재현 절차, 환경, 기대·실제 결과, 회귀 테스트, 수정 버전과 날짜를 기록하면 좋습니다.
