입력 크기 관계·점화식·도식 예제 10선

입력 크기 n인 문제를 더 작은 부분 문제로 나누고, 부분 문제의 크기와 개수, 결합 비용의 관계를 점화식·호출 트리·단계 표로 표현합니다.

핵심 원리1. 자연수 합2. 팩토리얼 3. 피보나치4. 계단 오르기5. 이진 탐색 6. 거듭제곱7. 병합 정렬8. 최대값 9. 하노이 탑10. 종합 분석

1. 부분 문제 분해와 점화식

입력 크기문제의 규모를 나타내는 n을 정의합니다.
부분 문제n보다 작은 크기의 같은 형태 문제를 찾습니다.
결합 과정부분 문제의 해를 이용해 원래 문제의 해를 만듭니다.
점화식현재 문제와 작은 문제의 관계를 식으로 표현합니다.
점화식은 보통 부분 문제의 개수, 부분 문제의 크기, 분해·결합에 필요한 추가 작업으로 구성됩니다. 예를 들어 입력을 절반 크기의 두 문제로 나누고 결합에 n만큼의 작업이 필요하면 T(n)=2T(n/2)+n과 같이 표현할 수 있습니다.
확인 요소질문
기본 상태입력 크기가 얼마일 때 직접 답을 구할 수 있는가?
크기 변화n-1, n-2, n/2 등 어떤 크기로 줄어드는가?
부분 문제 수재귀 호출이 한 번인가, 두 번 이상인가?
추가 작업덧셈, 비교, 병합, 원판 이동 등이 얼마나 필요한가?
예제 1

1부터 n까지의 합

n-1 문제선형 점화식
def recursive_sum(n):
    if n == 1:
        return 1

    return n + recursive_sum(n - 1)

print(recursive_sum(5))

점화식

S(1) = 1
S(n) = n + S(n-1)

도식

S(5)
└─ 5 + S(4)
       └─ 4 + S(3)
              └─ 3 + S(2)
                     └─ 2 + S(1)
크기 변화
입력 크기가 매 호출마다 1씩 감소하므로 5→4→3→2→1의 관계를 가집니다.
예제 2

팩토리얼의 부분 문제 관계

곱셈 결합기본 상태
def factorial(n):
    if n == 0:
        return 1

    return n * factorial(n - 1)

for n in range(6):
    print(n, factorial(n))

값의 점화식

F(0) = 1
F(n) = n × F(n-1)

실행 시간 관계

T(0) = c
T(n) = T(n-1) + c

한 번의 재귀 호출과 한 번의 곱셈이 추가됩니다.

예제 3

피보나치 호출 트리와 중복 부분 문제

두 부분 문제호출 트리
def fibonacci(n):
    if n < 2:
        return n

    return (
        fibonacci(n - 1)
        + fibonacci(n - 2)
    )

print(fibonacci(5))

값의 점화식

F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2)

호출 도식

F(5)
├─ F(4)
│  ├─ F(3)
│  └─ F(2)
└─ F(3)
   ├─ F(2)
   └─ F(1)
중복
F(3)과 F(2)가 여러 번 나타납니다. 점화식은 자연스럽지만 단순 재귀 구현은 중복 계산이 많습니다.
예제 4

계단 오르기의 상태 분해

경우의 수마지막 선택
def count_ways(n):
    if n == 0:
        return 1
    if n == 1:
        return 1

    return (
        count_ways(n - 1)
        + count_ways(n - 2)
    )

print(count_ways(5))

분해 논리

n번째 칸에 도착하는 마지막 이동은 1칸 또는 2칸입니다.

W(n) = W(n-1) + W(n-2)
W(0) = 1, W(1) = 1

입력 크기 n이 n-1과 n-2의 두 부분 문제로 나뉩니다.

예제 5

이진 탐색의 입력 절반 분해

n/2 문제로그 단계
def binary_search(data, target, left, right):
    if left > right:
        return -1

    middle = (left + right) // 2

    if data[middle] == target:
        return middle

    if data[middle] < target:
        return binary_search(
            data, target, middle + 1, right
        )

    return binary_search(
        data, target, left, middle - 1
    )

numbers = list(range(1, 33))
print(binary_search(
    numbers, 29, 0, len(numbers) - 1
))

시간 점화식

T(1) = c
T(n) = T(n/2) + c

크기 변화

32 → 16 → 8 → 4 → 2 → 1

입력이 두 배가 되어도 탐색 단계는 약 한 단계만 증가합니다.

예제 6

빠른 거듭제곱의 절반 분해

분할 정복짝수·홀수
def fast_power(base, exponent):
    if exponent == 0:
        return 1

    half = fast_power(
        base,
        exponent // 2
    )

    if exponent % 2 == 0:
        return half * half

    return base * half * half

print(fast_power(2, 13))
관계
지수 n을 매번 절반으로 줄입니다. 짝수이면 절반 결과를 제곱하고, 홀수이면 밑을 한 번 더 곱합니다.
P(a, 0) = 1
P(a, n) = P(a, n/2)²              (n이 짝수)
P(a, n) = a × P(a, floor(n/2))²   (n이 홀수)
예제 7

병합 정렬의 두 부분 문제와 결합

두 절반병합 비용
def merge_sort(data):
    if len(data) <= 1:
        return data

    middle = len(data) // 2

    left = merge_sort(data[:middle])
    right = merge_sort(data[middle:])

    result = []
    i = j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    result.extend(left[i:])
    result.extend(right[j:])

    return result

print(merge_sort([7, 3, 8, 2, 5, 1]))

점화식

T(1) = c
T(n) = 2T(n/2) + n

절반 크기 문제 두 개와 n개 원소 병합 작업으로 구성됩니다.

분해 도식

[7,3,8,2]
├─ [7,3]
│  ├─ [7]
│  └─ [3]
└─ [8,2]
   ├─ [8]
   └─ [2]
예제 8

배열 최대값을 절반으로 나누어 구하기

최댓값두 결과 결합
def divide_max(data, left, right):
    if left == right:
        return data[left]

    middle = (left + right) // 2

    left_max = divide_max(
        data, left, middle
    )
    right_max = divide_max(
        data, middle + 1, right
    )

    return max(left_max, right_max)

numbers = [12, 5, 19, 3, 21, 8]
print(divide_max(
    numbers, 0, len(numbers) - 1
))
결합
왼쪽 절반의 최대값과 오른쪽 절반의 최대값을 비교하면 전체 최대값을 얻을 수 있습니다.
M(1) = 원소값
M(n) = max(M(왼쪽 절반), M(오른쪽 절반))

T(n) = 2T(n/2) + c
예제 9

하노이 탑의 점화식과 이동 도식

n-1 두 번이동 횟수
def hanoi(n, start, target, helper):
    if n == 1:
        print(start, "→", target)
        return

    hanoi(
        n - 1,
        start,
        helper,
        target
    )

    print(start, "→", target)

    hanoi(
        n - 1,
        helper,
        target,
        start
    )

hanoi(3, "A", "C", "B")

이동 횟수 점화식

H(1) = 1
H(n) = 2H(n-1) + 1

문제 분해

n개 이동
├─ n-1개를 보조 기둥으로
├─ 가장 큰 원판 1개 이동
└─ n-1개를 목표 기둥으로
예제 10

점화식과 입력 크기 관계 종합 분석

종합 비교도식 출력
def show_size_change(
    n,
    rule,
    minimum=1
):
    sizes = [n]

    while n > minimum:
        if rule == "minus_one":
            n -= 1

        elif rule == "half":
            n //= 2

        else:
            raise ValueError(
                "지원하지 않는 규칙입니다."
            )

        sizes.append(max(n, minimum))

    return sizes

print(
    "n-1 감소:",
    show_size_change(8, "minus_one")
)

print(
    "절반 감소:",
    show_size_change(32, "half")
)
문제부분 문제 관계대표 점화식입력 크기 도식
자연수 합·팩토리얼크기 n-1 한 개T(n)=T(n-1)+cn→n-1→…→1
피보나치·계단n-1, n-2 두 개T(n)=T(n-1)+T(n-2)+c두 갈래 호출 트리
이진 탐색·빠른 거듭제곱크기 n/2 한 개T(n)=T(n/2)+cn→n/2→n/4→…
병합 정렬크기 n/2 두 개T(n)=2T(n/2)+n이진 분해 후 병합
하노이 탑크기 n-1 두 번H(n)=2H(n-1)+1이동 문제가 두 번 반복
설명 예시
“병합 정렬은 입력 크기 n인 배열을 n/2 크기의 두 부분 배열로 나눈다. 두 부분 배열을 각각 정렬한 뒤 n개 원소를 비교하며 병합한다. 따라서 실행 시간은 두 개의 절반 크기 문제와 선형 병합 비용의 합인 T(n)=2T(n/2)+n으로 표현할 수 있다.”

마무리 진단 문제

1. 점화식은 무엇을 표현하는가?
정답 보기입력 크기 n인 문제와 더 작은 부분 문제 사이의 값 또는 실행 시간 관계를 표현합니다.
2. 이진 탐색에서 입력 크기는 재귀 호출마다 어떻게 변하는가?
정답 보기탐색 범위가 대략 절반으로 줄어듭니다.
3. 병합 정렬의 점화식에서 n 항은 무엇을 의미하는가?
정답 보기두 개의 정렬된 절반 배열을 하나로 병합하는 선형 작업량을 의미합니다.
4. 피보나치 단순 재귀에서 중복 계산이 발생하는 이유는 무엇인가?
정답 보기서로 다른 호출 가지에서 같은 입력 크기의 부분 문제가 반복해서 나타나기 때문입니다.
5. 하노이 탑의 이동 횟수 점화식은 무엇인가?
정답 보기H(1)=1, H(n)=2H(n-1)+1입니다.
