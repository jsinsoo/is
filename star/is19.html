동전 교환·회의실 배정
지역 최적 선택과 전체 최적해 예제 10선

매 단계에서 가장 좋아 보이는 선택을 하는 탐욕 알고리즘의 원리를 이해하고, 지역 최적 선택이 전체 최적해로 이어지는 조건과 그렇지 않은 반례를 코드로 탐구합니다.

핵심 원리1. 동전 교환2. 동전 반례 3. DP 비교4. 회의실 배정5. 정렬 기준 비교 6. 활동 선택7. 분할 배낭8. 작업 스케줄 9. 최소 연결10. 조건 종합

1. 탐욕 알고리즘의 핵심 조건

지역 최적 선택현재 단계에서 가장 좋아 보이는 후보를 즉시 선택합니다.
탐욕 선택 속성현재의 최선 선택이 전체 최적해 일부가 될 수 있어야 합니다.
최적 부분 구조선택 이후 남은 문제의 최적해가 전체 최적해를 구성해야 합니다.
반례 검증작은 입력을 완전 탐색·동적 계획법과 비교하여 실패 사례를 찾습니다.
탐욕 알고리즘은 선택을 되돌리지 않으므로 빠르고 간단하지만, 모든 문제에서 최적해를 보장하지는 않습니다. 정렬 기준과 선택 규칙이 왜 안전한지 교환 논증이나 문제 구조로 설명해야 합니다.
문제지역 최적 선택최적 보장 여부
표준 동전 체계가장 큰 동전부터 사용동전 체계에 따라 다름
회의실 배정가장 빨리 끝나는 회의 선택보장됨
분할 가능한 배낭단위 무게당 가치가 큰 물건 선택보장됨
0/1 배낭가치 비율이 큰 물건 선택보장되지 않음
예제 1

큰 동전부터 사용하는 탐욕적 동전 교환

동전 교환내림차순
coins = [500, 100, 50, 10]
amount = 1260

count = 0
used = []

for coin in coins:
    number = amount // coin

    if number > 0:
        used.append((coin, number))
        count += number
        amount %= coin

print("사용 동전:", used)
print("동전 수:", count)
print("남은 금액:", amount)

지역 선택

현재 남은 금액을 넘지 않는 가장 큰 동전을 가능한 많이 사용합니다.

결과

500원 2개, 100원 2개, 50원 1개, 10원 1개로 총 6개입니다.

조건
우리나라의 대표적인 동전 단위처럼 큰 동전과 작은 동전의 관계가 잘 구성된 경우에는 이 방법이 최적해를 만들 수 있습니다.
예제 2

탐욕적 동전 교환이 실패하는 반례

반례최적 미보장

탐욕 방식

coins = [4, 3, 1]
amount = 6

used = []

for coin in coins:
    number = amount // coin

    used.extend([coin] * number)
    amount %= coin

print("탐욕 결과:", used)
print("동전 수:", len(used))

더 좋은 해

optimal = [3, 3]

print("최적 결과:", optimal)
print("동전 수:", len(optimal))
실패 이유
가장 큰 4원을 먼저 선택하면 4+1+1로 3개가 필요하지만, 3+3을 선택하면 2개로 해결됩니다. 지역 최적 선택이 전체 최적해를 보장하지 않습니다.
예제 3

동적 계획법과 비교하여 동전 체계 검증

DP 비교최적성 검증
def greedy_coin_count(coins, amount):
    count = 0

    for coin in sorted(coins, reverse=True):
        count += amount // coin
        amount %= coin

    return count if amount == 0 else float("inf")

def dp_coin_count(coins, amount):
    dp = [float("inf")] * (amount + 1)
    dp[0] = 0

    for value in range(1, amount + 1):
        for coin in coins:
            if coin <= value:
                dp[value] = min(
                    dp[value],
                    dp[value - coin] + 1
                )

    return dp[amount]

coin_sets = [
    [500, 100, 50, 10],
    [4, 3, 1],
    [10, 6, 1]
]

for coins in coin_sets:
    for amount in range(1, 21):
        greedy = greedy_coin_count(coins, amount)
        optimal = dp_coin_count(coins, amount)

        if greedy != optimal:
            print(
                "반례:",
                coins,
                "금액:", amount,
                "탐욕:", greedy,
                "최적:", optimal
            )
            break
탐구 방법
작은 금액 범위에서 탐욕 결과와 동적 계획법의 최적 결과를 비교하면 해당 동전 체계의 반례를 찾을 수 있습니다.
예제 4

종료 시각 기준 회의실 배정

회의실 배정종료 시각 정렬
meetings = [
    (1, 4),
    (3, 5),
    (0, 6),
    (5, 7),
    (3, 9),
    (5, 9),
    (6, 10),
    (8, 11),
    (8, 12),
    (2, 14),
    (12, 16)
]

meetings.sort(
    key=lambda meeting: (
        meeting[1],
        meeting[0]
    )
)

selected = []
last_end = 0

for start, end in meetings:
    if start >= last_end:
        selected.append((start, end))
        last_end = end

print("선택 회의:", selected)
print("회의 수:", len(selected))
지역 최적 선택
현재 선택 가능한 회의 중 가장 빨리 끝나는 회의를 선택하면 이후 회의를 배정할 수 있는 시간이 가장 많이 남습니다.
예제 5

회의실 배정 정렬 기준 비교

시작 시각짧은 길이종료 시각
meetings = [
    (1, 10),
    (2, 3),
    (3, 4),
    (4, 5),
    (5, 6)
]

def select_meetings(meetings, key):
    ordered = sorted(meetings, key=key)

    selected = []
    last_end = -1

    for start, end in ordered:
        if start >= last_end:
            selected.append((start, end))
            last_end = end

    return selected

by_start = select_meetings(
    meetings,
    key=lambda item: item[0]
)

by_length = select_meetings(
    meetings,
    key=lambda item: item[1] - item[0]
)

by_end = select_meetings(
    meetings,
    key=lambda item: item[1]
)

print("시작 시각 기준:", by_start)
print("짧은 회의 기준:", by_length)
print("종료 시각 기준:", by_end)
핵심
가장 일찍 시작하는 회의를 선택하면 긴 회의 하나가 나머지를 막을 수 있습니다. 회의 개수를 최대로 하려면 종료 시각이 가장 빠른 회의를 선택해야 합니다.
예제 6

교환 논증으로 회의실 배정 최적성 이해

교환 논증최적성 근거
meetings = [
    ("A", 1, 4),
    ("B", 2, 5),
    ("C", 4, 7),
    ("D", 5, 9),
    ("E", 7, 8)
]

ordered = sorted(
    meetings,
    key=lambda x: x[2]
)

selected = []
last_end = -1

for name, start, end in ordered:
    if start >= last_end:
        selected.append((name, start, end))
        last_end = end

for meeting in selected:
    print(meeting)

논리적 설명

어떤 최적해의 첫 회의가 가장 빨리 끝나는 회의가 아니라고 가정합니다.

그 첫 회의를 더 빨리 끝나는 회의로 바꾸어도 이후 회의 선택 가능성은 줄지 않습니다.

따라서 가장 빨리 끝나는 회의를 포함하는 최적해가 존재합니다.

예제 7

분할 가능한 배낭: 가치 밀도 기준 선택

분할 배낭가치 밀도
items = [
    ("A", 10, 60),
    ("B", 20, 100),
    ("C", 30, 120)
]

capacity = 50

items.sort(
    key=lambda item: item[2] / item[1],
    reverse=True
)

total_value = 0
selected = []

for name, weight, value in items:
    if capacity == 0:
        break

    take = min(weight, capacity)
    ratio = take / weight

    total_value += value * ratio
    selected.append((name, take, value * ratio))
    capacity -= take

print("선택:", selected)
print("총 가치:", total_value)
최적 보장 조건
물건을 쪼갤 수 있으므로 단위 무게당 가치가 큰 부분부터 채우는 선택이 전체 최적해로 이어집니다.
예제 8

0/1 배낭에서 가치 밀도 탐욕법의 실패

0/1 배낭반례

탐욕 선택

items = [
    ("A", 10, 60),
    ("B", 20, 100),
    ("C", 30, 120)
]

capacity = 50

ordered = sorted(
    items,
    key=lambda x: x[2] / x[1],
    reverse=True
)

value = 0
chosen = []

for item in ordered:
    name, weight, item_value = item

    if weight <= capacity:
        capacity -= weight
        value += item_value
        chosen.append(name)

print(chosen, value)

최적 선택

# B와 C 선택
optimal_weight = 20 + 30
optimal_value = 100 + 120

print(
    ["B", "C"],
    optimal_weight,
    optimal_value
)
실패 이유
물건을 쪼갤 수 없으면 현재 가치 밀도가 높은 물건을 선택한 결과가 이후 조합을 제한할 수 있습니다. 0/1 배낭은 일반적으로 동적 계획법이 필요합니다.
예제 9

작업 마감시간과 최대 이익 스케줄링

작업 스케줄이익 내림차순
jobs = [
    ("A", 2, 100),
    ("B", 1, 19),
    ("C", 2, 27),
    ("D", 1, 25),
    ("E", 3, 15)
]

jobs.sort(
    key=lambda job: job[2],
    reverse=True
)

max_deadline = max(job[1] for job in jobs)
slots = [None] * max_deadline
total_profit = 0

for name, deadline, profit in jobs:
    for slot in range(
        min(deadline, max_deadline) - 1,
        -1,
        -1
    ):
        if slots[slot] is None:
            slots[slot] = name
            total_profit += profit
            break

print("작업 배치:", slots)
print("총 이익:", total_profit)
지역 선택
이익이 큰 작업부터 고려하되, 다른 작업의 기회를 최대한 남기기 위해 마감시간 안의 가장 늦은 빈 슬롯에 배치합니다.
예제 10

탐욕 선택의 성공 조건 종합 판별

종합 분석성공·실패 조건
def explain_greedy(problem):
    explanations = {
        "회의실 배정": {

    print("지역 선택:", result["선택"])

검토 질문설명
선택 규칙이 명확한가?가장 큰 동전, 가장 빠른 종료, 가장 높은 가치 밀도 등
선택을 교환해도 되는가?다른 최적해의 선택을 탐욕 선택으로 바꿔도 결과가 나빠지지 않는가?
남은 문제가 같은 형태인가?선택 이후 남은 부분도 원래 문제와 같은 최적화 문제인가?
반례가 존재하는가?작은 입력을 완전 탐색·DP와 비교했을 때 실패 사례가 있는가?
되돌리기가 필요한가?탐욕법은 선택을 되돌리지 않으므로 선택의 안전성이 반드시 필요함
결과 설명 예시
“회의실 배정에서는 현재 선택 가능한 회의 중 종료 시각이 가장 빠른 회의를 선택하였다. 이 선택은 이후 회의를 배치할 수 있는 시간을 가장 많이 남긴다. 어떤 최적해의 첫 회의를 이 회의로 교환해도 선택 가능한 회의 수가 줄지 않으므로 지역 최적 선택이 전체 최적해로 이어진다.”

마무리 진단 문제

1. 탐욕 알고리즘은 매 단계에서 어떤 선택을 하는가?
정답 보기현재 상태에서 가장 좋아 보이는 지역 최적 선택을 합니다.
2. 회의실 배정에서 최적 회의 수를 얻기 위한 정렬 기준은 무엇인가?
정답 보기종료 시각이 빠른 순서입니다.
3. 동전 [4, 3, 1]로 6을 만들 때 큰 동전 우선이 실패하는 이유는 무엇인가?
정답 보기4+1+1은 3개지만 3+3은 2개이므로 지역 최적 선택이 전체 최적해를 만들지 못합니다.
4. 분할 배낭에서 탐욕법이 최적해를 보장하는 이유는 무엇인가?
정답 보기물건을 나눌 수 있어 가치 밀도가 높은 부분부터 채우는 것이 항상 더 유리하기 때문입니다.
5. 탐욕 알고리즘의 최적성을 확인하는 대표적인 논증 방법은 무엇인가?
정답 보기탐욕 선택 속성과 최적 부분 구조를 확인하고, 교환 논증이나 반례 비교를 사용할 수 있습니다.
