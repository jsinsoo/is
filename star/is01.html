교과의 네 영역과 문제 해결 절차
함수 정의·호출·반환값 진단 실습 10선

디지털 문화, 데이터, 알고리즘과 프로그래밍, 인공지능의 네 영역을 문제 해결 절차와 연결하고, 파이썬 함수의 정의·호출·매개변수·반환값을 직접 실행하며 점검하는 수업 자료입니다.

학습 개요 1. 정의와 호출 2. 반환값 3. 매개변수 4. 여러 반환값 5. 입력 검증 6. 자료 분석 7. 정보 윤리 8. 인공지능 규칙 9. 오류 진단 10. 종합 문제

1. 교과의 네 영역

디지털 문화정보기술이 사회에 미치는 영향, 개인정보 보호, 저작권, 디지털 시민의식을 다룹니다.
데이터데이터를 수집·정리·분석·시각화하고 의미 있는 정보를 도출합니다.
알고리즘과 프로그래밍문제를 단계적으로 분석하고 알고리즘을 설계하여 프로그램으로 구현합니다.
인공지능데이터와 규칙을 활용해 분류·예측·추천하는 원리를 이해하고 윤리 문제를 탐구합니다.

2. 문제 해결 절차

① 문제 이해무엇을 해결할까?
② 데이터·조건 분석입력과 출력은 무엇일까?
③ 알고리즘 설계어떤 순서로 처리할까?
④ 구현과 실행함수로 표현해 보자.
⑤ 평가와 개선오류와 한계를 점검하자.
함수 진단 핵심
함수는 특정 작업을 하나의 이름으로 묶은 코드입니다. def로 정의하고, 함수 이름 뒤에 괄호를 붙여 호출합니다. return은 함수가 계산한 값을 호출한 위치로 돌려보내며, print()는 화면에 내용을 보여 주는 기능입니다.
예제 1

함수 정의와 호출 구별하기

def함수 호출순차 실행
def greet():
    print("안녕하세요.")

print("수업을 시작합니다.")
greet()
print("수업을 마칩니다.")

예상 출력

수업을 시작합니다.
안녕하세요.
수업을 마칩니다.
진단 질문
def greet():가 실행될 때 바로 “안녕하세요.”가 출력될까요?
구문설명
def greet():greet라는 함수를 정의합니다. 정의만으로 함수 내부가 실행되지는 않습니다.
greet()정의된 함수를 호출합니다. 이때 함수 내부의 print()가 실행됩니다.
핵심 함수는 먼저 정의하고, 필요한 시점에 호출하여 사용합니다.
예제 2

print와 return의 차이 진단하기

return출력값 저장
def add_print(a, b):
    print(a + b)

def add_return(a, b):
    return a + b

x = add_print(3, 4)
y = add_return(3, 4)

print("x =", x)
print("y =", y)

예상 출력

7
x = None
y = 7
주의
print()로 화면에 값을 보였다고 해서 그 값이 함수의 결과로 반환되는 것은 아닙니다.
핵심 print()는 화면 출력, return은 계산 결과 전달입니다. 반환값이 없는 함수는 기본적으로 None을 돌려줍니다.
예제 3

매개변수와 인수 점검하기

매개변수인수자료형
def introduce(name, subject):

message = introduce("민준", "정보")

"민준"과 "정보"는 함수를 호출할 때 전달한 인수입니다.

확인 활동

예제 4

여러 값을 반환하는 함수

튜플 반환언패킹데이터 영역
def analyze_scores(scores):
    total = sum(scores)
    average = total / len(scores)
    highest = max(scores)
    lowest = min(scores)
    return total, average, highest, lowest

scores = [85, 92, 78, 96, 89]
total, avg, high, low = analyze_scores(scores)

print("합계:", total)
print("평균:", avg)
print("최고:", high)
print("최저:", low)

문제 해결 연결

  1. 문제 이해: 점수의 특징을 요약한다.
  2. 데이터 분석: 합계·평균·최댓값·최솟값을 선택한다.
  3. 알고리즘: 네 값을 순서대로 계산한다.
  4. 함수 구현: 계산 결과를 한 번에 반환한다.
핵심 파이썬 함수는 쉼표로 구분하여 여러 값을 반환할 수 있으며, 호출할 때 여러 변수로 나누어 저장할 수 있습니다.
예제 5

입력값을 검증하는 함수

조건문조기 반환오류 예방
def safe_divide(a, b):
    if b == 0:
        return "0으로 나눌 수 없습니다."
    return a / b

print(safe_divide(10, 2))
print(safe_divide(10, 0))

예상 출력

5.0
0으로 나눌 수 없습니다.

잘못된 입력을 먼저 검사하여 함수 실행을 안전하게 종료합니다.

진단 질문
첫 번째 return이 실행되면 아래쪽의 return a / b도 실행될까요?
정답: 실행되지 않습니다. return은 함수 실행을 즉시 끝냅니다.
예제 6

데이터 영역: 미세먼지 상태 분류 함수

데이터 분류다중 조건문반환값
def classify_pm10(value):
    if value <= 30:
        return "좋음"
    elif value <= 80:
        return "보통"
    elif value <= 150:
        return "나쁨"
    else:
        return "매우 나쁨"

measurements = [22, 57, 103, 171]

for value in measurements:
    result = classify_pm10(value)
    print(value, "→", result)

실행 결과

22 → 좋음
57 → 보통
103 → 나쁨
171 → 매우 나쁨

하나의 분류 규칙을 함수로 만들면 여러 데이터에 반복 적용할 수 있습니다.

문제 해결 절차 측정값 입력 → 구간 비교 → 상태 반환 → 반복 적용 → 결과 확인
예제 7

디지털 문화 영역: 개인정보 가리기

개인정보 보호문자열 슬라이싱함수 재사용
def mask_name(name):
    if len(name) <= 1:
        return "*"
    return name[0] + "*" * (len(name) - 1)

students = ["김민준", "이서연", "박준"]

for student in students:
    print(mask_name(student))

실행 결과

김**
이**
박*

개인정보를 그대로 공개하지 않고 일부 문자를 가리는 간단한 비식별화 활동입니다.

윤리 점검
이름 일부를 가리는 것만으로 완전한 익명성이 보장되지는 않습니다. 다른 정보와 결합되면 개인을 다시 알아볼 수 있으므로 데이터 공개 범위를 함께 검토해야 합니다.
예제 8

인공지능 영역: 규칙 기반 추천 함수

인공지능 기초규칙 기반추천
def recommend_activity(temperature, rain):
    if rain:
        return "실내 코딩 활동"
    elif temperature >= 28:
        return "그늘에서 센서 관찰"
    elif temperature <= 5:
        return "실내 데이터 분석"
    else:
        return "야외 사물인터넷 실험"

print(recommend_activity(30, False))
print(recommend_activity(18, True))

진단

이 코드는 학습 데이터를 이용한 기계학습이 아니라, 사람이 만든 조건을 따르는 규칙 기반 시스템입니다.

그늘에서 센서 관찰
실내 코딩 활동
확장 활동
습도, 미세먼지, 학생 선호도 등의 조건을 추가하고 추천 결과가 어떻게 달라지는지 살펴봅니다.
예제 9

함수 오류를 찾아 수정하기

디버깅들여쓰기반환 위치

오류 코드

def count_even(numbers):
    count = 0
    for number in numbers:
        if number % 2 == 0:
            count += 1
        return count

print(count_even([2, 4, 5, 8]))

수정 코드

def count_even(numbers):
    count = 0
    for number in numbers:
        if number % 2 == 0:
            count += 1
    return count

print(count_even([2, 4, 5, 8]))
오류 원인
오류 코드의 return count가 반복문 안에 있어 첫 번째 숫자를 처리한 직후 함수가 종료됩니다.
수정 결과 짝수 2, 4, 8을 모두 확인하므로 3이 출력됩니다.
예제 10

종합 문제: 교실 환경 진단 프로그램

문제 분해함수 조합데이터알고리즘
def check_temperature(value):
    if 20 <= value <= 26:
        return "적정"
    return "조절 필요"

def check_co2(value):
    if value < 1000:
        return "양호"
    return "환기 필요"

def make_report(temperature, co2):
    temp_result = check_temperature(temperature)
    co2_result = check_co2(co2)

    return {
        "온도": temp_result,
        "이산화탄소": co2_result
    }

report = make_report(28, 1350)

for item, result in report.items():
    print(item, ":", result)

실행 결과

온도 : 조절 필요
이산화탄소 : 환기 필요

문제 해결 절차

  1. 문제: 교실 환경 상태를 진단한다.
  2. 분해: 온도와 이산화탄소를 따로 판단한다.
  3. 추상화: 각 판단을 함수로 만든다.
  4. 통합: 보고서 함수에서 결과를 묶는다.
  5. 평가: 기준값과 실제 환경의 차이를 점검한다.
