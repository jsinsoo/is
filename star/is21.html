1. 분할 정복의 세 단계

분할입력 크기 n의 문제를 더 작은 부분 문제로 나눕니다.
해결부분 문제가 충분히 작아질 때까지 재귀적으로 해결합니다.
결합부분 문제의 결과를 모아 전체 문제의 답을 만듭니다.
종료 조건더 이상 나눌 필요가 없는 가장 작은 입력을 정합니다.
알고리즘분할해결결합
병합 정렬배열을 왼쪽·오른쪽 절반으로 나눔각 절반을 재귀적으로 정렬두 정렬 배열을 하나로 병합
이진 탐색중간값을 기준으로 탐색 범위를 절반으로 나눔목표가 있을 수 있는 한쪽만 탐색별도 결합 없이 찾은 위치 반환
병합 정렬은 두 부분 문제를 모두 해결하고 결과를 합치는 대표적인 분할 정복입니다. 이진 탐색은 두 부분 중 목표가 존재할 수 있는 한쪽만 선택하므로 결합 단계가 거의 없는 분할 정복입니다.
예제 1

정렬된 두 리스트 병합하기

결합 단계투 포인터
def merge(left, right):
    result = []
    i = 0
    j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    result.extend(left[i:])
    result.extend(right[j:])

    return result

print(merge(
    [1, 4, 7],
    [2, 3, 8]
))

결합 과정

[1,4,7] + [2,3,8]

1과 2 비교 → 1
4와 2 비교 → 2
4와 3 비교 → 3
4와 8 비교 → 4
7과 8 비교 → 7
남은 8 추가

결과: [1,2,3,4,7,8]
예제 2

기본 병합 정렬 구현

분할재귀 해결병합
def merge_sort(data):
    # 종료 조건
    if len(data) <= 1:
        return data

    # 1. 분할
    middle = len(data) // 2
    left_data = data[:middle]
    right_data = data[middle:]

    # 2. 해결
    left_sorted = merge_sort(left_data)
    right_sorted = merge_sort(right_data)

    # 3. 결합
    return merge(left_sorted, right_sorted)

numbers = [7, 3, 8, 2, 5, 1]
print(merge_sort(numbers))
구조
배열 길이가 1 이하가 될 때까지 절반으로 나눈 뒤, 정렬된 작은 배열부터 순서대로 병합합니다.
예제 3

병합 정렬의 분할 과정 추적

호출 깊이분할 도식
def merge_sort_trace(data, depth=0):
    indent = "  " * depth
    print(f"{indent}분할 대상: {data}")

    if len(data) <= 1:
        print(f"{indent}기본 상태: {data}")
        return data

    middle = len(data) // 2

    left = merge_sort_trace(
        data[:middle],
        depth + 1
    )

    right = merge_sort_trace(
        data[middle:],
        depth + 1
    )

    result = merge(left, right)
    print(f"{indent}결합 결과: {result}")

    return result

merge_sort_trace([7, 3, 8, 2])
관찰
들여쓰기가 깊어질수록 입력이 더 작은 부분 배열로 나뉘고, 기본 상태에 도달한 뒤 역순으로 결합됩니다.
예제 4

병합 단계의 비교 횟수 측정

비교 횟수성능 분석
comparison_count = 0

def merge_count(left, right):
    global comparison_count

    result = []
    i = j = 0

    while i < len(left) and j < len(right):
        comparison_count += 1

        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    result.extend(left[i:])
    result.extend(right[j:])

    return result

def merge_sort_count(data):
    if len(data) <= 1:
        return data

    middle = len(data) // 2

    left = merge_sort_count(data[:middle])
    right = merge_sort_count(data[middle:])

    return merge_count(left, right)

numbers = [8, 7, 6, 5, 4, 3, 2, 1]

result = merge_sort_count(numbers)

print("정렬 결과:", result)
print("비교 횟수:", comparison_count)
시간 관계
각 단계의 병합은 전체 원소 수에 비례하고, 분할 깊이는 대략 log₂n 단계이므로 전체 시간은 일반적으로 O(n log n)입니다.
예제 5

인덱스 범위를 이용한 병합 정렬

제자리 수정보조 배열
def merge_range(data, left, middle, right):
    temp = []
    i = left
    j = middle + 1

    while i <= middle and j <= right:
        if data[i] <= data[j]:
            temp.append(data[i])
            i += 1
        else:
            temp.append(data[j])
            j += 1

    temp.extend(data[i:middle + 1])
    temp.extend(data[j:right + 1])

    data[left:right + 1] = temp

def merge_sort_range(data, left, right):
    if left >= right:
        return

    middle = (left + right) // 2

    merge_sort_range(data, left, middle)
    merge_sort_range(data, middle + 1, right)

    merge_range(data, left, middle, right)

numbers = [9, 1, 6, 3, 8, 2]
merge_sort_range(numbers, 0, len(numbers) - 1)

print(numbers)
공간 관점
원본 리스트를 직접 수정하지만 병합을 위해 보조 리스트가 필요합니다. 병합 정렬은 일반적으로 O(n)의 추가 공간을 사용합니다.
예제 6

반복문으로 이진 탐색 구현

탐색 범위 절반반복 방식
def binary_search(data, target):
    left = 0
    right = len(data) - 1

    while left <= right:
        middle = (left + right) // 2

        if data[middle] == target:
            return middle

        if data[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

    return -1

numbers = [2, 5, 8, 12, 16, 23, 38]
print(binary_search(numbers, 16))

분할 구조

중간값과 목표값을 비교합니다.

목표가 크면 오른쪽 절반, 작으면 왼쪽 절반만 남깁니다.

정렬된 자료에서만 사용할 수 있습니다.

예제 7

재귀함수로 이진 탐색 구현

재귀 탐색종료 조건
def binary_search_recursive(
    data,
    target,
    left,
    right
):
    if left > right:
        return -1

    middle = (left + right) // 2

    if data[middle] == target:
        return middle

    if data[middle] < target:
        return binary_search_recursive(
            data,
            target,
            middle + 1,
            right
        )

    return binary_search_recursive(
        data,
        target,
        left,
        middle - 1
    )

numbers = [2, 5, 8, 12, 16, 23, 38]

print(binary_search_recursive(
    numbers,
    23,
    0,
    len(numbers) - 1
))
종료 조건
left > right이면 탐색 범위가 사라진 것이므로 목표값이 없다고 판단합니다.
예제 8

이진 탐색 범위 변화 추적

범위 추적입력 크기 변화
def binary_search_trace(data, target):
    left = 0
    right = len(data) - 1
    step = 1

    while left <= right:
        middle = (left + right) // 2

        print(
            f"{step}단계 | "
            f"범위={left}~{right} | "
            f"중간={middle} | "
            f"값={data[middle]}"
        )

        if data[middle] == target:
            return middle

        if data[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

        step += 1

    return -1

numbers = list(range(1, 33))
print("위치:", binary_search_trace(numbers, 29))
도식
32개 → 약 16개 → 약 8개 → 약 4개 → 약 2개 → 1개처럼 탐색 범위가 절반씩 줄어듭니다.
예제 9

중복값의 첫 번째 위치 찾기

변형 이진 탐색경계 탐색
def find_first(data, target):
    left = 0
    right = len(data) - 1
    answer = -1

    while left <= right:
        middle = (left + right) // 2

        if data[middle] == target:
            answer = middle
            right = middle - 1

        elif data[middle] < target:
            left = middle + 1

        else:
            right = middle - 1

    return answer

numbers = [1, 2, 2, 2, 3, 4, 5]

print("첫 위치:", find_first(numbers, 2))
해결 전략
목표값을 찾은 뒤 즉시 종료하지 않고, 더 앞쪽에 같은 값이 있는지 확인하기 위해 오른쪽 경계를 줄입니다.

병합 정렬과 이진 탐색 종합 프로그램

정렬 후 탐색종합 구현

def merge_sort(data):
    if len(data) <= 1:
        return data

    middle = len(data) // 2

    left = merge_sort(data[:middle])
    right = merge_sort(data[middle:])

def binary_search(data, target):

    right = len(data) - 1

raw_data = [23, 5, 17, 9, 31, 12, 4]
target = 17

sorted_data = merge_sort(raw_data)
position = binary_search(sorted_data, target)

print("원본:", raw_data)
print("정렬:", sorted_data)
print("목표값:", target)
print("위치:", position)
비교 항목병합 정렬이진 탐색
분할왼쪽·오른쪽 절반 모두 생성중간값 기준 한쪽 범위만 선택
해결두 절반을 각각 정렬선택한 절반에서 탐색 반복
결합정렬된 두 절반을 병합찾은 위치를 그대로 반환
시간 복잡도O(n log n)O(log n)
필수 조건정렬 전 자료에 사용 가능자료가 정렬되어 있어야 함
추가 공간병합을 위한 보조 공간 필요반복 방식은 추가 공간이 매우 적음
알고리즘 설명 예시
“병합 정렬은 배열을 절반 크기의 두 부분 배열로 분할하고, 각 부분을 재귀적으로 정렬한 뒤 두 결과를 크기순으로 병합하였다. 이진 탐색은 정렬된 배열의 중간값을 기준으로 목표가 존재할 수 있는 한쪽 절반만 남겨 탐색 범위를 반복해서 줄였다.”

마무리 진단 문제

1. 병합 정렬의 결합 단계에서는 무엇을 하는가?
정답 보기정렬된 두 부분 배열을 비교하면서 하나의 정렬된 배열로 합칩니다.
2. 이진 탐색을 사용하기 위한 필수 조건은 무엇인가?
정답 보기탐색할 자료가 정렬되어 있어야 합니다.
3. 병합 정렬에서 배열 길이가 1 이하일 때 재귀 호출을 멈추는 이유는 무엇인가?
정답 보기원소가 하나 이하인 배열은 이미 정렬된 상태이기 때문입니다.
4. 이진 탐색의 탐색 범위는 매 단계마다 어떻게 변하는가?
정답 보기중간값 비교 결과에 따라 대략 절반으로 줄어듭니다.
5. 병합 정렬과 이진 탐색의 결합 단계가 다른 점은 무엇인가?
정답 보기병합 정렬은 두 부분 결과를 실제로 합치지만, 이진 탐색은 한쪽 부분만 탐색하고 찾은 위치를 바로 반환합니다.
