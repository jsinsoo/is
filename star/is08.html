학교생활과 정보 시스템의 실제 사례를 통해 자료구조가 필요한 이유를 탐색하고, 리스트·스택·큐·덱·연결 리스트 등 선형 자료구조의 삽입·삭제·탐색 연산을 비교합니다.

1. 자료구조가 필요한 이유

자료 저장여러 값을 일정한 규칙에 따라 모아 관리합니다.
빠른 처리문제에 맞는 구조를 선택하여 삽입·삭제·탐색 시간을 줄입니다.
관계 표현자료의 순서, 연결, 우선순위와 같은 관계를 나타냅니다.
재사용자료 처리 방법을 구조화하여 다양한 프로그램에 적용합니다.
같은 자료라도 어떤 연산을 자주 수행하느냐에 따라 적합한 자료구조가 달라집니다. 중간 삽입과 삭제가 잦은지, 맨 앞과 뒤의 처리만 필요한지, 특정 값을 자주 찾는지 먼저 분석해야 합니다.
구조대표 특징적합한 사례
리스트인덱스로 순서 있는 자료 관리학생 명단, 점수 목록
스택나중에 들어온 자료를 먼저 처리실행 취소, 함수 호출
큐먼저 들어온 자료를 먼저 처리대기표, 인쇄 작업
연결 리스트노드를 연결하여 자료 관리중간 삽입·삭제가 많은 목록

예제 2

리스트의 위치별 삽입 비교

appendinsert
students = ["민준", "서연", "지우"]

students.append("하준")       # 맨 뒤 삽입
print(students)

students.insert(0, "예린")    # 맨 앞 삽입

연산특징일반적 비용

insert(i, x)중간 위치에 자료 추가뒤쪽 원소 이동 필요
예제 3

리스트의 삭제 연산 비교

popremovedel
numbers = [10, 20, 30, 40, 50]

last = numbers.pop()
print("마지막 삭제:", last)

first = numbers.pop(0)
print("첫 번째 삭제:", first)

numbers.remove(30)
print("값 30 삭제:", numbers)

삭제 방식

pop(): 위치를 기준으로 삭제하며 삭제값을 반환합니다.

remove(value): 처음 발견한 해당 값을 삭제합니다.

del numbers[i]: 인덱스 위치를 직접 삭제합니다.

주의
없는 값을 remove()로 삭제하면 오류가 발생합니다. 삭제 전에 포함 여부를 확인할 수 있습니다.
예제 4

선형 탐색 구현과 과정 확인

탐색순차 비교
def linear_search(data, target):
    for index, value in enumerate(data):
        print(index, "번째 확인:", value)

        if value == target:
            return index

    return -1

scores = [72, 85, 91, 68, 88]
result = linear_search(scores, 91)

print("찾은 위치:", result)

탐색 원리

첫 원소부터 목표값과 하나씩 비교합니다.

정렬 여부와 관계없이 사용할 수 있지만, 자료가 많으면 비교 횟수가 늘어납니다.

최악의 경우
목표값이 마지막에 있거나 목록에 없으면 모든 원소를 확인해야 합니다.
예제 5

스택으로 실행 취소 기능 구현

LIFOpush·pop
undo_stack = []

def do_action(action):
    undo_stack.append(action)
    print("실행:", action)

def undo():
    if not undo_stack:
        print("취소할 작업이 없습니다.")
        return

    action = undo_stack.pop()
    print("취소:", action)

do_action("문장 입력")
do_action("글자색 변경")
do_action("사진 삽입")

undo()
undo()
연산 비교
스택은 맨 뒤 삽입 append()와 맨 뒤 삭제 pop()를 사용하므로 효율적입니다. 중간 자료 탐색에는 적합하지 않습니다.
예제 6

큐로 급식 대기 순서 관리

FIFOdeque
from collections import deque

waiting = deque()

waiting.append("민준")
waiting.append("서연")
waiting.append("지우")

while waiting:
    student = waiting.popleft()
    print("급식 입장:", student)

큐 연산

append(): 대기열 뒤에 추가합니다.

popleft(): 대기열 앞에서 제거합니다.

리스트의 pop(0)보다 앞쪽 삭제에 적합합니다.

예제 7

덱의 양쪽 삽입·삭제

양방향 큐appendleft
from collections import deque

tasks = deque(["과제2", "과제3"])

tasks.appendleft("긴급 과제")
tasks.append("추가 과제")

print("왼쪽 처리:", tasks.popleft())
print("오른쪽 처리:", tasks.pop())
print("남은 작업:", list(tasks))
특징
덱은 양쪽 끝의 삽입과 삭제가 모두 빠릅니다. 앞과 뒤를 모두 자주 사용하는 문제에 적합합니다.
예제 8

단순 연결 리스트의 삽입·탐색

노드연결 구조
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def insert_front(self, data):
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node

    def search(self, target):
        current = self.head

        while current:
            if current.data == target:
                return True
            current = current.next

        return False

linked = LinkedList()
linked.insert_front("지우")
linked.insert_front("서연")
linked.insert_front("민준")

print(linked.search("서연"))
print(linked.search("하준"))
비교
연결 리스트는 맨 앞 삽입이 간단하지만 특정 위치로 바로 이동할 수 없어 탐색은 처음부터 연결을 따라가야 합니다.
예제 9

리스트 앞·뒤 삽입 실행 시간 비교

성능 측정perf_counter
import time

count = 20000

data1 = []
start = time.perf_counter()

for value in range(count):
    data1.append(value)

append_time = time.perf_counter() - start

data2 = []
start = time.perf_counter()

for value in range(count):
    data2.insert(0, value)

front_time = time.perf_counter() - start

print(f"뒤 삽입: {append_time:.6f}초")
print(f"앞 삽입: {front_time:.6f}초")
해석
앞 삽입은 기존 원소를 뒤로 이동시켜야 하므로 자료가 많을수록 시간이 증가합니다. 측정값은 실행 환경에 따라 달라집니다.

정답 보기어떤 자료를 저장하고 삽입·삭제·탐색 중 어떤 연산을 자주 수행하는지 분석해야 합니다.
