1. 백트래킹의 네 단계

① 선택현재 위치에 놓을 후보를 하나 결정합니다.
② 재귀선택이 반영된 다음 상태를 같은 함수로 해결합니다.
③ 목표·충돌 검사완성 상태인지 또는 더 진행할 수 없는지 판단합니다.
④ 되돌리기선택을 취소하여 이전 상태로 돌아가 다른 후보를 시도합니다.
백트래킹은 완전 탐색의 한 형태입니다. 모든 후보를 고려하지만, 이미 조건을 위반한 상태는 더 깊이 탐색하지 않는 가지치기를 적용합니다.
문제선택목표 상태되돌리기
순열아직 사용하지 않은 원소모든 자리 결정마지막 원소 제거·사용 해제
조합시작 인덱스 이후 원소r개 선택마지막 원소 제거
N-Queen현재 행의 안전한 열N개 퀸 배치열·대각선 사용 표시 해제
예제 1

선택·재귀·되돌리기의 최소 구조

백트래킹 틀append·pop
items = ["A", "B"]
selected = []

def search(depth):
    if depth == len(items):
        print("완성:", selected)
        return

    for item in items:
        if item in selected:
            continue

        # 1. 선택
        selected.append(item)
        print("선택:", selected)

        # 2. 재귀
        search(depth + 1)

        # 3. 되돌리기
        removed = selected.pop()
        print("복원:", removed, selected)

search(0)

핵심 코드

selected.append(item)  # 선택
search(depth + 1)      # 재귀
selected.pop()         # 되돌리기

세 줄이 백트래킹의 기본 구조입니다.

예제 2

방문 배열을 이용한 순열 생성

순열visited
items = ["A", "B", "C"]
selected = []
visited = [False] * len(items)

def permutation():
    if len(selected) == len(items):
        print(selected)
        return

    for index in range(len(items)):
        if visited[index]:
            continue

        visited[index] = True
        selected.append(items[index])

        permutation()

        selected.pop()
        visited[index] = False

permutation()
되돌리기
재귀 호출이 끝나면 선택 목록에서 원소를 제거하고 방문 표시를 False로 되돌려 다른 순열에서 다시 사용할 수 있게 합니다.
예제 3

순열 재귀 과정 들여쓰기 추적

호출 깊이추적 출력
items = [1, 2, 3]
selected = []
used = [False] * len(items)

def permutation_trace(depth=0):
    indent = "  " * depth

    if depth == len(items):
        print(indent, "완성:", selected)
        return

    for index, item in enumerate(items):
        if used[index]:
            continue

        print(indent, "선택 후보:", item)

        used[index] = True
        selected.append(item)

        permutation_trace(depth + 1)

        removed = selected.pop()
        used[index] = False

        print(indent, "되돌리기:", removed)

permutation_trace()
관찰
들여쓰기가 깊어질수록 더 많은 자리가 결정된 상태이며, 완성 후에는 가장 최근 선택부터 역순으로 취소됩니다.
예제 4

중복 원소가 있는 순열의 중복 방지

중복 순열같은 깊이 집합
items = ["A", "A", "B"]
selected = []
used = [False] * len(items)
results = []

def unique_permutation():
    if len(selected) == len(items):
        results.append(selected.copy())
        return

    used_this_depth = set()

    for index, item in enumerate(items):
        if used[index]:
            continue

        if item in used_this_depth:
            continue

        used_this_depth.add(item)
        used[index] = True
        selected.append(item)

        unique_permutation()

        selected.pop()
        used[index] = False

unique_permutation()

for result in results:
    print(result)
중복 방지 원리
같은 재귀 깊이에서 동일한 값은 한 번만 선택하여 결과가 같은 순열을 반복 생성하지 않습니다.
예제 5

시작 인덱스를 이용한 조합 생성

조합start
items = ["A", "B", "C", "D"]
r = 2
selected = []

def combination(start):
    if len(selected) == r:
        print(selected)
        return

    for index in range(start, len(items)):
        selected.append(items[index])

        combination(index + 1)

        selected.pop()

combination(0)

순열과 차이

순열은 아직 사용하지 않은 모든 원소를 후보로 봅니다.

조합은 선택 순서를 구분하지 않으므로 현재 위치보다 뒤의 원소만 선택합니다.

예제 6

목표 합을 만드는 조합과 가지치기

조합 합가지치기
numbers = [1, 2, 3, 4, 5]
target = 7
selected = []
solutions = []

def combination_sum(start, total):
    if total == target:
        solutions.append(selected.copy())
        return

    if total > target:
        return

    for index in range(start, len(numbers)):
        number = numbers[index]

        selected.append(number)
        combination_sum(
            index + 1,
            total + number
        )
        selected.pop()

combination_sum(0, 0)

for solution in solutions:
    print(solution)
가지치기
누적 합이 목표값을 넘으면 뒤의 수를 더 선택해도 해가 될 수 없으므로 재귀 호출을 중단합니다.
예제 7

N-Queen의 안전한 열 검사

N-Queen충돌 검사
n = 4

# queens[row] = col
queens = [-1] * n

def is_safe(row, col):
    for previous_row in range(row):
        previous_col = queens[previous_row]

        # 같은 열
        if previous_col == col:
            return False

        # 같은 대각선
        if abs(row - previous_row) == abs(
            col - previous_col
        ):
            return False

    return True

queens[0] = 1

for col in range(n):
    print(
        "1행", col, "열:",
        is_safe(1, col)
    )

상태 표현

queens[row] = col은 해당 행의 퀸 열 위치를 의미합니다.

행마다 퀸을 하나씩 놓으므로 행 충돌은 자동으로 방지됩니다.

예제 8

4-Queen의 모든 해 출력

4-Queen보드 출력
n = 4
queens = [-1] * n
solutions = []

def is_safe(row, col):
    for previous_row in range(row):
        previous_col = queens[previous_row]

        if previous_col == col:
            return False

        if abs(row - previous_row) == abs(
            col - previous_col
        ):
            return False

    return True

def solve(row):
    if row == n:
        solutions.append(queens.copy())
        return

    for col in range(n):
        if not is_safe(row, col):
            continue

        # 선택
        queens[row] = col

        # 재귀
        solve(row + 1)

        # 되돌리기
        queens[row] = -1

solve(0)

for solution in solutions:
    print("해:", solution)

    for row in range(n):
        line = [
            "Q" if solution[row] == col else "."
            for col in range(n)
        ]
        print(" ".join(line))

    print()
목표 상태
row == n이면 모든 행에 충돌 없이 퀸을 배치한 것이므로 하나의 해를 기록합니다.
예제 9

열·대각선 집합을 이용한 N-Queen 최적화

집합빠른 충돌 확인
def count_n_queens(n):
    columns = set()
    diagonal1 = set()  # row - col
    diagonal2 = set()  # row + col
    count = 0

    def solve(row):
        nonlocal count

        if row == n:
            count += 1
            return

        for col in range(n):
            d1 = row - col
            d2 = row + col

            if (
                col in columns
                or d1 in diagonal1
                or d2 in diagonal2
            ):
                continue

            # 선택
            columns.add(col)
            diagonal1.add(d1)
            diagonal2.add(d2)

            # 재귀
            solve(row + 1)

            # 되돌리기
            columns.remove(col)
            diagonal1.remove(d1)
            diagonal2.remove(d2)

    solve(0)
    return count

for n in range(1, 9):
    print(n, "Queen 해의 수:", count_n_queens(n))
효율성
이전 모든 퀸을 반복해서 확인하는 대신, 사용 중인 열과 두 대각선을 집합에서 빠르게 검사합니다.

마무리 진단 문제

1. 백트래킹의 핵심 세 단계는 무엇인가?
정답 보기후보 선택, 재귀 호출, 선택 되돌리기입니다.
2. 순열에서 방문 표시를 재귀 호출 후 해제하는 이유는 무엇인가?
정답 보기다른 순열을 만들 때 해당 원소를 다시 선택할 수 있도록 이전 상태로 복원하기 위해서입니다.
3. 조합에서 다음 재귀 호출에 index + 1을 전달하는 이유는 무엇인가?
정답 보기앞에서 선택한 원소를 다시 선택하지 않고 순서만 다른 중복 조합을 막기 위해서입니다.
4. N-Queen에서 검사해야 하는 세 가지 충돌은 무엇인가?
정답 보기같은 행, 같은 열, 같은 대각선 충돌입니다. 행별로 하나씩 배치하면 행 충돌은 자동으로 방지됩니다.
5. N-Queen에서 집합을 이용하면 어떤 점이 개선되는가?
정답 보기사용 중인 열과 대각선의 포함 여부를 빠르게 확인하여 충돌 검사 시간을 줄일 수 있습니다.
