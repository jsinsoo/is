동일 문제의 여러 알고리즘 구현
이론 복잡도·실제 실행 시간 비교 예제 10선

같은 문제를 서로 다른 알고리즘으로 해결하고, 시간 복잡도와 핵심 연산 횟수, 실제 실행 시간을 함께 측정하여 이론과 실험 결과가 달라지는 이유를 해석합니다.

측정 원리1. 합계 계산2. 탐색 3. 중복 검사4. 정렬5. 피보나치 6. 거듭제곱7. 최대값8. 두 수 합 9. 문자열 포함10. 종합 벤치마크

1. 이론 복잡도와 실제 시간 비교 방법

정확성 확인서로 다른 알고리즘이 같은 결과를 만드는지 먼저 검사합니다.
이론 분석입력 크기 n에 따른 핵심 연산 수를 빅오로 표현합니다.
실제 측정time.perf_counter()로 여러 번 반복 실행해 시간을 잽니다.
차이 해석상수 비용, 파이썬 내부 최적화, 입력 특성, 측정 오차를 함께 설명합니다.
작은 입력에서는 복잡도가 더 좋은 알고리즘이 오히려 느릴 수 있습니다. 함수 호출, 재귀, 자료구조 생성, 정렬 준비 같은 고정 비용이 상대적으로 크게 작용하기 때문입니다. 입력이 커질수록 성장률의 차이가 실제 시간에 더 뚜렷하게 나타납니다.
주의 요소설명
반복 측정매우 짧은 코드는 여러 번 실행한 총시간을 비교해야 함
입력 동일성모든 알고리즘에 같은 데이터 또는 같은 분포를 제공
내장 함수파이썬 내장 함수는 C로 구현되어 순수 파이썬 코드보다 매우 빠를 수 있음
최선·평균·최악입력 위치나 정렬 상태에 따라 실제 시간이 달라짐
메모리 비용빠른 알고리즘이 집합·테이블 등 추가 공간을 사용할 수 있음
예제 1

1부터 n까지 합: 반복문과 공식

O(n)O(1)
import time

def sum_loop(n):
    total = 0

    for value in range(1, n + 1):
        total += value

    return total

def sum_formula(n):
    return n * (n + 1) // 2

n = 1_000_000

start = time.perf_counter()
result1 = sum_loop(n)
time1 = time.perf_counter() - start

start = time.perf_counter()
result2 = sum_formula(n)
time2 = time.perf_counter() - start

print("결과 일치:", result1 == result2)
print(f"반복문 O(n): {time1:.8f}초")
print(f"공식 O(1):   {time2:.8f}초")
해석
공식은 입력 크기와 관계없이 일정한 수의 산술 연산만 수행합니다. 큰 n에서는 반복문과의 차이가 매우 커집니다.
예제 2

정렬된 자료 탐색: 선형 탐색과 이진 탐색

O(n)O(log n)
def linear_search(data, target):
    for index, value in enumerate(data):
        if value == target:
            return index
    return -1

def binary_search(data, target):
    left = 0
    right = len(data) - 1

    while left <= right:
        middle = (left + right) // 2

        if data[middle] == target:
            return middle

        if data[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

    return -1
import time

data = list(range(1_000_000))
target = data[-1]

start = time.perf_counter()
a = linear_search(data, target)
linear_time = time.perf_counter() - start

start = time.perf_counter()
b = binary_search(data, target)
binary_time = time.perf_counter() - start

print(a == b)
print("선형:", linear_time)
print("이진:", binary_time)
조건
이진 탐색은 정렬된 자료에서만 사용할 수 있습니다. 정렬 비용까지 포함해야 하는 일회성 탐색에서는 선형 탐색이 더 나을 수도 있습니다.
예제 3

중복 확인: 이중 반복문과 집합

O(n²)평균 O(n)
import time
import random

def has_duplicate_nested(data):
    for i in range(len(data)):
        for j in range(i + 1, len(data)):
            if data[i] == data[j]:
                return True
    return False

def has_duplicate_set(data):
    seen = set()

    for value in data:
        if value in seen:
            return True
        seen.add(value)

    return False

data = list(range(5000))
random.shuffle(data)

start = time.perf_counter()
r1 = has_duplicate_nested(data)
t1 = time.perf_counter() - start

start = time.perf_counter()
r2 = has_duplicate_set(data)
t2 = time.perf_counter() - start

print("결과 일치:", r1 == r2)
print("이중 반복:", t1)
print("집합:", t2)
시간·공간 절충
집합 방식은 평균적으로 빠르지만 최대 n개의 값을 추가 저장합니다. 이중 반복문은 추가 공간이 적지만 비교 횟수가 급격히 증가합니다.
예제 4

정렬: 버블 정렬과 파이썬 내장 정렬

O(n²)O(n log n)
def bubble_sort(data):
    result = data.copy()
    n = len(result)

    for end in range(n - 1, 0, -1):
        swapped = False

        for i in range(end):
            if result[i] > result[i + 1]:
                result[i], result[i + 1] = (
                    result[i + 1],
                    result[i]
                )
                swapped = True

        if not swapped:
            break

    return result
import random
import time

data = [
    random.randint(0, 100000)
    for _ in range(3000)
]

start = time.perf_counter()
a = bubble_sort(data)
bubble_time = time.perf_counter() - start

start = time.perf_counter()
b = sorted(data)
built_in_time = time.perf_counter() - start

print("결과 일치:", a == b)
print("버블:", bubble_time)
print("내장:", built_in_time)
해석
내장 정렬은 O(n log n) 계열의 팀소트이며 저수준 언어로 최적화되어 있습니다. 알고리즘 차이와 구현 언어 차이가 함께 실제 시간에 반영됩니다.
예제 5

피보나치: 단순 재귀·메모이제이션·반복

지수 시간O(n)
import time

def fib_recursive(n):
    if n < 2:
        return n

    return (
        fib_recursive(n - 1)
        + fib_recursive(n - 2)
    )

def fib_memo(n, memo={0: 0, 1: 1}):
    if n in memo:
        return memo[n]

    memo[n] = (
        fib_memo(n - 1)
        + fib_memo(n - 2)
    )
    return memo[n]

def fib_loop(n):
    a, b = 0, 1

    for _ in range(n):
        a, b = b, a + b

    return a

n = 35

for name, function in [
    ("단순 재귀", fib_recursive),
    ("메모", fib_memo),
    ("반복", fib_loop)
]:
    start = time.perf_counter()
    result = function(n)
    elapsed = time.perf_counter() - start

    print(name, result, elapsed)
실제 차이
메모이제이션과 반복 방식은 모두 O(n)이지만, 반복 방식은 재귀 호출과 딕셔너리 조회 비용이 없어 일반적으로 더 빠릅니다.
예제 6

거듭제곱: 반복 곱셈과 빠른 거듭제곱

O(n)O(log n)
import time

def power_linear(base, exponent):
    result = 1

    for _ in range(exponent):
        result *= base

    return result

def power_fast(base, exponent):
    result = 1

    while exponent > 0:
        if exponent % 2 == 1:
            result *= base

        base *= base
        exponent //= 2

    return result

base = 2
exponent = 20000

start = time.perf_counter()
a = power_linear(base, exponent)
t1 = time.perf_counter() - start

start = time.perf_counter()
b = power_fast(base, exponent)
t2 = time.perf_counter() - start

print("결과 일치:", a == b)
print("선형:", t1)
print("빠른 방식:", t2)
주의
큰 정수의 곱셈 비용은 숫자의 자릿수에 따라 증가합니다. 따라서 단순히 곱셈 횟수만으로 실제 시간을 완전히 설명할 수는 없습니다.
예제 7

최대값: 직접 순회와 정렬 후 선택

O(n)O(n log n)
def maximum_scan(data):
    maximum = data[0]

    for value in data[1:]:
        if value > maximum:
            maximum = value

    return maximum

def maximum_sort(data):
    return sorted(data)[-1]
import random
import time

data = [
    random.randint(0, 10_000_000)
    for _ in range(500_000)
]

start = time.perf_counter()
a = maximum_scan(data)
t1 = time.perf_counter() - start

start = time.perf_counter()
b = maximum_sort(data)
t2 = time.perf_counter() - start

print(a == b)
print("순회:", t1)
print("정렬:", t2)
알고리즘 선택
최대값 하나만 필요하다면 전체 순회가 적합합니다. 정렬은 필요 이상의 정보를 계산하므로 이론적으로도 실제로도 불리합니다.
예제 8

두 수의 합: 모든 쌍과 집합 탐색

O(n²)평균 O(n)
import time

def two_sum_nested(data, target):
    for i in range(len(data)):
        for j in range(i + 1, len(data)):
            if data[i] + data[j] == target:
                return i, j
    return None

def two_sum_set(data, target):
    positions = {}

    for index, value in enumerate(data):
        needed = target - value

        if needed in positions:
            return positions[needed], index

        positions[value] = index

    return None

data = list(range(10000))
target = 19997

start = time.perf_counter()
a = two_sum_nested(data, target)
t1 = time.perf_counter() - start

start = time.perf_counter()
b = two_sum_set(data, target)
t2 = time.perf_counter() - start

print(a, b)
print("모든 쌍:", t1)
print("딕셔너리:", t2)
입력 위치 영향
정답이 초반에 있으면 이중 반복문도 빠르게 끝날 수 있습니다. 최악 시간 복잡도와 한 번의 실제 측정 결과는 구분해야 합니다.
예제 9

문자열 포함 검사: 직접 구현과 in 연산자

순수 파이썬내부 최적화
import time

def contains_manual(text, pattern):
    m = len(pattern)

    for start in range(len(text) - m + 1):
        matched = True

        for offset in range(m):
            if text[start + offset] != pattern[offset]:
                matched = False
                break

        if matched:
            return True

    return False

text = "A" * 200000 + "PYTHON"
pattern = "PYTHON"

start = time.perf_counter()
a = contains_manual(text, pattern)
t1 = time.perf_counter() - start

start = time.perf_counter()
b = pattern in text
t2 = time.perf_counter() - start

print(a == b)
print("직접 구현:", t1)
print("in 연산자:", t2)
이론만으로 부족한 이유
두 방식 모두 입력을 탐색하지만, 내장 연산은 C 수준에서 최적화된 알고리즘과 빠른 메모리 처리를 사용하여 실제 시간이 훨씬 짧을 수 있습니다.
예제 10

종합 벤치마크 함수와 결과 해석

반복 측정중앙값종합 실습
import time
from statistics import median

def benchmark(function, *args, repeat=7):
    times = []
    result = None

    for _ in range(repeat):
        start = time.perf_counter()
        result = function(*args)
        elapsed = time.perf_counter() - start
        times.append(elapsed)

    return result, {
        "최소": min(times),
        "중앙값": median(times),
        "최대": max(times)
    }

def linear_search(data, target):
    for index, value in enumerate(data):
        if value == target:
            return index
    return -1

def binary_search(data, target):
    left = 0
    right = len(data) - 1

    while left <= right:
        middle = (left + right) // 2

        if data[middle] == target:
            return middle

        if data[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

    return -1

data = list(range(1_000_000))
target = data[-1]

r1, stats1 = benchmark(
    linear_search,
    data,
    target
)

r2, stats2 = benchmark(
    binary_search,
    data,
    target
)

print("결과 일치:", r1 == r2)
print("선형 탐색:", stats1)
print("이진 탐색:", stats2)
해석 항목확인할 내용
결과 일치빠른 알고리즘이 같은 정답을 만드는가?
이론 성장률n이 커질수록 연산 수가 어떤 비율로 증가하는가?
상수 비용함수 호출, 객체 생성, 해시 계산 등의 고정 비용은 얼마인가?
입력 특성정렬 상태, 정답 위치, 중복 정도가 시간에 영향을 주는가?
측정 변동운영체제 작업과 캐시로 실행 시간이 달라지는가?
메모리 절충시간을 줄이기 위해 추가 저장 공간을 사용하는가?
결과 해석 예시
“선형 탐색은 최악의 경우 n개의 원소를 확인하므로 O(n)이고, 이진 탐색은 범위를 절반씩 줄이므로 O(log n)이다. 작은 입력에서는 함수 구조와 반복문의 고정 비용 때문에 차이가 작게 나타날 수 있으나, 입력이 커질수록 이진 탐색의 비교 횟수가 훨씬 천천히 증가한다. 다만 이진 탐색을 사용하려면 자료가 정렬되어 있어야 하므로 정렬 비용과 탐색 반복 횟수를 함께 고려해야 한다.”

마무리 진단 문제

1. 서로 다른 알고리즘의 시간을 비교하기 전에 먼저 확인해야 하는 것은 무엇인가?
정답 보기모든 알고리즘이 동일한 입력에서 같은 결과를 만드는지 확인해야 합니다.
2. 복잡도가 더 좋은 알고리즘이 작은 입력에서 느릴 수 있는 이유는 무엇인가?
정답 보기함수 호출, 재귀, 자료구조 생성 같은 고정 비용이 상대적으로 크게 작용할 수 있기 때문입니다.
3. 메모이제이션과 반복 피보나치가 모두 O(n)인데 실제 시간이 다른 이유는 무엇인가?
정답 보기메모이제이션은 재귀 호출과 딕셔너리 조회 비용이 추가되지만 반복 방식은 단순한 변수 갱신만 수행하기 때문입니다.
4. 실행 시간을 한 번만 측정하지 않고 여러 번 측정하는 이유는 무엇인가?
정답 보기운영체제의 다른 작업, 캐시, 순간적인 부하로 인한 측정 변동을 줄이기 위해서입니다.
5. 집합이나 딕셔너리를 사용하는 빠른 알고리즘의 대표적인 단점은 무엇인가?
정답 보기추가 메모리 공간을 사용하고 해시 계산과 객체 관리 비용이 발생한다는 점입니다.
