1. 반복문과 재귀함수 비교 기준

가독성문제의 정의와 코드 구조가 얼마나 자연스럽게 연결되는지 살펴봅니다.
메모리반복 변수만 사용하는지, 함수 호출 정보를 스택에 계속 저장하는지 비교합니다.
실행 시간호출 오버헤드와 중복 계산 여부를 실제 시간 측정으로 확인합니다.
비교 항목반복문재귀함수
표현 방식반복 조건과 누적 변수를 직접 작성문제를 더 작은 같은 문제로 표현
메모리대체로 일정한 추가 메모리 사용호출마다 스택 프레임 추가
실행 속도함수 호출 부담이 적어 빠른 편호출 오버헤드가 있어 느릴 수 있음
적합한 문제단순 누적, 큰 반복 횟수트리, 분할 정복, 자기 유사 구조
예제 1

1부터 n까지의 합

누적 합기본 비교

반복문

def sum_loop(n):
    total = 0

    for number in range(1, n + 1):
        total += number

    return total

print(sum_loop(10))

재귀함수

def sum_recursive(n):
    if n == 1:
        return 1

    return n + sum_recursive(n - 1)

print(sum_recursive(10))
비교
재귀함수는 수학적 정의와 비슷해 간결하지만, 큰 n에서는 호출 스택이 깊어집니다. 반복문은 누적 과정이 명확하고 메모리 사용이 적습니다.
예제 2

팩토리얼 계산

팩토리얼곱셈 누적

반복문

def factorial_loop(n):
    result = 1

    for number in range(1, n + 1):
        result *= number

    return result

재귀함수

def factorial_recursive(n):
    if n == 0:
        return 1

    return n * factorial_recursive(n - 1)
항목반복문재귀함수
가독성곱셈 누적 과정을 직접 확인 가능n! = n×(n-1)! 정의와 유사
메모리result와 number만 주로 사용n만큼 호출 스택 사용
큰 입력상대적으로 안전재귀 깊이 제한에 걸릴 수 있음
예제 3

거듭제곱 계산

거듭제곱지수 감소

반복문

def power_loop(base, exponent):
    result = 1

    for _ in range(exponent):
        result *= base

    return result

재귀함수

def power_recursive(base, exponent):
    if exponent == 0:
        return 1

    return base * power_recursive(
        base, exponent - 1
    )
확장 생각
재귀함수는 지수를 절반씩 줄이는 분할 정복 방식으로 개선할 수 있어 빠른 거듭제곱 알고리즘으로 발전시키기 좋습니다.
예제 4

유클리드 호제법 최대공약수

최대공약수나머지

반복문

def gcd_loop(a, b):
    while b != 0:
        a, b = b, a % b

    return a

재귀함수

def gcd_recursive(a, b):
    if b == 0:
        return a

    return gcd_recursive(b, a % b)
비교
이 문제는 재귀 관계가 매우 분명하여 재귀 코드의 가독성이 높습니다. 하지만 반복문도 짧고 스택을 사용하지 않아 실용적입니다.
예제 5

정수의 자릿수 합

정수 분해나머지

반복문

def digit_sum_loop(n):
    total = 0

    while n > 0:
        total += n % 10
        n //= 10

    return total

재귀함수

def digit_sum_recursive(n):
    if n < 10:
        return n

    return (
        n % 10
        + digit_sum_recursive(n // 10)
    )
가독성 비교
재귀 방식은 “마지막 자리 + 나머지 자리의 합”이라는 문제 구조를 잘 드러냅니다. 반복문은 상태 변화가 한눈에 보입니다.
예제 6

리스트 원소의 합

리스트인덱스

반복문

def list_sum_loop(numbers):
    total = 0

    for number in numbers:
        total += number

    return total

재귀함수

def list_sum_recursive(numbers, index=0):
    if index == len(numbers):
        return 0

    return (
        numbers[index]
        + list_sum_recursive(numbers, index + 1)
    )
메모리 비교
리스트가 매우 길면 재귀 방식은 원소 수만큼 호출 스택을 사용하므로 비효율적입니다. 단순 합계는 반복문이나 내장 함수 sum()이 적합합니다.
예제 7

문자열 뒤집기

문자열역순

반복문

def reverse_loop(text):
    result = ""

    for character in text:
        result = character + result

    return result

재귀함수

def reverse_recursive(text):
    if len(text) <= 1:
        return text

    return (
        reverse_recursive(text[1:])
        + text[0]
    )
성능 주의
두 방식 모두 문자열을 계속 새로 만들 수 있습니다. 실제로는 슬라이싱 text[::-1]이나 reversed()가 더 효율적일 수 있습니다.
예제 8

이진 탐색

분할 정복탐색

반복문

def binary_search_loop(data, target):
    left = 0
    right = len(data) - 1

    while left <= right:
        middle = (left + right) // 2

        if data[middle] == target:
            return middle
        elif data[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

    return -1

재귀함수

def binary_search_recursive(
    data, target, left, right
):
    if left > right:
        return -1

    middle = (left + right) // 2

    if data[middle] == target:
        return middle
    if data[middle] < target:
        return binary_search_recursive(
            data, target, middle + 1, right
        )

    return binary_search_recursive(
        data, target, left, middle - 1
    )
알고리즘 구조
이진 탐색은 탐색 범위를 절반으로 줄이는 분할 정복 문제이므로 재귀 구조가 자연스럽습니다. 반복문은 함수 호출 없이 같은 효율을 얻습니다.
예제 9

피보나치 수열

중복 호출성능 차이

반복문

def fibonacci_loop(n):
    if n < 2:
        return n

    a, b = 0, 1

    for _ in range(2, n + 1):
        a, b = b, a + b

    return b

일반 재귀함수

def fibonacci_recursive(n):
    if n < 2:
        return n

    return (
        fibonacci_recursive(n - 1)
        + fibonacci_recursive(n - 2)
    )
중요 비교
일반 재귀 피보나치는 같은 값을 반복 계산하여 실행 시간이 급격히 증가합니다. 반복문은 각 항을 한 번씩 계산하므로 훨씬 빠르고 메모리도 적게 사용합니다.
예제 10

실행 시간과 메모리 특성 종합 비교

perf_counter종합 측정
import time

def factorial_loop(n):
    result = 1
    for number in range(1, n + 1):
        result *= number
    return result

def factorial_recursive(n):
    if n == 0:
        return 1
    return n * factorial_recursive(n - 1)

n = 900

start = time.perf_counter()
loop_result = factorial_loop(n)
loop_time = time.perf_counter() - start

start = time.perf_counter()
recursive_result = factorial_recursive(n)
recursive_time = time.perf_counter() - start

print("결과 일치:", loop_result == recursive_result)
print(f"반복문 시간: {loop_time:.8f}초")
print(f"재귀 시간: {recursive_time:.8f}초")
실행 환경 주의
파이썬의 기본 재귀 깊이는 제한되어 있으므로 지나치게 큰 n은 RecursionError를 일으킬 수 있습니다. 시간 측정값은 컴퓨터 상태에 따라 달라집니다.
선택 기준권장 방식이유
단순 누적 계산반복문빠르고 메모리 사용이 적음
트리·폴더 탐색재귀함수중첩 구조를 자연스럽게 표현
분할 정복둘 다 가능재귀는 구조가 명확하고 반복문은 호출 부담이 적음
매우 큰 입력반복문재귀 깊이 제한을 피할 수 있음

마무리 진단 문제

1. 반복문이 재귀함수보다 일반적으로 메모리를 적게 사용하는 이유는 무엇인가?
정답 보기반복문은 반복 변수만 갱신하지만 재귀함수는 호출마다 지역 변수와 복귀 위치를 호출 스택에 저장하기 때문입니다.
2. 재귀함수의 가독성이 특히 높아질 수 있는 문제는 무엇인가?
정답 보기트리 탐색, 분할 정복, 폴더 구조처럼 자기와 비슷한 작은 구조를 포함하는 문제입니다.
3. 일반 재귀 피보나치가 느린 이유는 무엇인가?
정답 보기같은 피보나치 값을 여러 번 중복 계산하기 때문입니다.
4. 실행 시간을 측정할 때 사용할 수 있는 함수는 무엇인가?
정답 보기time.perf_counter()입니다.
5. 단순한 1부터 n까지의 합을 매우 큰 n에 대해 계산할 때 어느 방식이 더 적합한가?
정답 보기호출 스택을 사용하지 않는 반복문 방식이 더 적합합니다.
