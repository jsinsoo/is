계단 오르기 문제
상태·초기값·점화식 설계 예제 10선

한 번에 1칸 또는 2칸을 오르는 계단 문제를 중심으로 상태를 정의하고, 마지막 이동을 기준으로 부분 문제를 나누어 점화식을 설계한 뒤 재귀·메모이제이션·테이블·공간 최적화 방식으로 구현합니다.

설계 원리1. 작은 문제 관찰2. 상태 정의 3. 단순 재귀4. 호출 분석5. 메모이제이션 6. 테이블7. 공간 최적화8. 1·2·3칸 9. 금지 계단10. 최소 비용

1. 상태와 점화식 설계 절차

상태 정의dp[n]이 무엇을 의미하는지 한 문장으로 정합니다.
마지막 선택n번째 계단에 도착하기 직전의 가능한 위치를 찾습니다.
점화식현재 상태를 더 작은 상태의 값으로 표현합니다.
초기값점화식을 시작할 수 있는 가장 작은 상태를 정합니다.
기본 문제: 한 번에 1칸 또는 2칸을 오를 수 있을 때, 0번째 위치에서 n번째 계단에 도착하는 방법의 수를 구합니다.
설계 요소기본 계단 문제
상태dp[n] = n번째 계단에 도착하는 방법의 수
마지막 이동n-1에서 1칸 이동 또는 n-2에서 2칸 이동
점화식dp[n] = dp[n-1] + dp[n-2]
초기값dp[0]=1, dp[1]=1
예제 1

작은 계단의 경우를 직접 나열하기

규칙 발견작은 입력
ways = {
    0: [[]],
    1: [[1]],
    2: [[1, 1], [2]],
    3: [
        [1, 1, 1],
        [1, 2],
        [2, 1]
    ]
}

for stair, cases in ways.items():
    print(stair, "칸:", len(cases), "가지")
    for case in cases:
        print(" ", case)

관찰

0칸: 1가지
1칸: 1가지
2칸: 2가지
3칸: 3가지
4칸: 5가지

현재 값이 앞의 두 값의 합으로 나타나는 규칙을 발견할 수 있습니다.

예제 2

마지막 이동으로 상태 분해하기

상태 전이점화식
def explain_state(n):
    print(f"목표 상태: {n}번째 계단")

    if n >= 1:
        print(
            f"경우 1: {n-1}번째 계단에서 "
            "1칸 이동"
        )

    if n >= 2:
        print(
            f"경우 2: {n-2}번째 계단에서 "
            "2칸 이동"
        )

    print(
        f"점화식: dp[{n}] = "
        f"dp[{n-1}] + dp[{n-2}]"
    )

explain_state(6)
중복 없음
마지막 이동이 1칸인 경우와 2칸인 경우는 서로 겹치지 않으므로 두 경우의 수를 더할 수 있습니다.
예제 3

점화식을 단순 재귀함수로 구현

재귀초기값
def stairs_recursive(n):
    if n == 0:
        return 1

    if n == 1:
        return 1

    return (
        stairs_recursive(n - 1)
        + stairs_recursive(n - 2)
    )

for n in range(8):
    print(n, "칸:", stairs_recursive(n))

코드와 점화식

W(0) = 1
W(1) = 1
W(n) = W(n-1) + W(n-2)

점화식을 코드에 직접 옮겨 가독성은 높지만 중복 호출이 발생합니다.

예제 4

재귀 호출 횟수와 중복 부분 문제 분석

중복 호출호출 수
call_count = 0
state_calls = {}

def stairs_recursive(n):
    global call_count

    call_count += 1
    state_calls[n] = state_calls.get(n, 0) + 1

    if n <= 1:
        return 1

    return (
        stairs_recursive(n - 1)
        + stairs_recursive(n - 2)
    )

result = stairs_recursive(8)

print("결과:", result)
print("전체 호출 수:", call_count)

for state in sorted(state_calls):
    print(
        state,
        "상태 호출:",
        state_calls[state]
    )
문제점
같은 계단 수에 대한 결과가 여러 번 계산됩니다. 예를 들어 stairs_recursive(4)가 여러 호출 경로에서 반복됩니다.
예제 5

하향식 메모이제이션 적용

Top-down캐시
memo = {0: 1, 1: 1}
new_calculations = 0

def stairs_memo(n):
    global new_calculations

    if n in memo:
        return memo[n]

    new_calculations += 1

    memo[n] = (
        stairs_memo(n - 1)
        + stairs_memo(n - 2)
    )

    return memo[n]

print("결과:", stairs_memo(30))
print("새 계산 수:", new_calculations)
print("저장된 상태:", memo)
개선
각 n의 값은 한 번만 새로 계산되고, 이후 같은 상태가 필요하면 딕셔너리에서 즉시 반환됩니다.
예제 6

상향식 테이블 방식 구현

Bottom-upDP 테이블
def stairs_table(n):
    dp = [0] * (n + 1)
    dp[0] = 1

    if n >= 1:
        dp[1] = 1

    for stair in range(2, n + 1):
        dp[stair] = (
            dp[stair - 1]
            + dp[stair - 2]
        )

    print("테이블:", dp)
    return dp[n]

print("결과:", stairs_table(10))

계산 순서

dp[0], dp[1]
→ dp[2]
→ dp[3]
→ ...
→ dp[n]

이전 상태가 먼저 계산되도록 작은 계단부터 표를 채웁니다.

예제 7

두 변수만 사용하는 공간 최적화

O(1) 공간상태 압축
def stairs_optimized(n):
    if n <= 1:
        return 1

    previous2 = 1  # dp[0]
    previous1 = 1  # dp[1]

    for stair in range(2, n + 1):
        current = previous1 + previous2

        previous2 = previous1
        previous1 = current

        print(
            f"{stair}칸: "
            f"이전2={previous2}, "
            f"현재={previous1}"
        )

    return previous1

print("결과:", stairs_optimized(10))
공간 최적화 근거
현재 상태는 바로 앞의 두 상태만 필요하므로 전체 테이블을 저장하지 않아도 됩니다.
예제 8

한 번에 1·2·3칸을 오르는 확장 문제

점화식 확장세 이전 상태
def stairs_123(n):
    dp = [0] * (n + 1)
    dp[0] = 1

    for stair in range(1, n + 1):
        if stair >= 1:
            dp[stair] += dp[stair - 1]

        if stair >= 2:
            dp[stair] += dp[stair - 2]

        if stair >= 3:
            dp[stair] += dp[stair - 3]

    return dp

table = stairs_123(10)

for stair, value in enumerate(table):
    print(stair, "칸:", value)

상태 전이

dp[n] =
dp[n-1]
+ dp[n-2]
+ dp[n-3]

마지막 이동이 1칸, 2칸, 3칸인 세 경우로 나눕니다.

예제 9

밟을 수 없는 계단이 있는 경우

제약 조건도달 불가 상태
def stairs_with_blocked(n, blocked):
    dp = [0] * (n + 1)
    dp[0] = 1

    for stair in range(1, n + 1):
        if stair in blocked:
            dp[stair] = 0
            continue

        if stair >= 1:
            dp[stair] += dp[stair - 1]

        if stair >= 2:
            dp[stair] += dp[stair - 2]

    return dp

blocked = {3, 7}
dp = stairs_with_blocked(10, blocked)

for stair, count in enumerate(dp):
    state = "금지" if stair in blocked else "가능"
    print(stair, state, count)
상태 처리
금지된 계단은 도착 방법의 수를 0으로 설정합니다. 이후 계단의 점화식은 이 0을 자연스럽게 반영합니다.
예제 10

계단 오르기 최소 비용 상태 설계

최솟값최적 부분 구조경로 복원
def minimum_cost_stairs(cost):
    n = len(cost)

    dp = [0] * n
    parent = [-1] * n

    dp[0] = cost[0]

    if n >= 2:
        dp[1] = cost[1]

    for stair in range(2, n):
        if dp[stair - 1] <= dp[stair - 2]:
            dp[stair] = (
                dp[stair - 1]
                + cost[stair]
            )
            parent[stair] = stair - 1
        else:
            dp[stair] = (
                dp[stair - 2]
                + cost[stair]
            )
            parent[stair] = stair - 2

    # 마지막 계단에서 끝나는 것으로 가정
    path = []
    current = n - 1

    while current >= 0:
        path.append(current)
        current = parent[current]

    path.reverse()

    return dp[-1], path, dp

cost = [3, 5, 2, 6, 1, 4]

minimum, path, table = (
    minimum_cost_stairs(cost)
)

print("비용:", cost)
print("최소 누적 비용:", minimum)
print("선택 계단:", path)
print("DP 테이블:", table)
설계 요소최소 비용 계단 문제
상태dp[i] = i번째 계단에 도착하는 최소 누적 비용
마지막 이동i-1에서 1칸 또는 i-2에서 2칸 이동
점화식dp[i] = cost[i] + min(dp[i-1], dp[i-2])
초기값dp[0]=cost[0], dp[1]=cost[1]
경로 복원최솟값을 만든 이전 계단을 parent에 저장
알고리즘 설명 예시
“상태 dp[i]를 i번째 계단에 도착하는 최소 누적 비용으로 정의하였다. i번째 계단에는 i-1번째 또는 i-2번째 계단에서 올 수 있으므로, 두 이전 상태 중 비용이 작은 값을 선택하고 현재 계단 비용을 더하였다. 따라서 점화식은 dp[i]=cost[i]+min(dp[i-1], dp[i-2])로 설계하였다.”

마무리 진단 문제

1. 기본 계단 오르기 문제에서 dp[n]은 무엇을 의미하는가?
정답 보기0번째 위치에서 n번째 계단에 도착하는 방법의 수입니다.
2. 한 번에 1칸 또는 2칸을 오를 때 점화식은 무엇인가?
정답 보기dp[n]=dp[n-1]+dp[n-2]입니다.
3. dp[0]=1로 설정하는 이유는 무엇인가?
정답 보기아무 이동도 하지 않고 출발 위치에 있는 한 가지 방법을 의미하며, 이후 점화식을 자연스럽게 계산하기 위해서입니다.
4. 테이블을 두 변수로 줄일 수 있는 이유는 무엇인가?
정답 보기현재 상태 계산에 바로 이전 두 상태만 필요하기 때문입니다.
5. 최소 비용 문제의 상태는 경우의 수 문제와 어떻게 다른가?
정답 보기방법의 수가 아니라 해당 계단에 도착하는 최소 누적 비용을 저장하며, 덧셈 대신 이전 상태의 최솟값을 선택합니다.
