1 자료원의 유형

공공·센서·설문·실험

개념 상세 설명: 공공데이터는 넓은 범위와 장기간 자료를 얻기 좋고, 센서는 현장을 연속적으로 측정하기 좋다. 설문은 태도와 경험을 조사할 수 있으며, 실험은 조건을 조절해 인과 관계를 탐색하기 좋다. 연구 질문에 따라 한 가지 또는 여러 자료원을 결합할 수 있다.

sources = {
    "공공데이터": "장기간·광범위 자료",
    "센서": "현장 연속 측정",
    "설문": "인식·경험 조사",
    "실험": "조건 통제와 비교"
}
for source, feature in sources.items():
    print(source, ":", feature)
코드 상세 설명: 딕셔너리를 이용해 자료원의 종류와 핵심 특성을 연결했다. 자료원은 연구 질문, 비용, 기간, 윤리 조건을 고려해 선택한다.
[개념 1] 자료원의 유형에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 각 자료원은 서로 다른 강점을 가지며 결합할 수도 있다.

2 출처 신뢰도 평가

누가·어떻게 만들었는가

개념 상세 설명: 자료의 신뢰도는 생산 기관, 수집 목적, 조사 방법, 변수 정의, 갱신 날짜, 오류 수정 절차 등을 확인하여 평가한다. 기관 이름만 보고 판단하기보다 실제 수집 과정과 문서화 수준을 살펴야 한다.

source_info = {
    "기관": "국가기관",
    "갱신일": "2026-07-01",
    "수집방법": "자동 측정망",
    "변수설명": True
}
score = sum([source_info["기관"] != "",
             source_info["갱신일"] != "",
             source_info["수집방법"] != "",
             source_info["변수설명"]])
print("출처 점검 항목 충족:", score, "/ 4")
코드 상세 설명: 출처 평가 항목이 제공되는지 확인해 간단한 점검 점수를 계산한다. 실제 평가에서는 각 항목의 내용과 품질을 함께 살펴야 한다.
[개념 2] 자료 출처를 평가할 때 확인할 내용 3개를 고르시오.
정답: ①, ②, ④ — 신뢰도는 생산 주체와 수집·관리 과정으로 판단한다.

3 메타데이터

자료를 설명하는 자료

개념 상세 설명: 메타데이터는 자료의 변수명, 단위, 수집 시각, 위치, 결측값 표시, 파일 형식, 이용 조건 등을 설명하는 정보이다. 메타데이터가 없으면 숫자의 의미를 잘못 해석할 위험이 커진다.

metadata = {
    "변수명": "pm25",
    "단위": "㎍/㎥",
    "수집주기": "1시간",
    "결측표시": -999
}
for key, value in metadata.items():
    print(key, "=", value)
코드 상세 설명: 측정 자료를 해석하는 데 필요한 설명 정보를 딕셔너리에 저장한다. 결측값 코드와 단위를 반드시 확인해야 한다.
[개념 3] 메타데이터에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 메타데이터는 자료의 의미와 수집 조건을 설명한다.

4 수집 주기와 기간

언제 얼마나 자주

개념 상세 설명: 수집 주기는 측정 간격이며 수집 기간은 자료를 모으는 전체 시간이다. 빠르게 변하는 온도나 CO2는 짧은 주기가 필요할 수 있고, 계절 변화는 긴 기간이 필요하다. 지나치게 짧거나 긴 주기는 자료량과 배터리, 저장 공간에 영향을 준다.

minutes = 10
hours = 6
count = (hours * 60) // minutes
print("수집 주기:", minutes, "분")
print("예상 측정 횟수:", count)
코드 상세 설명: 10분 간격으로 6시간 측정할 때 예상 자료 개수를 계산한다. 수집 계획에서 자료량을 미리 계산하면 저장 공간과 분석 시간을 예측할 수 있다.
[개념 4] 수집 주기와 기간에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 현상, 자원, 분석 목적에 맞춰 주기와 기간을 설계한다.

5 표본과 모집단

누구를 조사할 것인가

개념 상세 설명: 모집단은 알고 싶은 전체 대상이고 표본은 실제로 조사한 일부 대상이다. 표본은 가능한 한 모집단의 특성을 잘 반영해야 하며, 특정 집단만 편리하게 선택하면 결과를 전체로 일반화하기 어렵다.

population = 240
sample = 60
sampling_rate = sample / population * 100
print("모집단:", population, "명")
print("표본:", sample, "명")
print("표본 비율:", sampling_rate, "%")
코드 상세 설명: 전체 학생 중 조사한 학생의 비율을 계산한다. 표본의 크기뿐 아니라 학년, 성별, 참여 경로 등 구성의 대표성도 중요하다.
[개념 5] 표본과 모집단에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ⑤ — 대표성 있는 표본 설계가 일반화 가능성에 영향을 준다.

6 표본 편향

치우친 선택

개념 상세 설명: 표본 편향은 특정 특성을 가진 사람이나 상황이 과도하게 포함되거나 빠져 결과가 한쪽으로 치우치는 현상이다. 온라인 자율 설문은 관심이 높은 사람만 참여할 수 있고, 점심시간 조사만 하면 다른 시간대 이용자가 제외될 수 있다.

sample_counts = {"1학년": 45, "2학년": 10, "3학년": 5}
total = sum(sample_counts.values())
for grade, count in sample_counts.items():
    print(grade, round(count / total * 100, 1), "%")
코드 상세 설명: 표본에서 학년별 비율을 계산해 특정 학년이 과도하게 포함되었는지 확인한다. 모집단 구성과 비교하면 편향을 더 정확히 판단할 수 있다.
[개념 6] 표본 편향을 줄이는 방법 3개를 고르시오.
정답: ②, ④, ⑤ — 표본의 다양성을 확보하고 한계를 투명하게 기록해야 한다.

7 센서 정확도와 보정

측정값 점검

개념 상세 설명: 센서 자료에는 장비 오차, 위치 차이, 예열 시간, 환경 간섭이 포함될 수 있다. 기준 장비와 비교하거나 같은 조건에서 반복 측정하여 오차를 확인하고 필요한 경우 보정값을 적용한다.

sensor = [24.8, 25.0, 25.1]
reference = [24.3, 24.5, 24.6]
offset = sum(s-r for s, r in zip(sensor, reference)) / len(sensor)
corrected = [round(x-offset, 1) for x in sensor]
print("평균 보정값:", round(offset, 2))
print("보정 후:", corrected)
코드 상세 설명: 센서와 기준 장비의 평균 차이를 계산하고 측정값에서 빼 간단히 보정한다. 실제 센서 보정은 장비 특성과 온도 범위 등을 고려해야 한다.
[개념 7] 센서 자료의 신뢰도를 높이는 방법 3개를 고르시오.
정답: ①, ③, ⑤ — 비교·반복·기록을 통해 측정 신뢰도를 높인다.

8 설문 문항과 응답 품질

편향 없는 질문

개념 상세 설명: 설문 문항은 한 문항에 한 가지 내용만 묻고, 특정 답을 유도하지 않으며, 응답 기준과 기간을 분명히 해야 한다. 익명성과 자발적 참여를 보장하고 민감한 질문은 꼭 필요한 경우에만 포함한다.

questions = [
    "지난 7일 동안 하루 평균 학습 시간은 몇 분입니까?",
    "수업 자료의 이해 도움 정도를 1~5점으로 평가하세요."
]
for i, q in enumerate(questions, start=1):
    print(i, q)
코드 상세 설명: 기간과 응답 단위가 분명한 설문 문항을 번호와 함께 출력한다. 모호하거나 두 가지를 동시에 묻는 문항은 수정해야 한다.
[개념 8] 좋은 설문 문항의 조건 3개를 고르시오.
정답: ①, ②, ④ — 단일성, 명확성, 이해 가능성이 응답 품질을 높인다.

9 개인정보와 연구 윤리

최소 수집·동의·보호

개념 상세 설명: 자료 수집은 연구 목적에 필요한 최소한의 정보만 모으고, 참여 목적과 이용 범위를 알린 뒤 동의를 받아야 한다. 직접 식별 정보는 가능하면 수집하지 않으며, 가명 처리와 접근 제한, 보관 기간 설정이 필요하다.

raw_record = {"이름":"홍길동", "학년":1, "만족도":4}
safe_record = {"참여자ID":"P001", "학년":1, "만족도":4}
print("분석용 자료:", safe_record)
코드 상세 설명: 이름 대신 임의의 참여자 ID를 사용해 직접 식별 가능성을 줄였다. 가명화만으로 위험이 완전히 사라지는 것은 아니므로 접근 권한과 보관 방식도 관리해야 한다.
[개념 9] 책임 있는 개인정보 수집 방법 3개를 고르시오.
정답: ②, ③, ⑤ — 최소 수집, 동의, 안전한 보관이 핵심이다.

10 자료 수집 계획서

재현 가능한 설계

개념 상세 설명: 자료 수집 계획서는 무엇을, 어디에서, 누가, 언제, 어떤 단위와 도구로 수집하는지 기록한 문서이다. 파일명 규칙, 결측 처리, 품질 점검, 백업, 윤리 절차까지 포함하면 다른 사람이 과정을 이해하고 재현할 수 있다.

plan = {
    "변수": "교실 CO2",
    "도구": "CO2 센서",
    "주기": "5분",
    "기간": "수업일 10일",
    "위치": "교실 중앙 1.2m",
    "파일형식": "CSV"
}
for key, value in plan.items():
    print(key, ":", value)
