프로토타입 구현 Ⅱ

11차시에서 구현한 수집·전처리·분석 기능에 그래프나 사용자 인터페이스를 연결한다. 사용자가 무엇을 입력해야 하는지 이해하고, 버튼을 눌러 결과를 확인하며, 오류가 발생했을 때 원인과 해결 방법을 알 수 있도록 사용 흐름을 개선한다.

핵심 산출물
  • 입력–처리–결과 화면
  • 자료 특성에 맞는 시각화
  • 명확한 레이블과 버튼
  • 오류·성공·진행 상태 메시지
  • 사용성 점검표와 개선 기록

차시 운영 예시

도입 8분
같은 기능을 가진 두 화면을 비교하고 어느 화면이 더 쉽게 사용할 수 있는지 근거를 찾는다.
개념 17분
사용자 인터페이스, 상호작용, 정보 구조, 시각화, 피드백, 오류 메시지, 접근성, 일관성, 사용성 시험과 개선을 학습한다.
활동 20분
입력창·버튼·결과 영역 또는 그래프를 추가하고 동료가 실제 사용하며 불편한 지점을 기록한다.
정리 5분
개선 전후 사용 흐름을 비교하고 가장 효과가 컸던 수정 한 가지를 발표한다.

성취기준

영역수업에 적용한 성취기준
상호작용 설계사용자의 입력과 행동에 따라 결과가 적절히 변화하는 인터페이스를 구현한다.
시각화와 정보 전달자료의 특성과 전달 목적에 맞는 시각화 또는 결과 표현 방식을 선택한다.
사용성사용 흐름, 레이블, 버튼, 상태 피드백과 화면 배치를 사용자의 관점에서 개선한다.
오류 대응오류의 원인과 해결 방법을 알려 주는 구체적이고 친절한 메시지를 제공한다.
검증과 개선사용성 시험 결과를 바탕으로 인터페이스와 상호작용을 수정하고 개선 근거를 기록한다.

🧩 학습할 핵심 개념 10개

1 사용자 인터페이스와 상호작용

입력과 반응의 연결

개념 상세 설명: 사용자 인터페이스는 사람이 프로그램과 정보를 주고받는 화면과 조작 방법이다. 상호작용은 사용자의 입력이나 선택에 따라 프로그램의 상태와 결과가 변화하는 과정이다.

import tkinter as tk

def greet():
    name = name_entry.get().strip()
    result_label.config(text=f"{name}님, 탐구를 시작합니다!")

root = tk.Tk()
root.title("탐구 도우미")

name_entry = tk.Entry(root)
name_entry.pack(padx=12, pady=8)

tk.Button(root, text="시작", command=greet).pack()
result_label = tk.Label(root, text="이름을 입력하세요.")
result_label.pack(pady=8)

root.mainloop()
코드 상세 설명: 사용자가 입력창에 이름을 쓰고 버튼을 누르면 greet() 함수가 실행되어 결과 레이블이 바뀐다. 입력–행동–피드백이 연결된 기본 상호작용이다.
[개념 1] 사용자 인터페이스와 상호작용에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 상호작용은 사용자 행동과 시스템 반응을 연결한다.

2 사용자 목표와 사용 흐름

해야 할 일을 자연스러운 순서로

개념 상세 설명: 사용 흐름은 사용자가 목표를 달성하기 위해 거치는 화면과 행동의 순서이다. 불필요한 입력을 줄이고 중요한 작업을 먼저 제시하며 이전 단계로 돌아갈 방법을 제공해야 한다.

steps = [
    "1. 자료 입력",
    "2. 입력 확인",
    "3. 분석 실행",
    "4. 결과 확인",
    "5. 결과 저장 또는 다시 입력"
]

print("사용 흐름")
for step in steps:
    print(step)
코드 상세 설명: 사용자의 행동을 순서대로 정리한 간단한 흐름 모델이다. 구현 전에 단계가 빠졌거나 불필요하게 반복되는 부분이 없는지 점검할 수 있다.
[개념 2] 사용 흐름 설계에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 사용자의 목표와 행동 순서를 중심으로 흐름을 구성한다.

3 명확한 레이블과 버튼

행동 결과를 예측하게 한다

개념 상세 설명: 레이블은 입력해야 할 자료와 단위를 알려 주고, 버튼 문구는 눌렀을 때 일어날 행동을 분명하게 나타내야 한다. ‘확인’보다 ‘분석 시작’, ‘저장’보다 ‘분석 결과 저장’처럼 구체적인 표현이 좋다.

import tkinter as tk

root = tk.Tk()
root.title("온도 분석")

tk.Label(root, text="교실 온도(℃)").pack()
temperature_entry = tk.Entry(root)
temperature_entry.pack()

tk.Button(
    root,
    text="온도 분석 시작",
    command=lambda: print(temperature_entry.get())
).pack(pady=8)

root.mainloop()
코드 상세 설명: 입력 레이블에 자료의 의미와 단위를 함께 표시하고 버튼에는 실행 결과를 예상할 수 있는 문구를 사용했다.
[개념 3] 레이블과 버튼 설계에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 이름, 단위와 행동을 구체적으로 표현해야 한다.

4 자료 특성에 맞는 시각화

그래프가 질문에 답하게 한다

개념 상세 설명: 시각화는 장식이 아니라 자료의 비교, 변화, 분포와 관계를 이해하도록 돕는 도구이다. 범주 비교에는 막대그래프, 시간 변화에는 선그래프, 분포에는 히스토그램, 두 수치의 관계에는 산점도가 적합하다.

import matplotlib.pyplot as plt

hours = ["09시", "10시", "11시", "12시", "13시"]
co2 = [620, 740, 880, 1120, 930]

plt.rcParams["font.family"] = "Malgun Gothic"
plt.rcParams["axes.unicode_minus"] = False

plt.plot(hours, co2, marker="o")
plt.axhline(1000, linestyle="--", label="환기 점검 기준")
plt.title("시간에 따른 교실 이산화탄소 농도")
plt.xlabel("측정 시각")
plt.ylabel("이산화탄소 농도(ppm)")
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()
코드 상세 설명: 시간에 따른 변화이므로 선그래프를 사용하고, 기준선을 추가해 어느 시점에 환기가 필요한지 쉽게 판단하도록 했다.
[개념 4] 시각화 선택에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 자료 구조와 전달 목적에 맞는 그래프를 선택한다.

5 즉각적인 상태 피드백

진행·성공·실패를 알린다

개념 상세 설명: 사용자의 행동 뒤에는 프로그램이 무엇을 하는지 알려 주는 피드백이 필요하다. 입력 중, 처리 중, 완료와 실패 상태를 적절한 문구로 보여 주면 사용자는 같은 버튼을 여러 번 누르거나 프로그램이 멈췄다고 오해하지 않는다.

import tkinter as tk

def analyze():
    status.config(text="자료를 분석하는 중입니다...")
    root.after(800, finish)

def finish():
    status.config(text="분석이 완료되었습니다. 결과를 확인하세요.")

root = tk.Tk()
root.title("상태 피드백")

tk.Button(root, text="분석 시작", command=analyze).pack(pady=8)
status = tk.Label(root, text="분석할 자료를 준비하세요.")
status.pack(padx=15, pady=10)

root.mainloop()
코드 상세 설명: 버튼을 누른 직후 진행 상태를 보여 주고 일정 시간이 지난 뒤 완료 메시지로 바꾼다. 실제 긴 작업에서는 진행 표시나 버튼 비활성화를 함께 사용할 수 있다.
[개념 5] 상태 피드백에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ⑤ — 시스템 상태를 즉시 알려 주면 사용자가 다음 행동을 판단할 수 있다.

6 친절한 오류 메시지

원인과 해결 방법을 함께

개념 상세 설명: 좋은 오류 메시지는 단순히 ‘오류’라고 말하지 않는다. 어떤 입력이 잘못되었는지, 허용되는 형식은 무엇인지, 사용자가 어떻게 수정해야 하는지를 구체적으로 알려 준다.

def validate_score(raw):
    try:
        score = float(raw)
    except ValueError:
        return False, "점수는 숫자로 입력하세요. 예: 85.5"

    if not 0 <= score <= 100:
        return False, "점수는 0점 이상 100점 이하로 입력하세요."

    return True, "입력이 확인되었습니다."

for sample in ["구십", "130", "92.5"]:
    ok, message = validate_score(sample)
    print(sample, "→", message)
코드 상세 설명: 숫자 형식 오류와 범위 오류를 구분하며 올바른 입력 예시나 허용 범위를 안내한다. 사용자는 메시지만 보고도 수정 방법을 알 수 있다.
[개념 6] 오류 메시지에 대한 옳은 설명 3개를 고르시오.
정답: ②, ④, ⑤ — 오류의 위치, 원인과 해결 방법을 알려 주어야 한다.

7 입력 중 검증과 오류 위치 표시

문제가 생긴 곳에서 바로 안내

개념 상세 설명: 입력을 모두 마친 뒤 한꺼번에 오류를 알리는 것보다 문제가 있는 입력창 가까이에 안내를 표시하는 것이 좋다. 입력 중 또는 입력 직후 검증하면 수정해야 할 위치를 쉽게 찾을 수 있다.

import tkinter as tk

def check_age(*args):
    raw = age_var.get().strip()

    if raw == "":
        message.config(text="나이를 입력하세요.")
    elif not raw.isdigit():
        message.config(text="나이는 정수로 입력하세요.")
    elif not 10 <= int(raw) <= 100:
        message.config(text="나이는 10~100 범위로 입력하세요.")
    else:
        message.config(text="올바른 입력입니다.")

root = tk.Tk()
age_var = tk.StringVar()
age_var.trace_add("write", check_age)

tk.Label(root, text="나이").pack()
tk.Entry(root, textvariable=age_var).pack()
message = tk.Label(root, text="나이를 입력하세요.")
message.pack()

root.mainloop()
코드 상세 설명: 사용자가 입력할 때마다 검증 함수가 실행되고 입력창 아래의 메시지가 바뀐다. 오류 위치와 수정 방법을 즉시 확인할 수 있다.
[개념 7] 입력 중 검증에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 오류를 입력 위치 가까이에서 즉시 안내하면 수정하기 쉽다.

8 일관성과 접근성

누구나 예측하고 사용할 수 있게

개념 상세 설명: 일관성은 같은 기능에 같은 표현과 배치를 사용하는 것이다. 접근성은 색각, 시력, 운동 능력이나 사용 환경이 달라도 정보를 이해하고 조작할 수 있도록 설계하는 원칙이다. 색상만으로 상태를 구분하지 않고 글자나 기호를 함께 사용한다.

status_messages = {
    "success": "✓ 저장 완료: 결과가 result.txt에 저장되었습니다.",
    "warning": "△ 확인 필요: 일부 값이 허용 범위를 벗어났습니다.",
    "error": "✕ 입력 오류: 숫자를 입력한 뒤 다시 시도하세요."
}

for state, message in status_messages.items():
    print(state, message)
코드 상세 설명: 성공·주의·오류 상태를 색상에만 의존하지 않고 기호와 구체적인 문장으로도 구분한다. 모든 화면에서 같은 표현을 반복하면 사용자가 상태를 빠르게 이해한다.
[개념 8] 일관성과 접근성에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 일관된 표현과 다양한 사용 조건을 고려해야 한다.

9 사용성 시험

실제 사용 행동을 관찰

개념 상세 설명: 사용성 시험은 사용자가 설명 없이 과제를 수행하게 하고 성공 여부, 걸린 시간, 막힌 지점과 오류를 관찰하는 과정이다. 사용자의 행동과 발언을 근거로 인터페이스를 개선한다.

test_results = [
    {"user": "A", "success": True,  "seconds": 28, "error": 0},
    {"user": "B", "success": False, "seconds": 75, "error": 2},
    {"user": "C", "success": True,  "seconds": 34, "error": 1}
]

success_count = sum(row["success"] for row in test_results)
mean_time = sum(row["seconds"] for row in test_results) / len(test_results)
error_count = sum(row["error"] for row in test_results)

print("성공 인원:", success_count)
print("평균 수행 시간:", round(mean_time, 1), "초")
print("전체 오류 횟수:", error_count)
코드 상세 설명: 과제 성공 여부, 수행 시간과 오류 횟수를 기록하여 사용성을 수치로 점검한다. 수치뿐 아니라 사용자가 혼란스러워한 표현도 함께 기록해야 한다.
[개념 9] 사용성 시험에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 실제 사용 행동과 결과를 근거로 문제를 찾는다.

10 반복적 개선과 비교

만들기–시험–수정의 순환

개념 상세 설명: 프로토타입은 한 번 만들고 끝나는 산출물이 아니다. 문제를 관찰하고 수정한 뒤 다시 시험하며, 개선 전후의 성공률, 수행 시간과 오류 수를 비교해 수정의 효과를 확인한다.

before = {"success_rate": 60, "mean_time": 72, "errors": 8}
after  = {"success_rate": 90, "mean_time": 41, "errors": 3}

print("성공률 변화:", after["success_rate"] - before["success_rate"], "%p")
print("평균 시간 감소:", before["mean_time"] - after["mean_time"], "초")
print("오류 감소:", before["errors"] - after["errors"], "회")
코드 상세 설명: 인터페이스 개선 전후의 사용성 지표를 비교한다. 성공률은 높아지고 시간과 오류가 줄었다면 개선 효과가 있었음을 근거로 설명할 수 있다.
[개념 10] 반복적 개선에 대한 옳은 설명 3개를 고르시오.
정답: ①, ④, ⑤ — 시험과 수정을 반복하고 변화의 근거를 남긴다.
