1. 구조 설계

한 행은 하나의 관측, 한 열은 하나의 변수가 되도록 표 구조를 설계하고 변수명을 정한다.

2. 기준 결정

자료형, 단위, 허용 범위, 범주 코드, 결측값과 이상값 처리 규칙을 정한다.

3. 사전 작성

다른 사람도 자료를 이해하고 같은 전처리를 반복할 수 있도록 데이터 사전과 변경 기록을 작성한다.

🧩 학습할 핵심 개념 10개

1 변수와 관측값

행과 열의 의미

개념 상세 설명: 변수는 관측 대상의 특성을 나타내는 항목이며, 관측값은 실제로 기록된 값이다. 일반적으로 한 행은 한 번의 관측이나 한 대상을, 한 열은 하나의 변수를 나타내도록 구성하면 분석이 쉽다.

records = [
    {"시간":"09:00", "온도":23.5, "습도":42},
    {"시간":"09:10", "온도":23.8, "습도":41}
]
for row in records:
    print(row["시간"], row["온도"], row["습도"])
코드 상세 설명: 각 딕셔너리는 한 번의 관측을, 키는 변수를 나타낸다. 모든 관측에서 같은 변수명을 사용하면 표 형태로 변환하기 쉽다.
[개념 1] 변수와 관측값에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 일관된 행과 열 구조가 분석의 기초가 된다.

2 자료형

숫자·문자·논리·날짜

개념 상세 설명: 자료형은 값의 성격을 나타낸다. 정수, 실수, 문자열, 논리값, 날짜·시간 등이 있으며, 학번처럼 숫자로 보이지만 계산하지 않는 값은 문자열로 저장하는 것이 적절할 수 있다.

student_id = "0107"
temperature = 24.6
count = 18
ventilated = True

print(type(student_id))
print(type(temperature))
print(type(count))
print(type(ventilated))
코드 상세 설명: 학번은 앞자리 0을 보존하기 위해 문자열로 저장했다. 온도는 실수, 인원은 정수, 환기 여부는 논리값으로 표현한다.
[개념 2] 자료형 선택에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 값의 의미와 분석 목적에 맞는 자료형을 선택한다.

3 데이터 사전

자료를 설명하는 설계서

개념 상세 설명: 데이터 사전은 변수명, 변수 설명, 자료형, 단위, 허용 범위, 범주 코드, 결측 표시, 출처 등을 정리한 문서이다. 분석자 간 해석 차이를 줄이고 전처리 기준을 일관되게 유지한다.

data_dictionary = {
    "co2": {
        "설명":"교실 이산화탄소 농도",
        "자료형":"정수",
        "단위":"ppm",
        "허용범위":"400~5000"
    }
}
print(data_dictionary["co2"])
코드 상세 설명: 변수 이름과 설명 정보를 중첩 딕셔너리로 저장한다. 실제 데이터 사전에는 결측 코드와 수집 출처도 포함할 수 있다.
[개념 3] 데이터 사전에 포함할 내용 3개를 고르시오.
정답: ②, ③, ⑤ — 데이터 사전은 자료의 의미와 처리 규칙을 문서화한다.

4 변수명과 코딩 규칙

일관된 이름과 값

개념 상세 설명: 변수명은 짧고 의미가 분명하며 일관된 규칙을 따라야 한다. 범주형 값은 ‘남/남자/M’처럼 섞지 않고 하나의 표준이나 숫자 코드로 통일하며 코드의 의미를 데이터 사전에 기록한다.

raw = ["남", "남자", "M", "여", "F"]
mapping = {"남":1, "남자":1, "M":1, "여":2, "F":2}
coded = [mapping[x] for x in raw]
print(coded)
코드 상세 설명: 서로 다르게 기록된 동일 범주를 숫자 코드로 통일했다. 1과 2의 의미를 반드시 데이터 사전에 기록해야 한다.
[개념 4] 코딩 규칙에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 이름과 범주값을 통일하고 코드 의미를 문서화한다.

5 형식과 단위 통일

비교 가능한 값

개념 상세 설명: 날짜, 시간, 소수점, 단위와 문자 표기가 섞이면 비교와 계산이 어렵다. 날짜 형식을 통일하고, 길이·온도·질량 등의 단위를 하나로 변환하며, 문자열 앞뒤의 불필요한 공백을 제거한다.

temperatures_c = [23.5, 24.0, 25.2]
temperatures_f = [x * 9/5 + 32 for x in temperatures_c]
print("섭씨:", temperatures_c)
print("화씨:", [round(x, 1) for x in temperatures_f])
코드 상세 설명: 섭씨 값을 화씨로 변환한다. 분석에서는 한 변수의 단위를 하나로 통일하고 변환 공식을 기록해야 한다.
[개념 5] 형식과 단위 통일에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ⑤ — 형식과 단위를 통일하고 변환 과정을 남겨야 한다.

6 결측값

비어 있는 이유부터

개념 상세 설명: 결측값은 측정 실패, 무응답, 해당 없음, 입력 누락 등으로 값이 비어 있는 상태이다. 삭제, 대체, 별도 범주 처리 등의 방법이 있지만 원인, 비율, 변수 특성과 연구 목적을 확인한 뒤 결정해야 한다.

values = [18, None, 21, 19, None]
valid = [x for x in values if x is not None]
mean = sum(valid) / len(valid)
filled = [mean if x is None else x for x in values]
print("결측 수:", values.count(None))
print("평균 대체:", filled)
코드 상세 설명: 결측값을 제외해 평균을 구한 뒤 평균으로 대체했다. 평균 대체는 간단하지만 분포를 왜곡할 수 있으므로 모든 상황에 자동 적용하면 안 된다.
[개념 6] 결측값 처리에 대한 옳은 설명 3개를 고르시오.
정답: ②, ④, ⑤ — 결측의 원인과 영향에 따라 처리하고 과정을 문서화한다.

7 이상값

오류인가 현상인가

개념 상세 설명: 이상값은 일반적인 범위에서 크게 벗어난 값이다. 입력 오류, 장비 고장일 수도 있지만 실제로 중요한 사건일 수도 있다. 범위 기준, 사분위수, 표준점수 등을 활용하되 원자료와 상황 기록을 함께 확인해야 한다.

values = [20, 21, 22, 23, 95]
lower, upper = 0, 50
outliers = [x for x in values if x < lower or x > upper]
print("범위 밖 값:", outliers)
코드 상세 설명: 사전에 정한 허용 범위를 벗어난 값을 후보로 찾는다. 후보를 곧바로 삭제하지 말고 단위, 센서 상태, 현장 상황을 확인한다.
[개념 7] 이상값 처리에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 이상값은 근거를 확인하고 민감도까지 점검해야 한다.

8 중복값과 고유 식별자

같은 기록 두 번

개념 상세 설명: 중복값은 같은 관측이 여러 번 기록된 경우이다. 측정 시각, 참여자 ID, 장치 ID처럼 관측을 구분할 고유 식별자를 활용해 중복 여부를 확인한다. 동일 값이라도 실제 반복 측정일 수 있으므로 기준이 필요하다.

ids = ["A01", "A02", "A02", "A03"]
seen = set()
duplicates = []
for x in ids:
    if x in seen:
        duplicates.append(x)
    seen.add(x)
print("중복 ID:", duplicates)
코드 상세 설명: 이미 확인한 ID를 집합에 저장하고 다시 등장한 값을 중복 후보로 찾는다. 실제 삭제 여부는 시간과 다른 변수까지 확인해 결정한다.
[개념 8] 중복 자료 점검에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 중복 후보와 실제 반복 관측을 구분해야 한다.

9 파생 변수와 범주화

원자료에서 새 정보

개념 상세 설명: 파생 변수는 기존 변수로 계산한 새로운 변수이다. 평균, 변화량, 비율, 위험 등급 등을 만들 수 있다. 계산식과 경계 기준을 명확히 기록하고 원자료를 보존해야 한다.

scores = [82, 91, 76]
average = sum(scores) / len(scores)

if average >= 90:
    level = "상"
elif average >= 80:
    level = "중"
else:
    level = "하"

print("평균:", round(average, 1), "수준:", level)
코드 상세 설명: 점수의 평균을 계산한 뒤 기준에 따라 범주를 만들었다. 경계값이 달라지면 결과가 달라지므로 기준의 근거를 설명해야 한다.
[개념 9] 파생 변수와 범주화에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 계산식, 기준, 의미를 명확히 남겨야 한다.

10 전처리 기록과 원자료 보존

재현 가능한 변화

개념 상세 설명: 전처리는 원자료를 직접 덮어쓰기보다 복사본에 적용하고, 각 단계에서 무엇을 왜 변경했는지 기록해야 한다. 코드와 버전, 처리 날짜, 제거된 행 수를 남기면 오류를 되돌리고 분석을 재현할 수 있다.

log = []
log.append("1단계: 변수명 소문자로 통일")
log.append("2단계: 결측값 3개 확인")
log.append("3단계: 중복 행 2개 제거")

for item in log:
    print(item)
코드 상세 설명: 전처리 단계와 결과를 순서대로 기록한다. 실제 작업에서는 원본 파일명, 코드 버전, 처리 시각과 행 수 변화도 함께 남긴다.
[개념 10] 전처리 기록에 대한 옳은 설명 3개를 고르시오.
정답: ①, ④, ⑤ — 원자료 보존과 변경 이력은 재현성의 핵심이다.

1. 교실 환경 센서 자료의 결측값 처리 방법 비교

전처리: 삭제, 앞값 대체, 선형 보간을 적용한다.

탐구 관점: 처리 방법에 따른 평균과 시간 변화 그래프의 차이를 비교한다.

2. 공공 기상자료의 이상값 탐지 기준에 따른 분석 결과 변화

전처리: 물리적 범위, 사분위수 범위, 표준점수 기준을 비교한다.

탐구 관점: 제거 전후 평균과 상관관계의 차이를 분석한다.

3. 설문 범주형 응답의 코딩 규칙이 통계 결과에 미치는 영향

전처리: 동일 의미 응답을 표준 범주로 통합한다.

탐구 관점: 코딩 전후 빈도표와 집단 비교 결과를 살펴본다.

4. 날짜·시간 형식 통일이 시계열 분석 정확도에 미치는 영향

전처리: 서로 다른 날짜 형식과 시간대를 하나의 기준으로 변환한다.

탐구 관점: 정렬 오류와 시간 간격 계산 오류의 감소를 확인한다.

5. 데이터 사전 작성이 협업 분석의 오류 감소에 미치는 영향

전처리: 변수명, 단위, 범주 코드, 결측 규칙을 문서화한다.

탐구 관점: 데이터 사전 제공 전후의 해석 오류와 작업 시간을 비교한다.
