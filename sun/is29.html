“계획은 미래를 정확히 예언하는 일이 아니라, 변화 속에서도 방향을 잃지 않게 하는 지도입니다.”

아리스토텔레스는 목적을 분명히 알아야 올바른 수단을 선택할 수 있다고 보았고, 공자는 일을 시작하기 전에 이름과 역할을 바로 세워야 질서가 생긴다고 말했습니다. 다빈치는 위대한 작품을 만들기 전에 수많은 스케치와 구조도를 남겼고, 가우스는 복잡한 문제를 작은 단계로 나누어 해결했습니다.

오늘 우리는 프로젝트를 바로 코딩하지 않습니다. 누가 사용할지, 어떤 자료가 들어오고 어떤 결과가 나가야 하는지, 무엇을 반드시 만들고 무엇은 이번 범위에서 제외할지 먼저 정합니다. 계획은 창의성을 막는 문서가 아니라 좋은 아이디어가 일정 속에서 실제 결과로 완성되도록 돕는 구조입니다.

다만 역할 분담에서 “기획은 친구, 개발도 친구, 발표도 친구, 나는 응원 담당”이라고 작성하면 협업 알고리즘이 공정성 검사에서 실패합니다. 오늘은 책임을 나누되 꿈은 함께 키우는 실행 가능한 계획을 세워 봅시다.

    current = 0
    waits = []

    for service in service_times:
        waits.append(current)
        current += service

    return sum(waits) / len(waits)

from collections import deque

def shortest_path(graph, start, target):
    queue = deque([(start, [start])])
    visited = {start}

    while queue:
        node, path = queue.popleft()

        if node == target:
            return path

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [neighbor]))

    return None

students = ["민준", "서연", "지우"]
selected = []

def arrange():
    if len(selected) == len(students):
        print(selected)
        return

    for student in students:
        if student in selected:
            continue

        selected.append(student)
        arrange()
        selected.pop()

arrange()

Ⅶ. 수업 끝 멘트

시는 한 편을 쓰기 전에 마음속에 리듬과 이미지를 세우고, 소설은 인물과 사건의 흐름을 설계하며 긴 이야기를 완성합니다. 희곡은 막과 장, 등장인물의 역할이 분명할 때 갈등이 살아나고, 연극은 배우·조명·음향·무대가 각자의 시간을 지킬 때 하나의 공연이 됩니다. 음악도 각 악기의 역할과 들어올 순간이 정확해야 아름다운 합주가 됩니다.

여러분의 꿈도 막연한 희망만으로 이루어지지 않습니다. 오늘 해야 할 작은 기능, 이번 주에 끝낼 산출물, 함께할 사람의 역할, 실패했을 때 고칠 기준을 정하면 꿈은 조금씩 실행 가능한 계획이 됩니다. 계획이 바뀌는 것은 실패가 아니라 더 나은 정보를 반영하는 성장입니다.

그리고 일정표에 모든 작업을 마지막 날로 배치하면 병렬 처리처럼 보여도 실제로는 단일 코어 밤샘 작업이 됩니다. 오늘 만든 계획이 여러분의 아이디어를 지키고, 협업을 즐겁게 하며, 끝까지 완성하는 힘이 되기를 바랍니다.
