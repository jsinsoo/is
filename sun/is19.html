  • 탐욕 선택 속성: 탐욕 선택을 포함하는 최적해가 존재하는 성질
  • 최적 부분 구조: 부분 문제의 최적해가 전체 최적해를 이루는 구조

탐욕 알고리즘은 현재의 최선 선택을 되돌리지 않고 반복하며, 전체 최적해를 보장하려면 선택의 안전성과 최적 부분 구조가 논리적으로 검증되어야 한다.

Ⅲ. 수업 시작 멘트

“아리스토텔레스는 좋은 선택은 목적에 맞는 선택이라고 했고, 공자는 눈앞의 이익보다 더 큰 질서를 살펴야 한다고 보았습니다. 다윈은 환경마다 유리한 선택이 달라진다는 사실을 보여 주었고, 가우스는 가장 좋은 규칙 하나가 수많은 계산을 줄일 수 있음을 알려 주었습니다. 오늘 우리는 매 순간 가장 좋아 보이는 선택을 합니다. 그러나 가장 큰 동전을 먼저 고른다고 언제나 가장 적은 동전이 되는 것은 아닙니다. 급식에서 가장 큰 반찬부터 고르는 전략도 전체 만족도를 보장하지 않을 수 있습니다. 이제 지역 최적 선택이 전체 최적해로 이어지는 조건을 코드와 반례로 확인해 봅시다.”

예제 3

동적 계획법과 비교하여 동전 체계 검증

DP 비교최적성 검증
def greedy_coin_count(coins, amount):
    count = 0

    for coin in sorted(coins, reverse=True):
        count += amount // coin
        amount %= coin

    return count if amount == 0 else float("inf")

def dp_coin_count(coins, amount):
    dp = [float("inf")] * (amount + 1)
    dp[0] = 0

    for value in range(1, amount + 1):
        for coin in coins:
            if coin <= value:
                dp[value] = min(
                    dp[value],
                    dp[value - coin] + 1
                )

    return dp[amount]

coin_sets = [
    [500, 100, 50, 10],
    [4, 3, 1],
    [10, 6, 1]
]

for coins in coin_sets:
    for amount in range(1, 21):
        greedy = greedy_coin_count(coins, amount)
        optimal = dp_coin_count(coins, amount)

        if greedy != optimal:
            print(
                "반례:",
                coins,
                "금액:", amount,
                "탐욕:", greedy,
                "최적:", optimal
            )
            break
탐구 방법
작은 금액 범위에서 탐욕 결과와 동적 계획법의 최적 결과를 비교하면 해당 동전 체계의 반례를 찾을 수 있습니다.

예제 5

회의실 배정 정렬 기준 비교

시작 시각짧은 길이종료 시각
meetings = [
    (1, 10),
    (2, 3),
    (3, 4),
    (4, 5),
    (5, 6)
]

def select_meetings(meetings, key):
    ordered = sorted(meetings, key=key)

    selected = []
    last_end = -1

    for start, end in ordered:
        if start >= last_end:
            selected.append((start, end))
            last_end = end

    return selected

by_start = select_meetings(
    meetings,
    key=lambda item: item[0]
)

by_length = select_meetings(
    meetings,
    key=lambda item: item[1] - item[0]
)

by_end = select_meetings(
    meetings,
    key=lambda item: item[1]
)

print("시작 시각 기준:", by_start)
print("짧은 회의 기준:", by_length)
print("종료 시각 기준:", by_end)

예제 7

분할 가능한 배낭: 가치 밀도 기준 선택

분할 배낭가치 밀도
items = [
    ("A", 10, 60),
    ("B", 20, 100),
    ("C", 30, 120)
]

capacity = 50

items.sort(
    key=lambda item: item[2] / item[1],
    reverse=True
)

total_value = 0
selected = []

for name, weight, value in items:
    if capacity == 0:
        break

    take = min(weight, capacity)
    ratio = take / weight

    total_value += value * ratio
    selected.append((name, take, value * ratio))
    capacity -= take

print("선택:", selected)
print("총 가치:", total_value)
최적 보장 조건
물건을 쪼갤 수 있으므로 단위 무게당 가치가 큰 부분부터 채우는 선택이 전체 최적해로 이어집니다.

예제 8

0/1 배낭에서 가치 밀도 탐욕법의 실패

0/1 배낭반례

탐욕 선택

items = [
    ("A", 10, 60),
    ("B", 20, 100),
    ("C", 30, 120)
]

capacity = 50

ordered = sorted(
    items,
    key=lambda x: x[2] / x[1],
    reverse=True
)

value = 0
chosen = []

for item in ordered:
    name, weight, item_value = item

    if weight <= capacity:
        capacity -= weight
        value += item_value
        chosen.append(name)

print(chosen, value)

최적 선택

실패 이유
물건을 쪼갤 수 없으면 현재 가치 밀도가 높은 물건을 선택한 결과가 이후 조합을 제한할 수 있습니다. 0/1 배낭은 일반적으로 동적 계획법이 필요합니다.

예제 9

작업 마감시간과 최대 이익 스케줄링

작업 스케줄이익 내림차순
jobs = [
    ("A", 2, 100),
    ("B", 1, 19),
    ("C", 2, 27),
    ("D", 1, 25),
    ("E", 3, 15)
]

jobs.sort(
    key=lambda job: job[2],
    reverse=True
)

max_deadline = max(job[1] for job in jobs)
slots = [None] * max_deadline
total_profit = 0

for name, deadline, profit in jobs:
    for slot in range(
        min(deadline, max_deadline) - 1,
        -1,
        -1
    ):
        if slots[slot] is None:
            slots[slot] = name
            total_profit += profit
            break

print("작업 배치:", slots)
print("총 이익:", total_profit)
지역 선택
이익이 큰 작업부터 고려하되, 다른 작업의 기회를 최대한 남기기 위해 마감시간 안의 가장 늦은 빈 슬롯에 배치합니다.

def explain_greedy(problem):
    explanations = {
        "회의실 배정": {
            "선택": "가장 빨리 끝나는 회의",
            "성공": True,
            "근거": (

                "다른 최적해의 첫 회의와 교환해도 "
                "해의 크기가 줄지 않습니다."
            )
        },
        "표준 동전 교환": {
            "선택": "가장 큰 동전",
            "성공": "동전 체계에 따라 다름",
            "근거": (
                "정규 동전 체계에서는 성공할 수 있지만 "
                "[4, 3, 1]과 같은 체계에서는 실패합니다."
            )
        },
        "분할 배낭": {
            "선택": "가치 밀도가 가장 큰 물건",
            "성공": True,
            "근거": (
                "물건을 나눌 수 있어 더 낮은 가치 밀도 "
                "부분과 교환하면 가치가 감소합니다."
            )
        },
        "0/1 배낭": {
            "선택": "가치 밀도가 가장 큰 물건",
            "성공": False,
            "근거": (
                "물건을 나눌 수 없어 현재 선택이 "
                "더 좋은 조합을 막을 수 있습니다."
            )
        }
    }

    return explanations.get(
        problem,
        {"선택": None, "성공": None, "근거": "분석 필요"}
    )

for problem in [
    "회의실 배정",
    "표준 동전 교환",
    "분할 배낭",
    "0/1 배낭"
]:
    result = explain_greedy(problem)

    print("\n문제:", problem)
