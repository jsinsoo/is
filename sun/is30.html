Ⅲ. 수업 시작 멘트

“배움은 다른 사람의 지식을 빌리는 일이지만, 그 지식의 주인을 잊지 않는 태도에서 품격이 생깁니다.”

공자는 배운 것을 익히고 다른 사람의 지혜를 존중해야 한다고 했고, 아리스토텔레스는 지식이 공동체 안에서 축적된다고 보았습니다. 뉴턴은 자신이 더 멀리 본 것은 거인의 어깨 위에 섰기 때문이라고 표현했고, 가우스는 앞선 수학자들의 성과를 바탕으로 새로운 질서를 만들었습니다.

오늘 우리는 다른 사람이 만든 코드와 라이브러리, 데이터와 이미지를 사용합니다. 그러나 ‘무료로 내려받았다’는 사실과 ‘아무 조건 없이 써도 된다’는 말은 다릅니다. 누구의 자료인지, 어떤 조건으로 사용할 수 있는지, 수정했다면 무엇을 바꾸었는지 정직하게 기록해야 합니다.

다만 인터넷에서 코드를 복사한 뒤 주석에 “출처: 인터넷”이라고 쓰면 검색 엔진 전체가 공동 저자가 되는 이상한 상황이 생깁니다. 오늘은 출처를 정확히 밝히고, 필요한 도구를 현명하게 선택하며, 책임 있는 개발자의 첫걸음을 내딛어 봅시다.

def make_attribution(
    title,
    author,
    source,
    license_name,
    modified=False
):
    text = (
        f"자료명: {title}\n"
        f"저작자: {author}\n"
        f"출처: {source}\n"
        f"라이선스: {license_name}\n"
    )

    if modified:
        text += "변경 사항: 원본을 수정하여 사용함\n"
    else:
        text += "변경 사항: 없음\n"

    return text

print(make_attribution(
    title="기후 데이터",
    author="데이터 제공 기관",
    source="공공데이터 포털",
    license_name="해당 페이지의 이용 조건 확인",
    modified=True
))

예제 5

csv 표준 라이브러리로 센서 자료 읽기

csv데이터 처리
import csv
from io import StringIO

sample = """time,co2
09:00,720
09:10,860
09:20,1040
09:30,1280
"""

file_like = StringIO(sample)
reader = csv.DictReader(file_like)

values = []

for row in reader:
    value = int(row["co2"])
    values.append(value)

    print(
        row["time"],
        value,
        "환기 필요" if value >= 1000 else "정상"
    )

print("평균:", sum(values) / len(values))

import platform
import sys
from importlib.metadata import version, PackageNotFoundError

print("운영체제:", platform.platform())
print("파이썬:", sys.version)

packages = [
    "pandas",
    "matplotlib",
    "numpy"
]

for package in packages:
    try:
        print(
            package,
            version(package)
        )
    except PackageNotFoundError:

Ⅶ. 수업 끝 멘트

시는 오래된 단어를 빌려 새로운 감정을 만들고, 소설은 앞선 이야기의 전통 위에 새로운 인물을 세웁니다. 희곡은 고전의 장면을 다시 해석하고, 연극은 수많은 배우와 기술자의 이름이 함께 올라갈 때 비로소 완성됩니다. 음악도 한 곡의 아름다움 뒤에는 작곡가·연주자·편곡자·제작자의 노력이 함께 있습니다.

여러분의 프로젝트도 혼자 만든 것처럼 보여도 많은 사람의 지식과 도구 위에 서 있습니다. 출처를 밝히는 일은 자신의 창의성을 줄이는 것이 아니라, 배움의 연결을 정직하게 보여 주는 일입니다. 올바른 라이선스 확인과 환경 기록은 다른 사람도 여러분의 결과를 믿고 이어 갈 수 있게 합니다.

그리고 라이브러리 이름을 전부 보고서에 적었다고 라이선스 의무가 자동으로 충족되는 것은 아닙니다. 책 제목만 외우고 내용을 읽지 않은 것과 비슷합니다. 오늘의 정직한 기록과 책임 있는 선택이 여러분의 코드뿐 아니라 꿈의 신뢰도까지 높여 주기를 바랍니다.
