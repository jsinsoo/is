예제 2

O(log n): 이진 탐색 비교 횟수

로그 시간범위 절반
def binary_search_count(data, target):
    left = 0
    right = len(data) - 1
    count = 0

    while left <= right:
        count += 1
        middle = (left + right) // 2

        if data[middle] == target:
            return middle, count

        if data[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

    return -1, count

for n in [8, 16, 32, 64, 128, 256]:
    data = list(range(n))
    _, count = binary_search_count(
        data, n - 1
    )

    print(f"n={n:3} | 비교={count}")
입력 크기 관계
입력이 8→16→32처럼 두 배가 되어도 최악의 비교 횟수는 대체로 한 번씩만 증가합니다.

def linear_search_count(data, target):
    count = 0

        count += 1

예제 8

같은 입력에서 실제 핵심 연산 횟수 측정

실험이론 검증
def operation_counts(n):
    # O(1)
    constant = 1

    # O(log n)
    value = n
    logarithmic = 0

    while value > 1:
        value //= 2
        logarithmic += 1

    # O(n)
    linear = 0
    for _ in range(n):
        linear += 1

    # O(n²)
    quadratic = 0
    for _ in range(n):
        for _ in range(n):
            quadratic += 1

    return {
        "O(1)": constant,
        "O(log n)": logarithmic,
        "O(n)": linear,
        "O(n²)": quadratic
    }

for n in [4, 8, 16, 32]:
    print(n, operation_counts(n))
결과
반복문의 실제 카운터 값이 이론식 1, log₂n, n, n²과 같은 증가 경향을 보이는지 확인합니다.

예제 9

입력을 두 배로 늘렸을 때 증가 비율

성장 비율2n / n
import math

def growth_ratio(n):
    functions = {
        "O(1)": lambda x: 1,
        "O(log n)": lambda x: math.log2(x),
        "O(n)": lambda x: x,
        "O(n²)": lambda x: x ** 2
    }

    for name, function in functions.items():
        before = function(n)
        after = function(2 * n)
        ratio = after / before

        print(
            f"{name:8} | "
            f"n={before:.2f} | "
            f"2n={after:.2f} | "
            f"비율={ratio:.2f}"
        )

여러분의 꿈도 무조건 많은 일을 하는 것으로 완성되지 않습니다. 같은 시간을 쓰더라도 더 의미 있는 선택을 하고, 반복되는 실수는 줄이며, 중요한 일에 집중할 때 삶의 시간 복잡도는 더 아름다워집니다. 오늘의 작은 습관 하나가 O(1)처럼 꾸준한 힘이 되고, 매일의 배움이 O(n)처럼 차곡차곡 쌓이기를 바랍니다.

다만 시험 공부를 O(log n)으로 줄이겠다며 교과서를 절반씩 버리면 탐색 범위는 줄어도 점수까지 줄어들 수 있습니다. 오늘 배운 분석의 눈으로 여러분의 코드와 시간, 그리고 꿈의 경로를 더 현명하게 설계해 보세요.
