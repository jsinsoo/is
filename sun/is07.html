Ⅱ. 학습 목표와 학습할 개념

학습 목표

  1. 문제의 상태와 이전 상태의 관계를 찾는다.
  2. 재귀관계와 초기값을 수식과 코드로 표현한다.
  3. 반복문과 재귀함수로 같은 문제를 구현한다.
  4. 가독성·메모리·실행 시간을 비교한다.
  5. 직접 공식, 반복, 재귀 중 적절한 방법을 선택한다.

학습할 개념

  • 상태: 문제의 한 단계 또는 n번째 값을 나타내는 정보
  • 재귀관계: 현재 상태를 이전 상태로 나타낸 관계
  • 초기값: 더 이상 이전 상태로 나누지 않는 가장 작은 값
  • 상태 갱신: 반복문에서 이전 값을 다음 값으로 바꾸는 과정
  • 논리적 비교: 표현, 메모리, 시간의 근거로 차이를 설명하는 과정
개념 일반화
재귀관계는 문제의 변화 구조를 드러내며, 같은 관계는 재귀 호출이나 반복적 상태 갱신으로 구현할 수 있다. 좋은 선택은 초기값, 입력 크기, 직접 공식, 시간과 메모리를 함께 고려한다.

예제 1

1부터 n까지의 자연수 합

누적 합기초 재귀관계
재귀관계: S(n) = n + S(n-1)
초기값: S(1) = 1

반복문

def sum_loop(n):
    total = 0
    for value in range(1, n + 1):
        total += value
    return total

재귀함수

def sum_recursive(n):
    if n == 1:
        return 1
    return n + sum_recursive(n - 1)
차이 설명
재귀 방식은 수학적 관계를 그대로 표현하지만 n번의 함수 호출이 필요합니다. 반복 방식은 누적 변수 하나를 갱신하여 메모리 사용이 적습니다.

예제 3

등차수열의 n번째 항

def arithmetic_loop(first, d, n):
    value = first
    for _ in range(2, n + 1):

    return value

def arithmetic_recursive(first, d, n):

예제 4

피보나치 수열

두 이전 항중복 호출
재귀관계: F(n) = F(n-1) + F(n-2)
초기값: F(0) = 0, F(1) = 1

반복문

def fibonacci_loop(n):
    if n < 2:
        return n

    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b

재귀함수

def fibonacci_recursive(n):

예제 5

1칸 또는 2칸 계단 오르기

경우의 수동적 계획법
재귀관계: W(n) = W(n-1) + W(n-2)
초기값: W(0) = 1, W(1) = 1

반복문

def stairs_loop(n):
    if n <= 1:
        return 1

    prev2, prev1 = 1, 1
    for _ in range(2, n + 1):
        current = prev1 + prev2
        prev2, prev1 = prev1, current
    return prev1

재귀함수

def stairs_recursive(n):
    if n <= 1:
        return 1
    return (
        stairs_recursive(n - 1)
        + stairs_recursive(n - 2)
    )
재귀관계 해석
마지막 이동이 1칸인 경우와 2칸인 경우를 나누면 두 개의 더 작은 계단 문제로 분해됩니다.

예제 6

2×n 직사각형을 1×2 타일로 채우기

타일링분해
재귀관계: T(n) = T(n-1) + T(n-2)
초기값: T(1) = 1, T(2) = 2

반복문

def tiling_loop(n):
    if n == 1:
        return 1

    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b

재귀함수

def tiling_recursive(n):

    )

예제 7

유클리드 호제법 최대공약수

나머지상태 축소
재귀관계: G(a, b) = G(b, a mod b)
초기값: G(a, 0) = a

반복문

def gcd_loop(a, b):
    while b != 0:
        a, b = b, a % b
    return a

재귀함수

def gcd_recursive(a, b):
    if b == 0:
        return a
    return gcd_recursive(b, a % b)

두 방식 모두 같은 재귀관계를 사용합니다. 재귀 코드는 관계를 직접 드러내고, 반복 코드는 함수 호출 없이 상태를 갱신합니다.

예제 8

거듭제곱

지수 감소분할 가능
기본 재귀관계: P(a, n) = a × P(a, n-1)
초기값: P(a, 0) = 1

반복문

def power_loop(base, exponent):
    result = 1
    for _ in range(exponent):
        result *= base
    return result

재귀함수

def power_recursive(base, exponent):
    if exponent == 0:
        return 1
    return base * power_recursive(
        base, exponent - 1
    )
확장
지수가 짝수일 때 절반으로 나누는 재귀관계를 사용하면 호출 깊이를 크게 줄이는 빠른 거듭제곱으로 개선할 수 있습니다.

예제 9

정수의 자릿수 합

정수 분해나머지
재귀관계: D(n) = n mod 10 + D(n div 10)
초기값: n < 10이면 D(n) = n

반복문

def digit_sum_loop(n):
    total = 0
    while n > 0:
        total += n % 10
        n //= 10
    return total

재귀함수

def digit_sum_recursive(n):
    if n < 10:
        return n
    return (
        n % 10
        + digit_sum_recursive(n // 10)
    )
가독성 비교
재귀 코드는 “마지막 자리와 나머지 자리”라는 분해를 명확히 나타냅니다. 반복 코드는 변수 변화가 직접 보여 실행 과정을 추적하기 쉽습니다.

예제 10

실행 시간·호출 수 종합 비교

성능 측정논리적 설명
import time

recursive_calls = 0

def fibonacci_recursive(n):
    global recursive_calls
    recursive_calls += 1

    if n < 2:
        return n

    return (
        fibonacci_recursive(n - 1)
        + fibonacci_recursive(n - 2)
    )

def fibonacci_loop(n):
    if n < 2:
        return n

    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b

n = 30

start = time.perf_counter()
r1 = fibonacci_recursive(n)
recursive_time = time.perf_counter() - start

start = time.perf_counter()
r2 = fibonacci_loop(n)
loop_time = time.perf_counter() - start

print("결과 일치:", r1 == r2)
print("재귀 호출 수:", recursive_calls)
print(f"재귀 시간: {recursive_time:.6f}초")
print(f"반복 시간: {loop_time:.6f}초")

“시는 앞 구절의 울림이 다음 구절을 만들고, 소설은 지난 장면의 선택이 다음 장면의 운명을 바꿉니다. 희곡은 앞선 대사의 의미를 받아 다음 대사가 완성되고, 연극은 어제의 연습이 오늘의 무대를 조금 더 단단하게 합니다. 음악도 앞 음의 긴장이 다음 화음에서 풀리며 하나의 곡이 됩니다. 여러분의 오늘 역시 어제와 무관한 값이 아닙니다. 작은 노력 하나가 다음 상태를 바꾸고, 그 변화가 쌓여 미래라는 수열을 만듭니다. 단, 성적이 떨어졌다고 ‘초기값이 틀렸다’며 출생부터 다시 시작할 수는 없습니다. 지금의 상태를 정확히 보고 다음 항을 더 좋게 계산해 봅시다.”
