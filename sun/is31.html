Ⅲ. 수업 시작 멘트

“함께 만든다는 것은 일을 나누는 데서 끝나지 않고, 서로의 생각이 만나는 경계를 정성껏 설계하는 일입니다.”

아리스토텔레스는 전체가 단순한 부분의 합을 넘어선다고 보았고, 공자는 이름과 역할이 바로 서야 공동체의 질서가 생긴다고 말했습니다. 뉴턴과 라이프니츠는 서로 다른 표현으로 미적분의 길을 열었지만, 수학은 공통된 기호와 정의를 통해 발전했습니다. 과학도 연구자마다 실험은 달라도 측정 단위와 기록 방식이 같을 때 지식을 연결할 수 있습니다.

오늘 우리는 입력·분석·출력 기능을 각자의 모듈로 나눕니다. 그러나 함수 이름, 매개변수, 반환값과 오류 규칙이 맞지 않으면 아무리 훌륭한 코드도 서로 대화할 수 없습니다. 좋은 인터페이스는 코드 사이의 언어이며, 단위 테스트와 코드 리뷰는 서로의 작업을 의심하는 절차가 아니라 함께 실패를 줄이는 안전망입니다.

다만 역할 분담에서 “나는 main.py 담당이니 마지막에 친구 코드를 전부 복사해 붙이겠습니다”라고 하면 통합이 아니라 코드 볶음밥이 될 수 있습니다. 오늘은 각자의 개성을 살리되 공통된 약속으로 아름답게 연결해 봅시다.

def classify_co2(value: float) -> str:
    """
    이산화탄소 농도를 상태 문자열로 변환한다.

    매개변수:

# data_io.py

def parse_values(text: str) -> list[float]:
    values = []

    for item in text.split(","):
        item = item.strip()

        if not item:
            continue

        value = float(item)

        if value < 0:
            raise ValueError(
                "음수 데이터는 허용하지 않습니다."
            )

        values.append(value)

    if not values:
        raise ValueError(
            "입력 데이터가 없습니다."
        )

    return values
책임
입력 모듈은 문자열을 숫자 목록으로 변환하고 잘못된 값만 검증합니다. 평균 계산이나 상태 판정은 다른 모듈에 맡깁니다.

예제 4

analyzer.py순수 함수
# analyzer.py

from statistics import mean

def summarize(
    values: list[float]
) -> dict[str, float]:
    if not values:
        raise ValueError(
            "분석할 값이 없습니다."
        )

    return {
        "평균": mean(values),
        "최댓값": max(values),
        "최솟값": min(values)
    }

def count_warning(
    values: list[float],
    threshold: float = 1000
) -> int:
    return sum(
        1
        for value in values
        if value >= threshold
    )
순수 함수
외부 파일이나 전역 변수에 의존하지 않고 입력만으로 결과를 만들면 테스트와 재사용이 쉬워집니다.

# reporter.py

def make_report(
    summary: dict[str, float],
    warning_count: int
) -> str:
    lines = [
        "[분석 결과]",
        f"평균: {summary['평균']:.1f}",
        f"최댓값: {summary['최댓값']:.1f}",
        f"최솟값: {summary['최솟값']:.1f}",
        f"경고 횟수: {warning_count}"
    ]

    return "\n".join(lines)

def safe_execute(text: str) -> str:
    try:
        values = parse_values(text)
        summary = summarize(values)
        warnings = count_warning(values)

        return make_report(
            summary,
            warnings
        )

    except ValueError as error:
        return (
            "[입력 오류]\n"
            + str(error)
        )

print(safe_execute("720, 860, 1040"))
print()
print(safe_execute("720, -1, 900"))
규칙

예제 7

단위 테스트로 인터페이스 검증

unittest경계값
import unittest

class TestAnalyzer(unittest.TestCase):
    def test_summary(self):
        result = summarize(
            [10, 20, 30]
        )

        self.assertEqual(
            result["평균"],
            20
        )

        self.assertEqual(
            result["최댓값"],
            30
        )

    def test_warning_count(self):
        self.assertEqual(
            count_warning(
                [900, 1000, 1200]
            ),
            2
        )

    def test_empty_values(self):
        with self.assertRaises(ValueError):
            summarize([])

if __name__ == "__main__":
    unittest.main()
검증 기준
정상 입력뿐 아니라 임계값 1000, 빈 목록, 음수 입력처럼 경계값과 오류 상황도 검사합니다.

review = {
    "함수 이름이 역할을 설명하는가": True,
    "타입 힌트가 있는가": True,
    "중복 코드가 제거되었는가": False,
    "예외 처리가 일관적인가": True,
    "테스트가 포함되었는가": True,
    "출처와 라이선스가 기록되었는가": True
}

failed = []

for item, passed in review.items():
    status = (
        "통과"
        if passed
        else "보완"
    )

    print(item, ":", status)

    if not passed:
        failed.append(item)

print("보완 항목 수:", len(failed))

# main.py

from dataclasses import dataclass

@dataclass
class AppConfig:
    warning_threshold: float = 1000
    version: str = "1.0.0"

def run(
    text: str,
    config: AppConfig
) -> str:
    values = parse_values(text)

    summary = summarize(values)

    warning_count = count_warning(
        values,
        config.warning_threshold
    )

    report = make_report(
        summary,
        warning_count
    )

    return (
        f"버전: {config.version}\n"
        + report
    )

if __name__ == "__main__":
    config = AppConfig(
        warning_threshold=1000,
        version="1.0.0"
    )

    sample = "720, 860, 1040, 1280"
    print(run(sample, config))

Ⅶ. 수업 끝 멘트

시는 서로 다른 단어가 정확한 자리에 놓일 때 한 편의 마음이 되고, 소설은 각 인물의 독립된 삶이 하나의 서사로 연결될 때 깊이를 얻습니다. 희곡은 배우마다 다른 역할을 맡지만 대사와 장면의 약속을 공유하고, 연극은 무대·조명·음향·배우가 서로의 신호를 믿을 때 비로소 살아납니다. 음악도 독주가 아름답지만 합주는 서로의 박자와 쉼을 존중할 때 더 큰 감동을 만듭니다.

여러분의 프로젝트도 모든 사람이 같은 코드를 쓰는 것이 협업은 아닙니다. 각자의 강점을 살려 다른 모듈을 만들고, 공통된 인터페이스와 테스트로 연결하며, 리뷰를 통해 서로의 생각을 더 나은 형태로 다듬는 것이 진짜 협업입니다. 충돌은 실패가 아니라 두 변화가 만났다는 신호이며, 기록과 대화로 해결할 수 있습니다.

그리고 Git 커밋 메시지를 매번 “최종”, “진짜 최종”, “진짜진짜 최종”로 작성하면 버전 관리는 되지만 역사학자가 필요해질 수 있습니다. 오늘 배운 작은 약속과 정직한 기록이 여러분의 코드와 관계, 그리고 큰 꿈을 안전하게 통합해 주기를 바랍니다.
