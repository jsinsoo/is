Ⅲ. 수업 시작 멘트

“한 계단을 오르는 일은 작아 보이지만, 그 한 걸음이 다음 가능성의 초기값이 됩니다.”

아리스토텔레스는 큰 목적도 작은 행동의 반복으로 이루어진다고 보았고, 공자는 천 리 길도 한 걸음에서 시작된다고 말했습니다. 갈릴레이는 작은 운동을 관찰해 자연의 법칙을 찾았고, 파스칼은 작은 수들의 관계가 커다란 수학적 구조를 만든다는 사실을 보여 주었습니다.

오늘 우리는 계단을 오르는 방법을 코드로 생각합니다. 한 번에 1칸 또는 2칸을 오른다는 단순한 규칙 속에서 상태·초기값·점화식이라는 중요한 설계 원리를 발견합니다. 지금 서 있는 계단은 이전 두 계단의 가능성이 모여 만들어진 결과이며, 우리가 오늘 내딛는 한 걸음도 내일의 선택을 넓혀 줍니다.

다만 실제 학교 계단에서 1·2·3칸 점프 알고리즘을 실험하면 체육 선생님이 예외 처리를 실행할 수 있습니다. 오늘은 몸이 아니라 코드로 안전하게 오르며, 작은 상태를 큰 해결로 연결해 봅시다.

def stairs_table(n):
    dp = [0] * (n + 1)
    dp[0] = 1

def stairs_optimized(n):
    if n <= 1:
        return 1

    previous2 = 1  # dp[0]
    previous1 = 1  # dp[1]

    for stair in range(2, n + 1):
        current = previous1 + previous2

        previous2 = previous1
        previous1 = current

        print(
            f"{stair}칸: "
            f"이전2={previous2}, "
            f"현재={previous1}"
        )

    return previous1

print("결과:", stairs_optimized(10))

예제 9

밟을 수 없는 계단이 있는 경우

제약 조건도달 불가 상태
def stairs_with_blocked(n, blocked):
    dp = [0] * (n + 1)
    dp[0] = 1

    for stair in range(1, n + 1):
        if stair in blocked:
            dp[stair] = 0
            continue

        if stair >= 1:
            dp[stair] += dp[stair - 1]

        if stair >= 2:
            dp[stair] += dp[stair - 2]

    return dp

blocked = {3, 7}
dp = stairs_with_blocked(10, blocked)

for stair, count in enumerate(dp):
    state = "금지" if stair in blocked else "가능"
    print(stair, state, count)
상태 처리
금지된 계단은 도착 방법의 수를 0으로 설정합니다. 이후 계단의 점화식은 이 0을 자연스럽게 반영합니다.

예제 10

계단 오르기 최소 비용 상태 설계

최솟값최적 부분 구조경로 복원
def minimum_cost_stairs(cost):
    n = len(cost)

    dp = [0] * n
    parent = [-1] * n

    dp[0] = cost[0]

    if n >= 2:
        dp[1] = cost[1]

    for stair in range(2, n):
        if dp[stair - 1] <= dp[stair - 2]:
            dp[stair] = (
                dp[stair - 1]
                + cost[stair]
            )
            parent[stair] = stair - 1
        else:
            dp[stair] = (
                dp[stair - 2]
                + cost[stair]
            )
            parent[stair] = stair - 2

    # 마지막 계단에서 끝나는 것으로 가정
    path = []
    current = n - 1

    while current >= 0:
        path.append(current)
        current = parent[current]

    path.reverse()

    return dp[-1], path, dp

cost = [3, 5, 2, 6, 1, 4]

minimum, path, table = (
    minimum_cost_stairs(cost)
)

print("비용:", cost)
print("최소 누적 비용:", minimum)
print("선택 계단:", path)
print("DP 테이블:", table)
설계 요소최소 비용 계단 문제
상태dp[i] = i번째 계단에 도착하는 최소 누적 비용
마지막 이동i-1에서 1칸 또는 i-2에서 2칸 이동
점화식dp[i] = cost[i] + min(dp[i-1], dp[i-2])
초기값dp[0]=cost[0], dp[1]=cost[1]
경로 복원최솟값을 만든 이전 계단을 parent에 저장
알고리즘 설명 예시
“상태 dp[i]를 i번째 계단에 도착하는 최소 누적 비용으로 정의하였다. i번째 계단에는 i-1번째 또는 i-2번째 계단에서 올 수 있으므로, 두 이전 상태 중 비용이 작은 값을 선택하고 현재 계단 비용을 더하였다. 따라서 점화식은 dp[i]=cost[i]+min(dp[i-1], dp[i-2])로 설계하였다.”
