Ⅲ. 수업 시작 멘트

“배움은 매번 처음부터 시작하는 일이 아니라, 이전의 이해를 다음 생각의 디딤돌로 삼는 일입니다.”

소크라테스는 질문을 반복했지만 같은 자리에서 맴돌지 않았고, 아리스토텔레스는 이전의 경험이 다음 판단의 토대가 된다고 보았습니다. 파블로프는 반복 속에서 학습의 연결을 관찰했고, 파스칼은 작은 수의 관계가 큰 수의 구조를 만든다는 사실을 보여 주었습니다.

오늘 우리는 같은 계산을 다시 하지 않습니다. 한 번 구한 답을 기억하고, 그 답을 더 큰 문제의 재료로 사용합니다. 피보나치 수열은 작은 두 값이 다음 값을 만들고, 계단 오르기는 이전 두 칸의 방법이 새로운 가능성을 만듭니다. 동적 계획법은 ‘기억하는 계산’이자 ‘연결하는 사고’입니다.

다만 시험 공부에서 어제 틀린 문제를 저장만 하고 다시 확인하지 않으면 memo에는 들어 있지만 정답으로 반환되지 않을 수 있습니다. 오늘은 좋은 상태를 정의하고, 필요한 값을 정확히 저장하며, 작은 성공을 큰 해결로 이어 가봅시다.

예제 1

피보나치에서 중복 부분 문제 확인

호출 횟수중복 상태
calls = {}

def fibonacci(n):
    calls[n] = calls.get(n, 0) + 1

    if n < 2:
        return n

    return (
        fibonacci(n - 1)
        + fibonacci(n - 2)
    )

print("결과:", fibonacci(6))
print("입력별 호출 횟수:")

for n in sorted(calls):
    print(n, ":", calls[n])

중복 구조

F(6)
├─ F(5)
│  ├─ F(4)
│  └─ F(3)
└─ F(4)
   ├─ F(3)
   └─ F(2)

F(4), F(3), F(2) 등이 여러 경로에서 반복됩니다.

같은 n에 대한 결과가 항상 같으므로 한 번 계산한 값을 저장해 재사용할 수 있습니다.

예제 4

피보나치 테이블 공간 최적화

O(1) 공간이전 두 상태
def fibonacci_optimized(n):
    if n < 2:
        return n

    previous2 = 0
    previous1 = 1

    for _ in range(2, n + 1):
        current = previous1 + previous2
        previous2, previous1 = (
            previous1,
            current
        )

    return previous1

print(fibonacci_optimized(100))
공간 분석
F(n)을 구할 때 F(n-1)과 F(n-2)만 필요하므로 전체 테이블 대신 두 개의 이전 값만 저장할 수 있습니다.

memo = {0: 1, 1: 1}

def stairs_table(n):
    dp = [0] * (n + 1)
    dp[0] = 1

    if n >= 1:
        dp[1] = 1

    for step in range(2, n + 1):
        dp[step] = (
            dp[step - 1]
            + dp[step - 2]
        )

    return dp[n]

print(stairs_table(10))

예제 6

격자 경로 수: 2차원 상태

2차원 DP행·열 상태
def count_paths(rows, cols):
    dp = [
        [0] * cols
        for _ in range(rows)
    ]

    for row in range(rows):
        dp[row][0] = 1

    for col in range(cols):
        dp[0][col] = 1

    for row in range(1, rows):
        for col in range(1, cols):
            dp[row][col] = (
                dp[row - 1][col]
                + dp[row][col - 1]
            )

    for line in dp:
        print(line)

    return dp[rows - 1][cols - 1]

print("경로 수:", count_paths(4, 5))
상태 전이
현재 칸으로 오는 방법은 위쪽 칸에서 내려오는 방법과 왼쪽 칸에서 오른쪽으로 오는 방법의 합입니다.

예제 7

최소 동전 수: 최적 부분 구조

최소화동전 교환
def min_coin_table(coins, amount):
    dp = [float("inf")] * (amount + 1)
    dp[0] = 0

    for value in range(1, amount + 1):
        for coin in coins:
            if coin <= value:
                dp[value] = min(
                    dp[value],
                    dp[value - coin] + 1
                )

    return (
        dp[amount]
        if dp[amount] != float("inf")
        else -1
    )

print(min_coin_table([1, 3, 4], 6))

점화 관계

dp[x] =
min(
 dp[x-1] + 1,
 dp[x-3] + 1,
 dp[x-4] + 1
)

마지막에 사용한 동전을 제거하면 더 작은 금액의 최적 동전 문제가 됩니다.

탐욕법과 차이
동전 [1, 3, 4]로 6을 만들 때 큰 동전 우선은 4+1+1이지만, 최적해는 3+3입니다.

예제 8

연속 부분 배열의 최대 합

카데인 알고리즘1차원 테이블
def maximum_subarray(numbers):
    dp = [0] * len(numbers)
    dp[0] = numbers[0]

    best = dp[0]

    for index in range(1, len(numbers)):
        dp[index] = max(
            numbers[index],
            dp[index - 1] + numbers[index]
        )

        best = max(best, dp[index])

    print("상태값:", dp)
    return best

numbers = [-2, 1, -3, 4, -1, 2, 1, -5, 4]

print("최대 합:", maximum_subarray(numbers))
상태 정의
dp[i]는 반드시 i번째 원소에서 끝나는 연속 부분 배열 중 최대 합입니다. 새로 시작할지 이전 부분 배열에 이어 붙일지 선택합니다.

예제 9

0/1 배낭 문제의 2차원 테이블

선택·미선택최대 가치
def knapsack(weights, values, capacity):
    item_count = len(weights)

    dp = [
        [0] * (capacity + 1)
        for _ in range(item_count + 1)
    ]

    for item in range(1, item_count + 1):
        weight = weights[item - 1]
        value = values[item - 1]

        for limit in range(capacity + 1):
            # 현재 물건을 넣지 않음
            dp[item][limit] = dp[item - 1][limit]

            # 현재 물건을 넣음
            if weight <= limit:
                dp[item][limit] = max(
                    dp[item][limit],
                    dp[item - 1][limit - weight]
                    + value
                )

    for row in dp:
        print(row)

    return dp[item_count][capacity]

weights = [2, 3, 4, 5]
values = [3, 4, 5, 8]

print("최대 가치:", knapsack(
    weights,
    values,
    capacity=8
))
최적 부분 구조
현재 물건을 넣지 않은 최적값과, 현재 물건을 넣고 남은 용량에서 이전 물건들로 만든 최적값을 비교합니다.

메모이제이션·테이블 방식 종합 비교

호출 수실행 시간종합 실험
import time

def compare_fibonacci(n):
    memo = {0: 0, 1: 1}
    memo_calls = 0

    def top_down(value):
        nonlocal memo_calls
        memo_calls += 1

        if value in memo:
            return memo[value]

        memo[value] = (
            top_down(value - 1)
            + top_down(value - 2)
        )

        return memo[value]

    start = time.perf_counter()
    top_result = top_down(n)
    top_time = time.perf_counter() - start

    start = time.perf_counter()

    if n < 2:
        bottom_result = n
        table = [n]
    else:
        table = [0] * (n + 1)
        table[1] = 1

        for index in range(2, n + 1):
            table[index] = (
                table[index - 1]
                + table[index - 2]
            )

        bottom_result = table[n]

    bottom_time = time.perf_counter() - start

    return {
        "결과 일치": top_result == bottom_result,
        "메모 호출 수": memo_calls,
        "메모 저장 수": len(memo),
        "테이블 칸 수": len(table),
        "메모 시간": top_time,
        "테이블 시간": bottom_time
    }

result = compare_fibonacci(500)

for key, value in result.items():
    if "시간" in key:
        print(f"{key}: {value:.8f}초")
    else:
        print(key, ":", value)

논리적 설명 예시
“피보나치 문제에서는 F(n-2), F(n-3) 등의 부분 문제가 여러 재귀 경로에서 반복되므로 중복 부분 문제가 존재한다. 또한 F(n)의 해는 F(n-1)과 F(n-2)의 해로 구성되므로 최적 부분 구조와 같은 부분 문제 결합 구조를 가진다. 메모이제이션은 필요한 값을 재귀적으로 계산하여 저장하고, 테이블 방식은 작은 인덱스부터 순서대로 값을 채운다.”

Ⅶ. 수업 끝 멘트

시는 한 줄의 기억을 다음 연의 이미지로 이어 가고, 소설은 앞 장면의 선택을 뒤 장면의 운명으로 연결합니다. 희곡은 이전 대사의 여운이 다음 갈등을 만들고, 연극은 반복 연습에서 얻은 감각을 무대 위 새로운 순간에 다시 사용합니다. 음악도 앞선 음의 울림이 사라지기 전에 다음 음이 이어질 때 아름다운 선율이 됩니다.

여러분의 꿈도 매일 처음부터 다시 시작하는 것이 아닙니다. 어제의 작은 이해, 오늘의 한 번 더 도전, 실패에서 얻은 교훈이 모두 마음속 테이블에 저장되어 내일의 더 큰 선택을 돕습니다. 때로는 모든 기록을 다 들고 갈 필요 없이 가장 중요한 이전 두 상태만 기억해도 충분합니다. 그러나 사람의 마음은 공간 최적화한다고 좋은 기억까지 지우면 안 됩니다.

그리고 숙제를 메모이제이션했다고 말하며 머릿속에만 저장하면 선생님은 캐시를 조회할 수 없습니다. 결과는 반드시 제출 파일로 반환하세요. 오늘 저장한 배움이 여러분의 다음 문제를 더 빠르고 정확하게 해결하는 힘이 되기를 바랍니다.
