6. 학습할 코드 1개

피보나치 수열로 단순 재귀와 동적계획법 비교하기

recursive_calls = 0
memo_calls = 0


def fib_recursive(n):
    global recursive_calls
    recursive_calls += 1

    if n <= 2:
        return 1

    return fib_recursive(n - 1) + fib_recursive(n - 2)


def fib_memo(n, memo):
    global memo_calls
    memo_calls += 1

    if n in memo:
        return memo[n]

    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]


def fib_bottom_up(n):
    if n <= 2:
        return 1

    table = [0] * (n + 1)
    table[1] = 1
    table[2] = 1

    for index in range(3, n + 1):
        table[index] = table[index - 1] + table[index - 2]

    return table[n]


n = int(input("n을 입력하세요: "))

memo = {1: 1, 2: 1}

recursive_result = fib_recursive(n)
memo_result = fib_memo(n, memo)
bottom_up_result = fib_bottom_up(n)

print("단순 재귀 결과:", recursive_result)
print("메모이제이션 결과:", memo_result)
print("상향식 결과:", bottom_up_result)
print("단순 재귀 호출 횟수:", recursive_calls)
print("메모이제이션 호출 횟수:", memo_calls)

코드 요소설명
fib_recursive재귀관계를 그대로 표현하지만 같은 fib(k)를 여러 번 계산한다.
recursive_calls단순 재귀 함수가 호출된 횟수를 세어 중복 계산 정도를 확인한다.
memo = {1:1, 2:1}기저 문제의 답을 미리 저장한 메모 표이다.
if n in memo이미 계산된 부분 문제라면 새로 계산하지 않고 저장값을 반환한다.
memo[n] = ...처음 계산한 fib(n)의 결과를 저장하여 다음 호출에서 재사용한다.
table상향식으로 작은 문제의 답부터 순서대로 저장하는 리스트이다.
table[index-1] + table[index-2]두 이전 부분 문제의 결과를 이용해 현재 문제의 답을 만든다.
결과 비교세 방식은 같은 피보나치 값을 내지만 계산 횟수와 호출 구조가 다르다.

핵심: 단순 재귀는 fib(3), fib(4) 등을 반복 계산하지만 메모이제이션과 상향식 방식은 각 부분 문제의 결과를 저장해 다시 사용한다.
