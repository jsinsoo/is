🧩 학습할 핵심 개념 10개

1 개인정보와 데이터 책임

사람과 연결되는 정보

개념 상세 설명: 개인정보는 개인을 직접 알아볼 수 있거나 다른 정보와 결합하여 개인을 알아볼 가능성이 있는 정보이다. 이름뿐 아니라 학번, 연락처, 위치, 정밀한 시간 기록, 희귀한 특성의 조합도 개인과 연결될 수 있다.

fields = ["이름", "학번", "학년", "교실 온도", "정확한 위치"]
personal = [x for x in fields if x in ["이름", "학번", "정확한 위치"]]
print("개인 식별 위험 항목:", personal)
코드 상세 설명: 자료 항목 중 개인을 알아볼 위험이 큰 항목을 목록에서 골라낸다. 실제 판단에서는 여러 항목이 결합될 가능성도 함께 살펴야 한다.
[개념 1] 개인정보와 데이터 책임에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 직접 식별자뿐 아니라 정보 결합과 개인 영향도 고려해야 한다.

2 데이터 최소 수집

필요한 만큼만

개념 상세 설명: 데이터 최소 수집은 연구 목적을 달성하는 데 꼭 필요한 정보만 수집하는 원칙이다. “나중에 쓸 수도 있다”는 이유로 항목을 늘리면 유출과 오용 위험도 함께 커진다.

requested = ["학년", "만족도", "이름", "전화번호"]
needed = ["학년", "만족도"]
final_fields = [x for x in requested if x in needed]
print("최종 수집 항목:", final_fields)
코드 상세 설명: 요청한 항목 중 연구에 꼭 필요한 항목만 남긴다. 최소 수집은 수집 단계뿐 아니라 저장 기간과 공유 범위에도 적용된다.
[개념 2] 데이터 최소 수집 원칙에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 필요한 범위, 기간, 공유 수준을 최소화해야 한다.

3 참여 동의

알고 선택할 권리

개념 상세 설명: 참여 동의는 연구 목적, 수집 항목, 이용 방법, 보관 기간, 예상 위험, 참여 철회 방법 등을 이해하기 쉬운 말로 안내한 뒤 자발적으로 참여하도록 하는 절차이다. 강요나 불이익이 없어야 한다.

consent = {
    "연구목적": "수업 만족도 분석",
    "수집항목": ["학년", "만족도"],
    "보관기간": "학기 종료 후 삭제",
    "철회가능": True
}
for key, value in consent.items():
    print(key, ":", value)
코드 상세 설명: 동의 안내문에 포함할 핵심 내용을 딕셔너리로 정리한다. 동의는 단순한 체크 표시가 아니라 충분한 설명과 자발적인 선택을 전제로 한다.
[개념 3] 참여 동의에 포함할 내용 3개를 고르시오.
정답: ②, ③, ⑤ — 충분한 설명, 자발성, 철회 가능성이 중요하다.

4 비식별화와 가명화

식별 위험 줄이기

개념 상세 설명: 비식별화는 개인을 알아볼 위험을 줄이기 위한 처리의 넓은 개념이다. 이름 삭제, 가명 ID 부여, 연령 범주화, 위치 범위 확대, 일부 값 숨김 등의 방법이 있다. 가명화된 자료도 추가 정보와 결합하면 다시 식별될 수 있다.

record = {"이름":"김하늘", "나이":17, "지역":"전주시 완산구"}
safe = {
    "참여자ID":"P014",
    "연령대":"10대",
    "지역":"전주시"
}
print(safe)
코드 상세 설명: 이름은 참여자 ID로 바꾸고, 나이와 지역은 더 넓은 범주로 변환했다. 정보 손실과 보호 수준 사이의 균형을 고려해야 한다.
[개념 4] 비식별화에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 비식별화 이후에도 재식별 가능성과 보안 위험을 관리해야 한다.

5 재식별 위험과 결합 정보

익명처럼 보여도

개념 상세 설명: 재식별은 익명 또는 가명처럼 보이는 자료를 다른 자료와 결합하여 개인을 다시 알아보는 것이다. 희귀한 조합, 정밀한 위치, 작은 집단, 정확한 시간 기록은 위험을 높인다.

records = [
    {"학년":1, "동아리":"천문", "지역":"A"},
    {"학년":1, "동아리":"천문", "지역":"A"},
    {"학년":3, "동아리":"희귀동아리", "지역":"B"}
]
for r in records:
    if r["동아리"] == "희귀동아리":
        print("재식별 위험이 큰 희귀 조합:", r)
코드 상세 설명: 표본에서 매우 드문 특성 조합을 찾아 재식별 위험을 점검한다. 실제 분석에서는 빈도 기준과 공개 범위를 더 체계적으로 검토한다.
[개념 5] 재식별 위험을 높일 수 있는 조건 3개를 고르시오.
정답: ①, ②, ⑤ — 희귀성, 정밀성, 외부 결합 가능성이 위험을 높인다.

6 안전한 저장과 접근 통제

보호는 수집 후에도

개념 상세 설명: 개인정보 보호는 수집 단계에서 끝나지 않는다. 파일 접근 권한, 비밀번호, 암호화, 백업, 전송 방법, 보관 기간과 삭제 절차를 정해야 한다. 분석에 필요한 사람만 접근하도록 제한한다.

access = {
    "연구담당자": "읽기·수정",
    "분석학생": "비식별 자료 읽기",
    "외부공개": "집계 결과만"
}
for role, permission in access.items():
    print(role, ":", permission)
코드 상세 설명: 역할에 따라 접근 권한을 다르게 정한다. 원자료, 비식별 자료, 공개 결과를 분리하면 불필요한 노출을 줄일 수 있다.
[개념 6] 안전한 데이터 관리 방법 3개를 고르시오.
정답: ②, ④, ⑤ — 최소 권한, 자료 분리, 보관·삭제 관리가 필요하다.

7 데이터 편향과 공정성

누가 빠졌는가

개념 상세 설명: 편향은 표본 선정, 측정 도구, 라벨링, 분석 방법에서 특정 집단에 불리한 왜곡이 생기는 현상이다. 전체 정확도만 보지 말고 집단별 포함 비율과 오류율을 비교해야 한다.

accuracy = {"집단A": 0.91, "집단B": 0.72, "집단C": 0.89}
gap = max(accuracy.values()) - min(accuracy.values())
print("집단별 정확도:", accuracy)
print("최대 격차:", round(gap, 2))
코드 상세 설명: 집단별 정확도를 비교하고 최대 격차를 계산한다. 격차가 발견되면 표본 수, 측정 조건, 라벨 기준과 모델 특징을 점검한다.
[개념 7] 데이터 편향을 점검하는 방법 3개를 고르시오.
정답: ①, ③, ⑤ — 대표성, 집단별 결과, 데이터 생성 과정을 함께 살펴야 한다.

8 저작권과 이용 허락

공개와 자유 이용은 다르다

개념 상세 설명: 인터넷에서 찾을 수 있다는 사실은 자유롭게 복사·수정·배포해도 된다는 뜻이 아니다. 글, 사진, 그림, 코드, 데이터의 저작자와 이용 조건을 확인하고 필요한 허락을 받아야 한다.

resources = [
    {"title":"공공 통계표", "license":"출처 표시 후 이용"},
    {"title":"개인 블로그 사진", "license":"이용 조건 확인 필요"}
]
for item in resources:
    print(item["title"], "-", item["license"])
코드 상세 설명: 자료마다 제목과 이용 조건을 함께 기록한다. 출처 표시와 이용 허락은 서로 다른 문제이므로 둘 다 확인해야 한다.
[개념 8] 저작권과 이용 허락에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 공개 여부, 출처 표시, 이용 허락을 구분해야 한다.

9 출처 표시

누가·무엇을·어디서

개념 상세 설명: 출처 표시는 저작자 또는 기관, 자료 제목, 게시 또는 갱신 시점, 자료 위치, 이용일, 라이선스를 가능한 범위에서 기록하는 것이다. 독자가 자료를 다시 확인할 수 있어야 한다.

citation = {
    "기관": "공공데이터 제공기관",
    "자료명": "지역별 대기질 자료",
    "갱신일": "2026-06-30",
    "라이선스": "이용 조건 확인"
}
print(" / ".join(f"{k}: {v}" for k, v in citation.items()))
코드 상세 설명: 출처 표시의 핵심 항목을 일정한 형식으로 출력한다. 보고서 전체에서 같은 인용 형식을 유지하면 확인이 쉽다.
[개념 9] 바람직한 출처 표시에 포함할 내용 3개를 고르시오.
정답: ②, ③, ⑤ — 자료를 다시 확인하고 이용 조건을 파악할 수 있어야 한다.

10 CCL 이해와 적용

조건을 조합한 이용 허락

개념 상세 설명: CCL은 저작자가 일정한 조건 아래 이용을 허락하는 표시 체계이다. BY는 저작자 표시, NC는 비영리, ND는 변경 금지, SA는 동일조건변경허락을 뜻한다. 구체적인 이용 가능 범위는 표시된 조건을 함께 읽어야 한다.

license_terms = {
    "BY": "저작자 표시",
    "NC": "비영리",
    "ND": "변경 금지",
    "SA": "동일조건변경허락"
}
selected = ["BY", "NC", "SA"]
print("적용 조건:", [license_terms[x] for x in selected])
코드 상세 설명: CCL 기호를 의미와 연결하고 선택한 조건을 출력한다. 여러 조건이 함께 표시되면 모두 지켜야 한다.
[개념 10] CCL에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 표시된 CCL 조건을 모두 확인하고 지켜야 한다.

📚 학습과 관련한 탐구 보고서 주제 예시 5개

1. 익명 설문에서 개인정보 최소 수집이 응답 신뢰도에 미치는 영향

핵심 윤리: 이름·연락처를 수집하지 않고 참여 목적과 철회 방법을 안내한다.

탐구 관점: 개인정보 요구 수준에 따른 응답률과 무응답률을 비교한다.

2. 가명화 수준에 따른 데이터 분석 가능성과 재식별 위험 비교

핵심 윤리: 이름 삭제, 연령 범주화, 지역 범위 확대를 단계별로 적용한다.

탐구 관점: 보호 수준이 높아질 때 분석 정보가 얼마나 줄어드는지 살펴본다.

3. 학년별 표본 불균형이 인공지능 분류 결과에 미치는 영향

핵심 윤리: 집단별 표본 수와 오류율을 공개하고 불리한 영향을 점검한다.

탐구 관점: 균형 표본과 불균형 표본의 성능 차이를 비교한다.

4. 공공데이터 출처 표시 방식에 따른 자료 재현 가능성 연구

핵심 윤리: 제공기관, 자료명, 갱신일, 이용일, 라이선스를 기록한다.

탐구 관점: 출처 정보의 구체성이 자료 재확인 성공률에 미치는 영향을 분석한다.

5. CCL 조건별 디지털 학습 자료 재사용 가능 범위 비교

핵심 윤리: BY, NC, ND, SA의 의미를 구분하고 조건을 지킨다.

탐구 관점: 수정·배포·상업적 이용 가능 범위를 사례별로 비교한다.
