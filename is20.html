1. 성취기준 총정리

성취기준핵심 수행학생이 할 수 있어야 하는 것
[12정과04-01]다양한 학문 분야에서 컴퓨터과학으로 해결할 수 있는 실생활 문제를 발견한다.사용자·상황·원인·영향을 관찰하고 데이터와 알고리즘 적용 가능성을 판단한다.
[12정과04-02]발견한 문제를 분석하고 프로젝트 수행 과정을 설계한다.목표, 요구사항, 입력·처리·출력, 제한 조건, 일정, 역할, 평가 기준을 정한다.
[12정과04-03]오픈소스와 라이브러리를 활용하여 협력적으로 문제 해결 프로그램을 작성한다.도구의 기능과 라이선스를 확인하고 역할별 코드를 명확한 인터페이스로 통합한다.
[12정과04-04]테스트와 디버깅을 통해 프로그램을 검증한다.정상값·경계값·예외 입력을 시험하고 오류 원인을 찾아 수정한 뒤 회귀 테스트한다.
[12정과04-05]산출물을 기능적·윤리적 관점에서 평가하고 결과를 공유한다.정확성·효율성·사용성·안정성과 개인정보·공정성·저작권·투명성을 함께 평가한다.
총정리 관점: 문제를 잘 발견하는 능력, 실행 가능한 계획을 세우는 능력, 협력하여 구현하는 능력, 오류를 검증하는 능력, 결과를 책임 있게 평가·공유하는 능력이 하나의 프로젝트 흐름으로 연결된다.

7. 학습할 코드 1개

프로젝트 계획·검증·윤리 평가를 종합하는 관리 프로그램

project = {
    "title": "교실 공기질 알림 시스템",
    "problem": "수업 중 공기질 악화를 즉시 알기 어렵다.",
    "requirements": [
        "이산화탄소 농도 입력",
        "상태 자동 분류",
        "환기 필요 알림",
        "개인 식별 정보 미수집"
    ],
    "tasks": [
        {"name": "문제 분석", "days": 2, "done": True},
        {"name": "데이터 수집", "days": 3, "done": True},
        {"name": "알고리즘 설계", "days": 3, "done": True},
        {"name": "프로그램 구현", "days": 5, "done": False},
        {"name": "테스트와 개선", "days": 3, "done": False},
        {"name": "결과 공유", "days": 2, "done": False}
    ],
    "tests": [
        {"name": "정상값 700ppm", "passed": True},
        {"name": "경계값 1000ppm", "passed": True},
        {"name": "오류값 -1ppm", "passed": False},
        {"name": "문자열 입력", "passed": False}
    ],
    "evaluation": {
        "정확성": 4,
        "효율성": 4,
        "사용성": 3,
        "안정성": 2,
        "개인정보 보호": 5,
        "공정성": 4,
        "저작권 준수": 5,
        "투명성": 3
    }
}

functional_items = ["정확성", "효율성", "사용성", "안정성"]
ethical_items = ["개인정보 보호", "공정성", "저작권 준수", "투명성"]


def average_score(items, evaluation):
    return sum(evaluation[item] for item in items) / len(items)


def calculate_progress(tasks):
    total_days = sum(task["days"] for task in tasks)
    completed_days = sum(
        task["days"] for task in tasks if task["done"]
    )
    return completed_days / total_days * 100


def calculate_test_rate(tests):
    passed = sum(1 for test in tests if test["passed"])
    return passed / len(tests) * 100


def find_improvements(project, limit=3):
    improvements = []

    for test in project["tests"]:
        if not test["passed"]:
            improvements.append("테스트 보완: " + test["name"])

    for item, score in project["evaluation"].items():
        if score < limit:
            improvements.append("평가 보완: " + item)

    return improvements


def create_project_report(project):
    progress = calculate_progress(project["tasks"])
    test_rate = calculate_test_rate(project["tests"])

    functional = average_score(
        functional_items,
        project["evaluation"]
    )
    ethical = average_score(
        ethical_items,
        project["evaluation"]
    )

    improvements = find_improvements(project)

    lines = [
        "프로젝트: " + project["title"],
        "문제: " + project["problem"],
        f"진행률: {progress:.1f}%",
        f"테스트 통과율: {test_rate:.1f}%",
        f"기능 평가 평균: {functional:.2f}",
        f"윤리 평가 평균: {ethical:.2f}",
        "개선 필요 항목"
    ]

    if improvements:
        for index, item in enumerate(improvements, start=1):
            lines.append(f"{index}. {item}")
    else:
        lines.append("없음")

    return "\\n".join(lines)


report = create_project_report(project)
print(report)

상세 설명

코드 요소설명
project문제 정의, 요구사항, 일정, 테스트, 기능·윤리 평가를 하나의 딕셔너리로 구조화한다.
requirements프로그램이 반드시 만족해야 할 기능 및 윤리 요구사항을 저장한다.
calculate_progress전체 작업 기간 중 완료된 작업 기간의 비율을 계산한다.
calculate_test_rate전체 테스트 중 통과한 테스트의 비율을 백분율로 구한다.
average_score기능 항목과 윤리 항목의 평균을 각각 계산하여 균형을 비교하게 한다.
find_improvements실패한 테스트와 기준 미달 평가 항목을 자동으로 찾아 개선 목록을 만든다.
enumerate(..., start=1)개선 필요 항목에 1번부터 번호를 붙여 보고서 가독성을 높인다.
create_project_report프로젝트 진행, 테스트, 기능·윤리 평가, 개선 방향을 하나의 보고서로 통합한다.

해석: 예시 프로그램은 일정 진행률뿐 아니라 테스트 실패와 안정성 부족을 함께 보여 준다. 즉, ‘진행이 많이 되었다’와 ‘완성도가 높다’는 같은 의미가 아니며, 기능과 윤리를 함께 개선해야 한다.

8. 수업 끝 멘트

“시는 한 편을 다 쓴 뒤에도 독자의 마음에서 새로운 의미를 만들고, 소설은 마지막 장을 덮은 뒤 인물의 선택을 다시 생각하게 합니다. 희곡은 막이 내려가도 배우와 관객에게 질문을 남기고, 음악은 마지막 음 뒤의 침묵 속에서 더 오래 울립니다. 여러분의 정보과학 프로젝트도 제출로 끝나지 않습니다. 누군가 사용하고 질문하고 개선할 때 비로소 살아 있는 결과가 됩니다. 문제를 발견한 눈, 계획을 세운 생각, 오류를 고친 끈기, 윤리를 지킨 책임감을 앞으로의 꿈에 저장해 두세요. 단, ‘완료’ 버튼을 눌렀다고 오류까지 졸업하는 것은 아닙니다. 버그는 졸업식에도 출석할 수 있습니다!”
