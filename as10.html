1 문제 정의와 요구사항

무엇을 왜 해결하는가

개념 상세 설명: 문제 정의는 해결하려는 상황, 사용자, 목표와 제한 조건을 분명하게 나타내는 과정이다. 모호한 표현을 측정 가능하고 검증 가능한 요구사항으로 바꾸어야 한다.

problem = {
    "사용자": "교실 학생과 교사",
    "목표": "이산화탄소 농도가 기준을 넘으면 환기 알림",
    "입력": "이산화탄소 농도(ppm)",
    "출력": "환기 필요 여부와 안내 문장",
    "제한": "센서 값이 없거나 잘못된 경우 처리"
}

for key, value in problem.items():
    print(key, ":", value)
코드 상세 설명: 문제의 핵심 요소를 딕셔너리로 정리한다. 목표와 입력·출력뿐 아니라 제한 조건을 함께 기록하면 구현과 검증이 쉬워진다.
[개념 1] 문제 정의에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 목표, 입력·출력과 제한을 구체적으로 정의해야 한다.

2 입력·처리·출력 분해

자료가 결과가 되는 과정

개념 상세 설명: 입력은 문제 해결에 필요한 자료, 처리는 입력을 변환하는 규칙과 계산, 출력은 사용자에게 제공되는 결과이다. 세 요소를 구분하면 알고리즘의 경계를 명확히 할 수 있다.

temperature = float(input("기온(℃): "))   # 입력

if temperature >= 30:                     # 처리
    message = "폭염 대비가 필요합니다."
else:
    message = "일반적인 기온 범위입니다."

print(message)                             # 출력
코드 상세 설명: 사용자가 입력한 기온을 기준과 비교하여 안내 문장을 만들고 출력한다. 입력, 처리와 출력이 차례로 연결된다.
[개념 2] 입력·처리·출력에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 입력을 처리하여 의미 있는 출력을 만든다.

3 기능 분해

큰 문제를 작은 기능으로

개념 상세 설명: 기능 분해는 복잡한 문제를 자료 입력, 검증, 계산, 판정, 출력과 저장처럼 작은 기능으로 나누는 과정이다. 각 기능은 한 가지 책임을 갖도록 설계하면 수정과 재사용이 쉬워진다.

def read_data():
    return [22.1, 23.4, 24.0, 25.2]

def calculate_mean(data):
    return sum(data) / len(data)

def make_report(mean):
    return f"평균 기온은 {mean:.1f}℃입니다."

data = read_data()
mean = calculate_mean(data)
print(make_report(mean))
코드 상세 설명: 입력, 계산과 보고서 생성 기능을 각각 함수로 분리했다. 한 함수가 한 역할을 담당해 전체 구조를 이해하고 수정하기 쉽다.
[개념 3] 기능 분해에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 기능 분해는 역할을 명확히 하고 유지보수를 돕는다.

4 알고리즘의 조건

명확성·유한성·실행 가능성

개념 상세 설명: 알고리즘은 문제를 해결하기 위한 유한하고 명확한 절차이다. 각 단계는 모호하지 않아야 하며 실제로 실행 가능하고 정해진 조건에서 종료되어야 한다.

numbers = [7, 3, 9, 2]
maximum = numbers[0]

for number in numbers[1:]:
    if number > maximum:
        maximum = number

print("최댓값:", maximum)
코드 상세 설명: 첫 값을 기준으로 두고 남은 값을 차례로 비교해 최댓값을 갱신한다. 입력 크기만큼 반복한 뒤 반드시 종료된다.
[개념 4] 알고리즘의 조건에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 알고리즘은 명확하고 실행 가능하며 유한해야 한다.

5 의사코드

코드 전에 절차를 표현

개념 상세 설명: 의사코드는 특정 프로그래밍 언어의 문법에 얽매이지 않고 알고리즘의 핵심 절차를 자연어와 구조화된 표현으로 나타낸다. 사람과의 의사소통과 코드 구현 전 점검에 유용하다.

# 의사코드
자료를 입력받는다
유효한 값만 남긴다
자료가 비어 있으면 오류를 알린다
그렇지 않으면 평균을 계산한다
평균과 자료 수를 출력한다

# 파이썬 구현
data = [12, None, 18, 20]
valid = [x for x in data if x is not None]

if not valid:
    print("유효한 자료가 없습니다.")
else:
    print("평균:", sum(valid) / len(valid))
코드 상세 설명: 먼저 언어와 독립적인 절차를 작성하고 이를 파이썬으로 구현했다. 의사코드는 구현 전에 빠진 단계와 예외를 찾는 데 도움을 준다.
[개념 5] 의사코드에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ⑤ — 의사코드는 언어와 독립적으로 해결 절차를 설명한다.

6 순서도

흐름을 도형과 화살표로

개념 상세 설명: 순서도는 시작·끝, 입력·출력, 처리와 판단을 정해진 도형으로 나타내고 화살표로 실행 흐름을 표현한다. 판단 도형의 각 경로에는 예·아니오 등의 조건을 명확히 표시한다.

# 순서도의 흐름을 파이썬으로 표현
score = int(input("점수: "))

if score >= 60:
    result = "통과"
else:
    result = "보완 필요"

print(result)

# 시작 → 점수 입력 → 점수≥60?
#              ├─ 예 → 통과 출력 → 끝
#              └─ 아니오 → 보완 필요 출력 → 끝
코드 상세 설명: 한 개의 판단 조건에서 두 경로로 나뉘고 다시 출력 단계로 이어진다. 순서도에서는 흐름이 끊기거나 조건 표시가 빠지지 않도록 한다.
[개념 6] 순서도에 대한 옳은 설명 3개를 고르시오.
정답: ②, ④, ⑤ — 순서도는 도형과 화살표로 절차의 흐름을 보여 준다.

7 순차·선택·반복 구조

알고리즘의 기본 제어 구조

개념 상세 설명: 순차 구조는 명령을 차례로 수행하고, 선택 구조는 조건에 따라 경로를 나누며, 반복 구조는 조건이나 횟수에 따라 명령을 되풀이한다. 세 구조를 조합해 대부분의 알고리즘을 표현할 수 있다.

values = [72, 85, 91, 58, 77]
passed = 0

for value in values:          # 반복
    if value >= 60:           # 선택
        passed += 1           # 순차 처리

print("통과 인원:", passed)
코드 상세 설명: 리스트의 값을 반복하여 확인하고, 조건에 따라 통과 인원을 증가시킨 뒤 결과를 출력한다.
[개념 7] 제어 구조에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 순차, 선택과 반복은 알고리즘의 기본 구조이다.

8 예외 상황과 입력 검증

잘못된 입력도 설계에 포함

개념 상세 설명: 예외 상황은 빈 입력, 잘못된 자료형, 허용 범위 밖의 값이나 파일 부재처럼 정상 흐름을 방해하는 경우이다. 입력을 사용하기 전에 형식과 범위를 검증하고 사용자에게 이해할 수 있는 안내를 제공해야 한다.

raw = input("습도(0~100): ")

try:
    humidity = float(raw)
    if not 0 <= humidity <= 100:
        raise ValueError("허용 범위를 벗어났습니다.")
    print("입력된 습도:", humidity)
except ValueError as error:
    print("입력 오류:", error)
코드 상세 설명: 숫자로 변환할 수 없는 입력과 0~100 범위를 벗어난 값을 모두 처리한다. 오류가 발생해도 프로그램이 갑자기 종료되지 않고 원인을 안내한다.
[개념 8] 예외 처리와 입력 검증에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 예외를 예상하고 안전하게 처리하는 것도 알고리즘의 일부이다.

9 경계값과 종료 조건

끝값과 반복의 끝 확인

개념 상세 설명: 경계값은 조건이 바뀌는 지점이나 허용 범위의 양끝값이다. 반복 알고리즘은 종료 조건과 상태 변화가 분명해야 무한 반복을 피할 수 있다.

count = 0
total = 0

while count < 5:
    value = int(input("정수 입력: "))
    total += value
    count += 1

print("합계:", total)
코드 상세 설명: count가 5보다 작은 동안 반복하고 매번 1씩 증가한다. 경계값 5에서 반복이 종료되는지 확인해야 한다.
[개념 9] 경계값과 종료 조건에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 경계와 종료 조건을 점검해야 무한 반복과 조건 오류를 막을 수 있다.

10 알고리즘 검증과 개선

실행 전에 논리를 시험

개념 상세 설명: 알고리즘 검증은 예시 입력을 단계별로 따라가며 예상 출력과 일치하는지 확인하는 과정이다. 정상값, 경계값과 예외값을 사용하고 정확성, 이해 가능성, 효율성과 재사용성을 기준으로 개선한다.

def classify(value):
    if value < 0:
        return "오류"
    if value == 0:
        return "영"
    return "양수"

tests = [
    (-1, "오류"),
    (0, "영"),
    (5, "양수")
]

for value, expected in tests:
    result = classify(value)
    print(value, result, result == expected)
코드 상세 설명: 정상, 경계와 예외 입력에 대한 예상 결과를 준비하고 실제 결과와 비교한다. 실패한 시험은 알고리즘을 수정해야 할 근거가 된다.
[개념 10] 알고리즘 검증에 대한 옳은 설명 3개를 고르시오.

📚 학습과 관련한 탐구 보고서 주제 예시 5개

1. 교실 환경 자동 알림 알고리즘 설계

설계: 온도·습도·이산화탄소를 입력으로 받고 기준 비교와 우선순위 처리를 설계한다.

탐구 관점: 센서 누락, 비정상값과 여러 위험 조건이 동시에 발생하는 예외를 분석한다.

2. 학교 급식 잔반 분류 절차의 기능 분해 연구

설계: 사진 입력, 품목 분류, 양 추정, 결과 저장 기능으로 분해한다.

탐구 관점: 기능 분해가 수정 가능성과 오류 추적에 미치는 영향을 비교한다.

3. 등교 안전 지수 산출 알고리즘 설계

설계: 기온, 강수, 미세먼지와 풍속을 입력으로 안전 등급을 출력한다.

탐구 관점: 임계값, 가중치와 경계값 변화에 따른 결과 차이를 분석한다.

4. 탐구 자료 결측값 처리 알고리즘 비교

설계: 제거, 평균 대체와 이전값 대체 절차를 의사코드로 표현한다.

탐구 관점: 각 방법의 입력 조건, 장단점과 예외 상황을 비교한다.

5. 과학 실험 오류 감지 순서도 설계

설계: 허용 범위, 변화율과 반복 측정 차이를 단계적으로 검사한다.

탐구 관점: 검사 순서가 실행 시간과 오류 탐지 결과에 미치는 영향을 연구한다.
