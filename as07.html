성취기준

영역수업에 적용한 성취기준
자료 수집수립한 계획과 데이터 사전에 따라 자료를 수집하고 수집 시각, 출처, 장비와 조건을 기록한다.
품질 검증정확성, 완전성, 일관성 기준에 따라 자료의 범위, 형식, 누락, 중복과 관계 오류를 점검한다.
오류 처리오류를 임의로 숨기지 않고 유형, 원인, 수정 방법과 처리 결과를 기록한다.
신뢰성 평가검증 전후의 자료 품질을 비교하고 남아 있는 한계와 불확실성을 설명한다.

학습 목표

  1. 수집 계획에 따라 자료의 출처, 시각과 조건을 일관되게 기록할 수 있다.
  2. 정확성, 완전성, 일관성의 의미를 사례로 구분할 수 있다.
  3. 허용 범위, 자료형, 필수값과 관계 조건을 검증 규칙으로 표현할 수 있다.
  4. 중복 후보를 찾아 실제 중복과 반복 관측을 구분할 수 있다.
  5. 오류의 유형, 원인, 수정 내용과 결과를 오류 기록표에 작성할 수 있다.
  6. 검증 전후 품질 지표를 비교하여 자료의 신뢰성과 한계를 설명할 수 있다.

1 계획에 따른 데이터 수집

같은 절차로 반복

개념 상세 설명: 자료 수집은 미리 정한 대상, 시각, 주기, 장비, 단위와 방법에 따라 수행해야 한다. 수집 조건이 달라지면 값의 차이가 실제 현상인지 방법 차이인지 구분하기 어렵다.

collection_log = {
    "측정시각": "2026-07-20 09:00",
    "장소": "과학실",
    "장비": "온습도센서-A",
    "단위": "℃",
    "측정자": "P01"
}
for key, value in collection_log.items():
    print(key, ":", value)
코드 상세 설명: 측정값뿐 아니라 측정 시각, 장소, 장비와 단위를 함께 기록한다. 이러한 정보는 오류 원인 분석과 재현에 도움을 준다.
[개념 1] 계획에 따른 자료 수집에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 수집 조건과 예외 상황을 기록해야 자료를 해석할 수 있다.

2 데이터 품질의 세 요소

정확성·완전성·일관성

개념 상세 설명: 정확성은 값이 실제 상태나 기준값에 가까운 정도, 완전성은 필요한 값이 빠짐없이 존재하는 정도, 일관성은 같은 규칙과 의미가 자료 전체에서 유지되는 정도이다.

total = 100
missing = 4
complete_rate = (total - missing) / total * 100
print("완전성:", complete_rate, "%")
코드 상세 설명: 전체 100개 중 결측 4개를 제외하여 완전성을 계산한다. 정확성과 일관성은 별도의 기준값 및 형식 검사로 확인해야 한다.
[개념 2] 데이터 품질에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 품질의 여러 요소는 각각 따로 점검해야 한다.

3 검증 규칙

옳은 값의 조건

개념 상세 설명: 검증 규칙은 자료가 만족해야 할 조건이다. 필수 여부, 자료형, 허용 범위, 허용 범주, 형식과 변수 간 관계를 규칙으로 정하면 오류를 체계적으로 찾을 수 있다.

rules = {
    "temperature": {"type": float, "min": -30, "max": 50},
    "grade": {"allowed": [1, 2, 3]}
}
print(rules)
코드 상세 설명: 온도의 자료형과 범위, 학년의 허용값을 규칙으로 저장한다. 검증 기준은 수집 전에 데이터 사전과 함께 정하는 것이 좋다.
[개념 3] 검증 규칙에 포함할 수 있는 조건 3개를 고르시오.
정답: ②, ③, ⑤ — 규칙은 자료 구조와 현실적 의미를 반영해야 한다.

4 범위와 허용값 검사

가능한 값인가

개념 상세 설명: 범위 검사는 수치가 물리적·논리적 범위 안에 있는지 확인한다. 허용값 검사는 범주형 변수가 정해진 코드나 목록에 포함되는지 점검한다.

temperatures = [22.1, 23.4, 125.0, 21.9]
invalid = [x for x in temperatures if not (-30 <= x <= 50)]
print("범위 오류:", invalid)
코드 상세 설명: 온도의 허용 범위를 -30℃에서 50℃로 정하고 벗어난 값을 오류 후보로 찾는다. 실제 이상 현상일 가능성도 있으므로 원자료를 확인한다.
[개념 4] 범위와 허용값 검사에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 규칙 위반값은 확인과 기록을 거쳐 처리한다.

5 형식과 자료형 검사

같은 모양과 성격

개념 상세 설명: 숫자 열에 문자가 들어가거나 날짜 형식이 섞이면 계산과 정렬 오류가 발생한다. 문자열 변환, 정규식, 날짜 변환 등을 활용해 형식과 자료형을 점검한다.

values = ["23.5", "24.1", "오류", "22.9"]
valid, errors = [], []

for x in values:
    try:
        valid.append(float(x))
    except ValueError:
        errors.append(x)

print("정상:", valid)
print("형식 오류:", errors)
코드 상세 설명: 문자열을 실수로 변환하고 실패한 값을 오류 목록에 저장한다. 오류값을 임의로 바꾸지 않고 원인을 확인해야 한다.
[개념 5] 형식과 자료형 검사에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ⑤ — 형식 오류를 발견하고 원인과 처리 결과를 남겨야 한다.

6 완전성과 결측 점검

빠진 값 찾기

개념 상세 설명: 완전성 검증은 필수 항목이 비어 있지 않은지, 결측이 특정 시각이나 집단에 집중되는지 확인하는 과정이다. 결측률뿐 아니라 결측의 패턴과 원인을 함께 분석한다.

rows = [
    {"id":"A01", "value":12},
    {"id":"A02", "value":None},
    {"id":"A03", "value":15}
]
missing_ids = [r["id"] for r in rows if r["value"] is None]
print("결측 관측 ID:", missing_ids)
코드 상세 설명: 값이 비어 있는 행의 ID를 찾아 목록으로 만든다. 결측이 발생한 장비, 시각, 집단을 확인하면 원인을 찾는 데 도움이 된다.
[개념 6] 완전성 검증에 대한 옳은 설명 3개를 고르시오.
정답: ②, ④, ⑤ — 완전성은 결측의 양과 패턴을 함께 점검한다.

7 일관성과 관계 검사

값 사이의 논리

개념 상세 설명: 일관성 검사는 자료 전체에서 단위, 코드와 형식이 같은지 확인하고, 시작 시각이 종료 시각보다 늦거나 최솟값이 최댓값보다 큰 것과 같은 관계 오류를 찾는다.

records = [
    {"min":12, "max":18},
    {"min":21, "max":19}
]
errors = [r for r in records if r["min"] > r["max"]]
print("관계 오류:", errors)
코드 상세 설명: 최솟값이 최댓값보다 큰 행을 관계 오류로 찾는다. 개별 값이 허용 범위 안에 있어도 변수 사이 관계가 잘못될 수 있다.
[개념 7] 일관성 검사에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 값 자체뿐 아니라 변수 사이의 논리도 확인해야 한다.

8 중복 탐지와 제거

같은 자료가 두 번인가

개념 상세 설명: 중복은 동일한 관측이 여러 번 저장된 상태이다. 고유 ID, 시각, 위치와 측정값을 조합하여 중복 후보를 찾고 실제 반복 측정인지 확인한 뒤 제거한다.

rows = [
    ("A01", "09:00", 22.1),
    ("A02", "09:10", 22.4),
    ("A02", "09:10", 22.4)
]
seen, duplicates = set(), []

for row in rows:
    if row in seen:
        duplicates.append(row)
    else:
        seen.add(row)

print("중복 후보:", duplicates)
코드 상세 설명: 행 전체가 동일한 경우를 중복 후보로 찾는다. 센서 반복 측정이나 설문 재응답처럼 실제 상황을 확인한 뒤 처리한다.
[개념 8] 중복 처리에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 중복의 근거와 처리 영향을 확인해야 한다.

9 교차 검증과 기준값 비교

다른 자료와 맞춰 보기

개념 상세 설명: 정확성을 높이기 위해 표준 장비, 원본 기록, 다른 센서나 공공자료와 값을 비교할 수 있다. 차이가 발생하면 보정, 시각 불일치, 단위와 측정 위치를 점검한다.

sensor = [22.3, 22.7, 23.0]
reference = [22.0, 22.5, 22.8]
differences = [round(a-b, 2) for a, b in zip(sensor, reference)]
print("기준값과의 차이:", differences)
코드 상세 설명: 센서값과 기준값의 차이를 계산한다. 차이가 일정하면 보정 문제, 특정 시점에만 크면 일시적 오류 가능성을 살펴볼 수 있다.
[개념 9] 교차 검증에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 비교 조건을 맞추고 차이의 원인을 분석해야 한다.

10 오류 기록과 품질 보고

문제를 숨기지 않는 연구

개념 상세 설명: 오류 기록에는 오류 ID, 발견 시각, 변수, 원래 값, 오류 유형, 원인, 처리 방법, 수정값과 담당자를 남긴다. 검증 전후 오류 수와 완전성 등을 비교하면 품질 개선을 설명할 수 있다.

error_log = {
    "오류ID": "E-007",
    "변수": "temperature",
    "원래값": 230,
    "오류유형": "단위 또는 입력 오류",
    "처리": "원본 기록 확인 후 23.0으로 수정"
}
for key, value in error_log.items():
    print(key, ":", value)
코드 상세 설명: 오류의 발견부터 수정까지 근거를 기록한다. 값을 몰래 바꾸는 것이 아니라 변경 이력을 남겨 분석의 투명성을 높인다.
[개념 10] 오류 기록과 품질 보고에 대한 옳은 설명 3개를 고르시오.
정답: ①, ④, ⑤ — 오류와 수정 과정을 투명하게 남겨야 한다.

📚 학습과 관련한 탐구 보고서 주제 예시 5개

1. 교실 온습도 센서 자료의 품질 검증 규칙 설계

검증: 물리적 범위, 측정 간격, 결측과 급격한 변화 규칙을 적용한다.

탐구 관점: 검증 전후 평균과 변화 추세의 차이를 비교한다.

2. 설문 응답의 중복 탐지 방법에 따른 결과 변화

검증: 응답 ID, 제출 시각, 응답 패턴을 조합해 중복 후보를 찾는다.
