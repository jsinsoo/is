6. 학습할 코드 1개

feedback = [
    {
        "content": "환기 알림 기준을 설명해 주세요.",
        "category": "투명성",
        "impact": 5,
        "urgency": 4,
        "feasibility": 5
    },
    {
        "content": "글자가 작아서 읽기 어렵습니다.",
        "category": "사용성",
        "impact": 4,
        "urgency": 3,
        "feasibility": 5
    },
    {
        "content": "측정 기록에서 개인정보가 보입니다.",
        "category": "개인정보 보호",
        "impact": 5,
        "urgency": 5,
        "feasibility": 4
    }
]


def priority_score(item):
    return (
        item["impact"] * 0.4
        + item["urgency"] * 0.35
        + item["feasibility"] * 0.25
    )


for item in feedback:
    item["priority"] = priority_score(item)

ranked_feedback = sorted(
    feedback,
    key=lambda item: item["priority"],
    reverse=True
)

print("개선 우선순위")
for rank, item in enumerate(ranked_feedback, start=1):
    print(
        rank,
        item["category"],
        round(item["priority"], 2),
        item["content"]
    )
코드 요소상세 설명
feedback사용자 의견과 분류, 영향도, 시급성, 실현 가능성을 저장한 리스트이다.
category의견을 사용성, 기능 오류, 투명성, 개인정보 보호 등으로 분류한 값이다.
priority_score세 평가 기준에 가중치를 적용해 개선 우선순위 점수를 계산한다.
impact * 0.4사용자와 프로젝트에 미치는 영향을 가장 큰 비중으로 반영한다.
urgency * 0.35즉시 해결해야 할 필요성을 반영한다.
feasibility * 0.25현재 시간과 기술로 실제 수정할 수 있는 정도를 반영한다.
reverse=True우선순위 점수가 높은 의견부터 정렬한다.
enumerate(..., start=1)개선 항목에 1위부터 순위를 부여한다.

해석: 개인정보 보호와 같은 윤리적 위험은 영향도와 시급성이 높아 우선 개선 대상이 된다. 점수만으로 결정하지 말고 법적·윤리적 의무와 사용자 의견도 함께 검토해야 한다.
