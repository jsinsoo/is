6. 학습할 코드 1개

목표합 조합 찾기: 완전 탐색과 가지치기 비교

numbers = [2, 3, 5, 7, 9]
target = 12

complete_checked = 0
pruned_checked = 0
complete_solutions = []
pruned_solutions = []


def complete_search(index, selected):
    global complete_checked
    complete_checked += 1

    if index == len(numbers):
        if sum(selected) == target:
            complete_solutions.append(selected[:])
        return

    complete_search(index + 1, selected)
    selected.append(numbers[index])
    complete_search(index + 1, selected)
    selected.pop()


def pruning_search(index, selected, current_sum):
    global pruned_checked
    pruned_checked += 1

    if current_sum > target:
        return

    if index == len(numbers):
        if current_sum == target:
            pruned_solutions.append(selected[:])
        return

    pruning_search(index + 1, selected, current_sum)

    selected.append(numbers[index])
    pruning_search(
        index + 1,
        selected,
        current_sum + numbers[index]
    )
    selected.pop()


complete_search(0, [])
pruning_search(0, [], 0)

print("완전 탐색 해:", complete_solutions)
print("가지치기 해:", pruned_solutions)
print("완전 탐색 검사 수:", complete_checked)
print("가지치기 검사 수:", pruned_checked)

상세 설명

코드 요소설명
index현재 선택 여부를 결정할 숫자의 위치를 나타낸다.
selected현재까지 선택한 숫자들의 목록으로 부분 상태를 표현한다.
current_sum현재까지 선택한 숫자의 합이다. 매번 sum을 다시 계산하지 않아 효율적이다.
선택하지 않는 호출현재 숫자를 제외하고 다음 상태로 이동한다.
선택하는 호출현재 숫자를 selected에 추가하고 합을 갱신한 뒤 다음 상태로 이동한다.
if current_sum > target모든 수가 양수이므로 이미 목표값을 넘었다면 이후 어떤 수를 더해도 목표값이 될 수 없다. 이 분기를 배제한다.
selected[:]현재 해의 복사본을 저장하여 이후 리스트 변경의 영향을 막는다.
검사 수 비교두 알고리즘이 같은 해를 찾는지와 가지치기가 탐색 상태 수를 얼마나 줄였는지 확인한다.

주의: 이 가지치기 조건은 모든 수가 양수일 때 안전하다. 음수가 포함되면 현재 합이 목표를 넘더라도 이후 음수를 더해 목표에 도달할 수 있으므로 같은 조건을 그대로 쓰면 해를 누락할 수 있다.
