1 교과 융합

한 문제를 여러 교과의 눈으로 본다

개념 상세 설명: 교과 융합은 정보 기술만으로 문제를 보는 것이 아니라 과학의 원리, 수학의 모형, 사회의 제도, 윤리의 가치와 환경의 지속가능성을 함께 연결하는 것이다. 서로 다른 개념을 단순 나열하지 않고 문제 해결 과정에서 각 개념이 어떤 역할을 하는지 설명해야 한다.

topic = "교실 공기질 개선"

connections = {
    "정보": "센서 자료 수집과 시각화",
    "과학": "이산화탄소 농도와 환기 원리",
    "수학": "평균·변화율·상관관계 분석",
    "사회": "학교 시설 운영과 학생 건강권",
    "윤리": "개인 식별 자료 최소 수집",
    "환경": "환기와 냉난방 에너지의 균형"
}

print("탐구 주제:", topic)
for subject, role in connections.items():
    print(f"{subject}: {role}")
코드 상세 설명: 하나의 탐구 주제에 대해 교과별 연결 내용을 사전으로 정리한다. 각 교과가 실제 문제 해결에서 담당하는 역할을 구체적으로 표현한다.
[개념 1] 교과 융합에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 교과 이름의 나열이 아니라 개념의 기능적 연결이 중요하다.

2 실제 문제와 적용 조건

누구의 어떤 문제를 해결하는가

개념 상세 설명: 실제 문제는 대상, 장소, 시간, 제약 조건과 성공 기준이 구체적이어야 한다. 기술이 어떤 상황에서 유용하며 어떤 조건에서는 사용하기 어려운지 함께 밝혀야 한다.

problem = {
    "대상": "교실 사용자",
    "문제": "수업 중 공기질 악화 시점을 알기 어려움",
    "장소": "일반 교실",
    "제약": ["센서 1대", "인터넷 불안정", "수업 방해 최소화"],
    "성공 기준": "기준 초과 후 1분 이내 알림"
}

for key, value in problem.items():
    print(f"{key}: {value}")
코드 상세 설명: 문제를 대상·상황·제약·성공 기준으로 구체화한다. 이렇게 정의하면 기술의 적용 가능성과 한계를 평가하기 쉽다.
[개념 2] 실제 문제 정의에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 실제 적용을 위해 대상, 조건, 제약과 기준을 명확히 해야 한다.

3 시스템 사고

부분의 변화가 전체에 미치는 영향

개념 상세 설명: 시스템 사고는 문제를 하나의 요소로만 보지 않고 서로 연결된 원인, 결과, 되먹임과 부작용까지 살피는 관점이다. 한 부분을 개선했을 때 다른 부분에서 비용이나 위험이 커질 수 있음을 고려한다.

system = {
    "환기 증가": ["공기질 개선", "냉난방 에너지 증가 가능"],
    "센서 확대": ["측정 정확도 향상", "구매 비용과 전자폐기물 증가"],
    "자동 알림": ["빠른 대응", "알림 피로 발생 가능"]
}

for action, effects in system.items():
    print(action)
    for effect in effects:
        print(" -", effect)
코드 상세 설명: 하나의 개선 행동이 여러 긍정적·부정적 영향을 만드는 구조를 나타낸다. 단일 목표만 최적화하는 오류를 줄일 수 있다.
[개념 3] 시스템 사고에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 전체 연결 구조와 예상하지 못한 파급 효과를 함께 본다.

4 이해관계자 분석

누가 혜택과 부담을 받는가

개념 상세 설명: 이해관계자는 기술의 설계·사용·결과에 영향을 받는 사람이나 집단이다. 사용자뿐 아니라 개발자, 관리자, 지역사회, 비용 부담자와 미래 세대까지 살펴야 한다.

stakeholders = [
    {"집단": "학생", "혜택": "빠른 공기질 안내", "위험": "행동 자료 추적 우려"},
    {"집단": "교사", "혜택": "환기 판단 지원", "위험": "알림 관리 부담"},
    {"집단": "학교", "혜택": "환경 관리 개선", "위험": "설치·유지 비용"},
    {"집단": "미래 세대", "혜택": "건강한 환경", "위험": "전자폐기물 증가"}
]

for item in stakeholders:
    print(item)
코드 상세 설명: 집단별 혜택과 위험을 같은 형식으로 정리한다. 기술의 이익과 부담이 고르게 분배되는지 비교할 수 있다.
[개념 4] 이해관계자 분석에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 직접·간접 영향을 받는 집단을 폭넓게 살펴야 한다.

5 지속가능성

환경·경제·사회가 오래 유지될 수 있는가

개념 상세 설명: 지속가능성은 현재의 문제를 해결하면서도 미래의 자원과 기회를 해치지 않는지를 평가하는 개념이다. 에너지 사용, 탄소 배출, 유지 비용, 수리 가능성, 전자폐기물과 사회적 수용성을 함께 본다.

devices = 30
power_watt = 4
hours_per_day = 8
days = 200

energy_kwh = devices * power_watt * hours_per_day * days / 1000
carbon_factor = 0.45
carbon_kg = energy_kwh * carbon_factor

print("연간 전력 사용량:", round(energy_kwh, 1), "kWh")
print("예상 탄소 배출량:", round(carbon_kg, 1), "kgCO2e")
print("개선: 절전 모드, 센서 공유, 수리 가능한 장비 사용")
코드 상세 설명: 장치 수와 사용 시간을 이용해 연간 전력과 예상 탄소 배출량을 계산한다. 정확한 평가는 지역별 전력 배출계수와 장비 수명도 추가로 고려해야 한다.
[개념 5] 지속가능성에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ⑤ — 지속가능성은 여러 세대와 여러 측면의 균형을 본다.

6 공정성과 편향

집단별 결과 차이를 점검한다

개념 상세 설명: 공정성은 기술의 기회, 판단과 혜택이 특정 집단에 부당하게 유리하거나 불리하지 않은지를 살피는 것이다. 전체 정확도만 보지 않고 집단별 오류율과 누락률을 비교해야 한다.

results = {
    "A집단": {"전체": 50, "정확": 45},
    "B집단": {"전체": 50, "정확": 35}
}

for group, value in results.items():
    accuracy = value["정확"] / value["전체"]
    print(group, "정확도:", round(accuracy, 2))

gap = 45/50 - 35/50
print("집단 간 정확도 차이:", round(gap, 2))
코드 상세 설명: 전체 성능이 아니라 집단별 정확도를 따로 계산한다. 차이가 크다면 자료 대표성, 측정 방식과 알고리즘 기준을 다시 점검해야 한다.
[개념 6] 공정성과 편향에 대한 옳은 설명 3개를 고르시오.
정답: ②, ④, ⑤ — 전체 평균 뒤에 숨은 집단별 차이를 살펴야 한다.

7 접근성과 포용성

다양한 사용자가 이용할 수 있는가

개념 상세 설명: 접근성은 장애, 연령, 언어, 기기와 네트워크 환경이 달라도 기술을 이용할 수 있게 설계하는 것이다. 색상만으로 정보를 구분하지 않고 텍스트 설명, 키보드 조작과 쉬운 문장을 제공한다.

interface_check = {
    "색상 외 표시": True,
    "키보드 조작": True,
    "대체 텍스트": True,
    "쉬운 오류 메시지": False,
    "저속 네트워크 지원": False
}

passed = sum(interface_check.values())
total = len(interface_check)

print(f"접근성 점검: {passed}/{total}")
for item, ok in interface_check.items():
    if not ok:
        print("개선 필요:", item)
코드 상세 설명: 접근성 항목을 점검표로 만들고 미충족 항목을 자동으로 찾아낸다. 실제 평가에서는 다양한 사용자가 직접 사용해 보는 시험이 필요하다.
[개념 7] 접근성과 포용성에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 다양한 능력과 환경을 고려해 이용 장벽을 줄인다.

8 개인정보와 데이터 책임

필요한 자료만 안전하게 사용한다

개념 상세 설명: 개인정보 보호는 단순히 이름을 지우는 것에 그치지 않는다. 목적에 필요한 자료만 수집하고 보관 기간을 정하며 식별 가능성을 낮추고 접근 권한을 제한해야 한다.

raw_record = {
    "이름": "홍길동",
    "학번": "1101",
    "온도": 25.4,
    "집중도": 4
}

analysis_record = {
    "가상번호": "S001",
    "온도": raw_record["온도"],
    "집중도": raw_record["집중도"]
}

print("분석용 자료:", analysis_record)
print("수집 원칙: 목적에 필요한 항목만 사용")
print("보관 원칙: 분석 종료 후 식별 정보 삭제")
코드 상세 설명: 분석에 필요하지 않은 이름과 학번을 제외하고 가상번호를 사용한다. 실제 운영에서는 동의, 암호화와 접근 권한 관리가 추가로 필요하다.
[개념 8] 개인정보와 데이터 책임에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ④ — 최소 수집, 분리 보관, 권한과 기간 관리가 중요하다.

9 사회적 비용과 편익

좋은 점과 숨은 비용을 함께 계산한다

개념 상세 설명: 기술의 편익에는 시간 절약, 안전 향상과 정보 접근 개선이 있고, 비용에는 구매비, 유지비, 교육 시간, 개인정보 위험과 환경 부담이 포함된다. 수치화가 어려운 영향도 설명으로 함께 기록해야 한다.

benefits = {
    "관리 시간 절약": 120,
    "에너지 절감": 80,
    "안전 향상 추정": 100
}

costs = {
    "장비 구매": 150,
    "유지 관리": 50,
    "교육 시간": 30
}

total_benefit = sum(benefits.values())
total_cost = sum(costs.values())
net_value = total_benefit - total_cost

print("총 편익:", total_benefit)
print("총 비용:", total_cost)
print("순편익:", net_value)
print("주의: 개인정보와 환경 영향은 별도로 질적 평가")
코드 상세 설명: 금액이나 점수로 표현할 수 있는 편익과 비용을 합산한다. 정량화하기 어려운 가치와 위험은 빠뜨리지 않고 별도의 질적 평가로 남긴다.
[개념 9] 사회적 비용과 편익에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 직접·간접 비용과 편익을 폭넓게 평가한다.

10 책임 있는 기술 개선

효율성과 사회적 가치를 함께 높인다

개념 상세 설명: 책임 있는 기술 개선은 정확도와 속도만 높이는 것이 아니라 투명성, 설명 가능성, 공정성, 개인정보, 접근성, 안전성과 환경 부담을 함께 개선하는 것이다. 목표 간 충돌이 있을 때 우선순위와 근거를 공개해야 한다.

improvements = [
    {"문제": "집단별 정확도 차이", "개선": "자료 대표성 보완과 집단별 성능 공개"},
    {"문제": "전력 사용 증가", "개선": "절전 모드와 처리 주기 조정"},
    {"문제": "설명 부족", "개선": "판단 근거와 한계 표시"},
    {"문제": "접근성 부족", "개선": "키보드 조작과 쉬운 문장 제공"},
    {"문제": "개인정보 위험", "개선": "최소 수집과 보관 기간 단축"}
]

for item in improvements:
    print(item["문제"], "→", item["개선"])
코드 상세 설명: 발견된 사회적 문제와 대응하는 설계 변경을 연결한다. 개선 후에는 이해관계자와 함께 효과와 부작용을 다시 평가해야 한다.
[개념 10] 책임 있는 기술 개선에 대한 옳은 설명 3개를 고르시오.
정답: ①, ④, ⑤ — 기술 성능과 사회적 책임을 함께 높이는 개선이 필요하다.

사회적 평가: 노인·장애인·교통 취약 지역의 접근성을 비교한다.
