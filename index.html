Ⅰ. 연간 운영 개요

수업 목표컴퓨터과학의 개념과 원리를 바탕으로 문제를 구조화하고, 적절한 데이터 구조와 알고리즘을 선택하여 효율적인 프로그램을 구현한다.
수업 운영개념 이해 → 코드 추적 → 구현 실습 → 성능 비교 → 프로젝트 적용의 순환 구조로 진행하며 짝 프로그래밍과 코드 검토를 병행한다.
핵심 역량컴퓨팅 사고력, 협력적 문제 해결력, 정보문화 소양, 창의·융합적 설계 역량을 균형 있게 기른다.
권장 환경Python 3.x, IDLE 또는 VS Code, 온라인 판정 시스템, Git 기반 공유 저장소, 표준 라이브러리와 공개 라이브러리

수행평가 영역별 운영 기준

A · 20%
수업시간 형성·진단 평가

매 차시의 짧은 진단과 형성평가를 누적한다. 정답만이 아니라 풀이 과정, 오류 수정, 질문과 피드백 반영, 실습 참여도를 함께 평가한다.

B · 20%
프로그래밍 분석 평가

주어진 프로그램을 추적하여 결과를 예측하고, 오류 원인을 설명하며, 더 읽기 쉽고 효율적인 코드로 개선하는 능력을 평가한다.

C · 10%
Coding Challenge

문제 해결 전략의 타당성, 코드 정확성, 예외 처리, 시간·공간 효율, 테스트 결과를 종합 평가한다.

D · 10%
정보과학 소논문

연구 문제의 구체성, 이론적 근거, 알고리즘·실험 설계, 결과 해석, 인용과 출처 표시, 정보 윤리 및 한계 성찰을 평가한다.

Ⅳ. 최소 성취수준 보장 및 심화 운영

기초 지원핵심 개념 카드, 코드 빈칸 채우기, 실행 과정 추적표, 짧은 자동 채점 문제를 제공하고 교사 예시와 학생 코드를 단계적으로 비교한다.
심화 확장온라인 판정 문제, 그래프 최단 경로, 고급 동적계획법, 과학 데이터 분석, 오픈소스 기여와 같은 선택 과제를 제공한다.
피드백오류 유형을 개념 오류·설계 오류·구현 오류·검증 오류로 구분하여 다음 학습 행동을 구체적으로 안내한다.
과정 기록개발 일지와 버전 기록을 통해 시도, 실패, 수정, 협력 과정을 평가에 반영하고 결과물만으로 판단하지 않는다.
