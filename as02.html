🔎 핵심 활동

1. 문제 상황 관찰

보이는 사실, 느낀 불편, 궁금한 이유를 구분해 기록한다.

2. 5Why 분석

“왜?”를 반복하되 앞 답변을 근거로 다음 질문을 만들고, 단정 대신 원인 후보를 확장한다.

3. 주제 후보 비교

관심도·자료·기간·정보기술·윤리 기준을 1~5점으로 평가하고 최종 후보를 선택한다.

🧩 학습할 핵심 개념 10개

1 문제 발견

현상을 질문으로

개념 상세 설명: 문제 발견은 불편함이나 이상한 현상을 단순히 말하는 데서 끝나지 않고, 누구에게 어떤 영향을 주며 왜 탐구할 가치가 있는지 찾는 과정이다. 좋은 문제는 관찰 근거가 있고 자료로 살펴볼 수 있으며 개선 가능성이 있다.

observations = [
    "점심시간 뒤 교실이 답답하다",
    "복도 쓰레기통이 자주 넘친다",
    "비 오는 날 통학 시간이 길어진다"
]
for i, item in enumerate(observations, start=1):
    print(i, item)
코드 상세 설명: 관찰한 문제 상황을 리스트로 저장하고 번호를 붙여 출력한다. 문제 발견 단계에서는 평가나 결론보다 다양한 현상을 빠짐없이 기록하는 것이 중요하다.
[개념 1] 문제 발견에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ④ — 문제 발견은 관찰, 대상 이해, 탐구 가능성 판단을 포함한다.

2 사실과 해석

관찰의 정확성

개념 상세 설명: 사실은 관찰하거나 측정한 내용이고, 해석은 그 사실에 의미나 원인을 부여한 판단이다. “온도가 29℃였다”는 사실에 가깝고, “학생들이 게을러서 집중하지 못했다”는 근거가 필요한 해석이다.

record = {
    "사실": "오후 2시 교실 온도는 29℃였다",
    "해석": "높은 온도가 집중도 저하와 관련 있을 수 있다"
}
for key, value in record.items():
    print(key, ":", value)
코드 상세 설명: 딕셔너리로 사실과 해석을 분리한다. 탐구에서는 해석을 사실처럼 단정하지 않고, 추가 자료로 검증해야 한다.
[개념 2] 사실과 해석을 구분하는 태도로 옳은 것 3개를 고르시오.
정답: ①, ②, ⑤ — 사실과 해석을 분리하고 해석을 검증해야 한다.

3 이해관계자

누구의 문제인가

개념 상세 설명: 이해관계자는 문제의 영향을 받거나 해결 과정에 참여하는 사람과 집단이다. 학생, 교사, 학교, 지역 주민, 행정 기관 등이 서로 다른 요구를 가질 수 있으므로 주제 선정 전에 관점을 비교해야 한다.

stakeholders = {
    "학생": "쾌적한 학습 환경",
    "교사": "수업 집중도 향상",
    "학교": "안전하고 효율적인 운영"
}
for person, need in stakeholders.items():
    print(f"{person}의 요구: {need}")
코드 상세 설명: 이해관계자와 요구를 딕셔너리로 연결했다. 같은 문제라도 누구의 관점에서 보느냐에 따라 중요한 변수가 달라질 수 있다.
[개념 3] 이해관계자 분석에 대한 옳은 설명 3개를 고르시오.
정답: ①, ③, ⑤ — 다양한 관점을 확인하면 문제 정의와 윤리 검토가 좋아진다.

4 증상과 원인

겉과 속 구분하기

개념 상세 설명: 증상은 눈에 보이는 문제의 결과이고, 원인은 그 결과를 만들 가능성이 있는 요인이다. 쓰레기통이 넘치는 것은 증상이며, 배출량 증가, 수거 주기, 용량 부족 등이 원인 후보가 될 수 있다.

symptom = "쓰레기통이 자주 넘친다"
causes = ["배출량 증가", "수거 주기 부족", "용량 부족"]
print("증상:", symptom)
for cause in causes:
    print("원인 후보:", cause)
코드 상세 설명: 하나의 증상에 여러 원인 후보를 연결한다. 원인을 하나로 단정하지 않고 자료를 통해 비교하는 태도가 중요하다.
[개념 4] 증상과 원인을 구분하는 방법으로 옳은 것 3개를 고르시오.
정답: ②, ③, ④ — 원인은 후보로 두고 자료를 통해 확인한다.

5 5Why

원인 깊이 탐색

개념 상세 설명: 5Why는 문제에 대해 “왜?”를 반복해 표면적 증상에서 더 근본적인 원인 후보로 이동하는 기법이다. 반드시 정확히 다섯 번 질문해야 하는 규칙이라기보다 충분히 깊게 사고하도록 돕는 틀이다.

why_chain = [
    "왜 교실이 답답한가? → CO2 농도가 높기 때문이다.",
    "왜 CO2가 높은가? → 환기 횟수가 적기 때문이다.",
    "왜 환기가 적은가? → 환기 기준을 알기 어렵기 때문이다."
]
for step, text in enumerate(why_chain, start=1):
    print(f"{step}단계:", text)
코드 상세 설명: 질문과 답을 순서대로 리스트에 저장해 원인 탐색 흐름을 보여 준다. 각 답은 가설이므로 측정 자료로 확인해야 한다.
[개념 5] 5Why 활용에 대한 옳은 설명 3개를 고르시오.
정답: ①, ④, ⑤ — 5Why는 원인 후보를 깊게 탐색하는 도구이며 검증이 필요하다.

6 자료 가능성

탐구할 수 있는가

개념 상세 설명: 흥미로운 문제라도 필요한 자료를 구할 수 없거나 측정이 불가능하면 탐구 주제로 적절하지 않을 수 있다. 공개 자료, 직접 측정, 설문, 문헌 등 가능한 자료원과 품질을 미리 확인해야 한다.

data_sources = {
    "기온": "기상 공공데이터",
    "교실 CO2": "센서 직접 측정",
    "집중도": "익명 설문 또는 행동 지표"
}
for variable, source in data_sources.items():
    print(variable, "→", source)
코드 상세 설명: 탐구 변수와 가능한 자료원을 연결한다. 자료가 없으면 변수나 범위를 바꾸어 주제를 조정해야 한다.
[개념 6] 자료 가능성을 점검하는 방법 3개를 고르시오.
정답: ①, ②, ④ — 자료원, 접근성, 수집 조건을 주제 선정 전에 확인해야 한다.

7 정보기술 적합성

기술은 수단

개념 상세 설명: 정보기술 적합성은 자료 수집, 자동화, 분석, 시각화, 예측, 공유 과정에서 정보기술이 실제로 기여할 수 있는지 판단하는 것이다. 단순히 인공지능이라는 말을 붙이는 것이 아니라 문제와 방법의 연결이 분명해야 한다.

tasks = {
    "수집": "센서 또는 API",
    "분석": "파이썬 통계 처리",
    "표현": "그래프와 대시보드"
}
for stage, technology in tasks.items():
    print(stage, ":", technology)
코드 상세 설명: 탐구 단계별로 사용할 수 있는 정보기술을 연결한다. 기술은 목적이 아니라 문제 해결을 돕는 수단이다.
[개념 7] 정보기술 적합성에 대한 옳은 설명 3개를 고르시오.
정답: ②, ③, ⑤ — 기술은 수집·분석·표현을 돕고 문제와의 관련성이 분명해야 한다.

8 탐구 범위

넓이를 조절하기

개념 상세 설명: 탐구 범위는 대상, 장소, 기간, 변수, 자료량을 조절하여 한 학기 안에 수행 가능한 크기로 만드는 과정이다. “환경 문제 연구”보다 “우리 학교 교실의 시간대별 CO2 변화와 환기 효과 분석”이 구체적이다.

broad = "학교 환경 문제"
narrow = {
    "장소": "1학년 교실 2곳",
    "기간": "2주",
    "변수": ["CO2", "환기 여부"],
    "목표": "환기 전후 변화 비교"
}
print("넓은 주제:", broad)
print("구체화:", narrow)
코드 상세 설명: 넓은 주제를 장소, 기간, 변수, 목표로 나누어 범위를 좁혔다. 범위를 좁히면 자료 수집과 분석 계획이 선명해진다.
[개념 8] 탐구 범위를 적절히 조절하는 방법 3개를 고르시오.
정답: ①, ③, ⑤ — 장소, 기간, 변수, 비교 조건을 조절하면 실행 가능성이 높아진다.

9 주제 후보 비교

근거 있는 선택

개념 상세 설명: 주제 후보 비교는 관심도만으로 선택하지 않고 자료 접근성, 실행 가능성, 정보기술 활용, 사회적 가치, 윤리성 등을 같은 기준으로 점검하는 과정이다. 점수는 결정을 돕는 도구이며 최종 판단에는 설명이 필요하다.

topics = {
    "교실 CO2 분석": [5, 5, 4, 5],
    "전국 미세먼지 예측": [4, 3, 2, 4],
    "급식 잔반 분석": [5, 4, 4, 5]
}
for topic, scores in topics.items():
    print(topic, "총점:", sum(scores))
코드 상세 설명: 각 주제의 기준별 점수를 리스트로 저장하고 합계를 계산한다. 기준의 의미와 점수 근거를 함께 기록해야 공정한 비교가 된다.
[개념 9] 주제 후보 비교에 대한 옳은 설명 3개를 고르시오.
정답: ①, ②, ⑤ — 공통 기준과 설명 가능한 근거가 필요하다.

10 탐구 주제 문장화

무엇을 어떻게

개념 상세 설명: 최종 탐구 주제는 대상, 현상, 핵심 변수, 분석 방법 또는 목적이 드러나도록 작성한다. 주제 문장만 읽어도 무엇을 조사하고 어떤 관계를 살펴볼지 예측할 수 있어야 한다.

target = "학교 교실"
x = "시간대와 환기 여부"

코드 상세 설명: 대상, 독립 변수, 결과 변수, 방법을 문자열로 결합해 구체적인 주제 문장을 만든다. 실제 주제는 탐구 과정에서 자료 상황에 따라 수정할 수 있다.

정답: ②, ④, ⑤ — 대상·변수·방법이 드러나며 필요하면 수정 가능한 주제가 좋다.

📚 학습과 관련한 탐구 보고서 주제 예시 5개

1. 교실 환기 방식에 따른 이산화탄소 농도 변화 분석

시간대와 창문 개방 방식에 따른 CO2 농도를 센서로 측정하고 효과적인 환기 시점을 찾는다.

문제 발견: 오후 교실의 답답함과 집중도 저하

2. 학교 급식 잔반량 증가 원인과 예측 방법 탐구

메뉴, 요일, 기온, 행사 여부를 분석하여 잔반량 증가 원인을 찾고 감축 방안을 제안한다.

문제 발견: 특정 메뉴와 요일의 잔반 증가

3. 통학 시간 지연에 영향을 주는 기상·교통 요인 분석

강수량, 출발 시간, 교통량과 실제 통학 시간을 비교하여 지연 가능성을 예측한다.

문제 발견: 비 오는 날 반복되는 지각 위험

4. 교내 쓰레기통 넘침 현상의 원인과 배치 개선

장소별 배출량, 수거 주기, 쓰레기통 용량을 조사하여 적절한 배치와 수거 기준을 제안한다.

문제 발견: 특정 시간과 장소에서 반복되는 넘침

5. 생성형 인공지능 학습 답변의 오류 유형 분석

교과 질문별 답변을 사실 오류, 출처 부족, 과도한 일반화로 분류하고 검증 절차를 만든다.

문제 발견: 그럴듯하지만 틀린 답변을 구별하기 어려움
